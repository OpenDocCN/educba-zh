## 入侵检测系统简介

IDS 代表入侵检测系统，它是一种用于监视网络或系统中任何不安全活动的设备或应用程序。生成一份报告，由应用程序发送给管理员，或者收集信息并存储在事件管理系统中。有主动和被动 id，网络和主机 id。IDS 主要监控网络流量，因此可以轻松发现恶意活动。它扫描网络或系统，以检查违反策略的情况，并通知有关当局或应用程序。

### 入侵检测系统的分类

IDS 分为两种类型:

HIDS(主机入侵检测系统)
·NIDS(网络入侵检测系统)

#### HIDS(主机入侵检测系统)

主机入侵检测(HIDS)系统运行在不同的主机或网络设备上。HIDS 只跟踪传入和传出的网络数据包，并提醒管理员异常或恶意行为。它将拍摄当前系统文件的照片，并将其与之前的图像进行比较。当设备文件被更改或删除时，将会向管理员发送一个警报以供查看。您可以看到一个在任务关键型系统上使用 HIDS 的示例，它不会改变系统的配置。

#### NIDS(网络入侵检测系统)

网络入侵检测(NIDS)系统设置在网络中的特定位置，用于监控所有网络设备的流量。它监控通过整个子网的流量，并与通过子网的流量相对应，以获取已知的攻击。当检测到攻击或观察到异常活动时，管理员会收到警报。NIDS 的一个例子是在防火墙子网上测试是否有人试图破解防火墙。

### 为什么需要网络入侵检测系统？

没有防火墙是完全安全的，没有网络是不充分的。攻击者不断开发新的专长和技术来阻止您的防御。大多数攻击使用其他社交工程或恶意软件来获取用户凭据，从而为他们提供网络和信息访问权限。允许您检测和响应恶意流量的网络入侵系统(NIDS)对网络安全至关重要。入侵检测系统的主要目标是确保 IT 专业人员了解可能的攻击或网络入侵。网络上的入站和出站流量以及网络中设备之间的数据传输由 NIDS 系统(网络入侵检测系统)控制。

### 对网络 IDS 警报的操作

网络 IDS 对于全面的安全性至关重要，但是为了有效地使用 NIDS，您必须注意几件事情。It 人员是否具备根据网络通知的 IDSs 做出选择并采取所需行动的知识和能力，对于流量监控和评估异常或潜在的恶意活动非常重要。

*   **误报:**基于签名的威胁检测通常是准确的，但在异常检测和识别潜在可疑或恶意活动时，您可能会遇到误报。如果 IDS 网络标记了恶意的正常活动或合法流量，就会出现误报。该计划必须关注常规交通的外观，并且必须进行适当的校准，以忽略合法或经批准的交通入侵检测。
*   **假** **负:**另一方面，您也面临着无法百分之百检测到可疑或恶意活动的风险。它特别关注基于新漏洞和 IDS 未知攻击技术的零日或新兴威胁。
*   **安全** **专家:**除了假阴性和假阳性的变化，网络入侵检测系统面临的最大挑战可能是大量的警告。成功使用网络入侵检测系统的一个关键因素是确保 IT 安全人员经过培训，能够消除错误警报并识别 IDS 可能无法提供的可疑或恶意流量。安全运营中心(SOC)应包括安全专家，他们可以跟踪和分析警告和日志数据，以识别潜在的攻击、确定其优先级，并采取适当的措施来阻止流量或避免攻击。
