## Clojure 和 Scala 的区别

Clojure 是 Java 平台上的现代动态语言，由 Rich Hickey 在 2007 年开发。Clojure vs Scala 因此，Clojure 的问题可以在 Clojure JIRA 项目页面中归档，它由一个社区驱动。该语言是为并发性而开发的。Clojure 遵循了状态不变的身份概念。Scala 是一种通用编程语言，由 Martin Odersky 于 2004 年 1 月首次开发。我们可以同时操作 Scala 和 Java。Scala 翻译成 Java 字节码，使用 Java 虚拟机。这个名字来源于 scalable，它可以随着用户的需求而增长。

### Clojure 和 Scala 的主要区别

让我们讨论一下 Clojure 和 Scala 之间的一些主要区别:

1.  Clojure 是一种动态编程语言，而 Scala 是函数式编程和面向对象编程的结合。
2.  Scala 类似于 Java，而 Clojure 与任何语言都没有相似之处。
3.  Clojure 的语法完全不同，与 lisp 相似，因此用户很难理解。由于 Java 编码，很多人对 Scala 语法很熟悉，因此人们开始使用 Scala 而不是 Clojure。
4.  在 Scala 中，大型项目很容易处理，而在 Clojure 中，维护大型和关键的项目就不容易了。
5.  对于初学者来说，Clojure 代码在程序中的任何地方看起来都是一样的，但是很容易区分 Scala 中的代码行。
6.  由于 Clojure 是动态类型的，因此很难维护大型代码库。但是 Scala 是静态类型的，因此很容易维护大型代码库。
7.  错误消息在 Clojure 中直接显示为堆栈错误跟踪。在 Scala 中，错误信息直接指出改变语法或其他错误。
8.  Clojure 没有特定的环境，依赖于文本编辑器，比如 Emacs，而 Scala 有来自 eclipse 和其他支持 Java 的环境支持。
9.  操作 Clojure 需要热键，对于不了解 lisp 的用户来说，学习热键并不容易。Scala 中不需要热键。
10.  在 Clojure 中优化字节码相当困难，因此 Scala 在字节码方面的性能更好。
11.  由于事务内存和持久数据结构，Clojure 的语言结构与 Scala 相比表现非常好。
12.  一个惰性序列在 Clojure 中实现，而在 Scala 中没有实现。
13.  Clojure 的数值编译有边界检查和没有边界检查，而 Scala 的数值编译没有这么复杂。
14.  默认情况下，Clojure 中的数据结构是不可变的。在 Scala 中，数据结构是可变的。

### Clojure 与 Scala 对照表

让我们讨论一下 Clojure 和 Scala 之间最重要的比较:

| **Clojure** | **Scala** |
| Clojure 是 lisp 的升级版。 | Scala 遵循静态类型。 |
| 编译速度更快。 | 编译速度很慢。 |
| Clojure 不好学。 | Scala 对于懂 Java 的人来说很容易。 |
| Clojure 不向用户销售。 | Scala 的营销很棒，和 Java enterprise 一起营销。 |
| 语法简单且一致。 | 语法不简单，不符合一致性。 |
| Clojure 用逻辑很好理解。 | Scala 很复杂。 |
| Clojure 使用了许多括号，这使得它看起来很困难。 | Scala 限制了括号的使用，用户觉得很容易。 |
| 文档较少，因为它是一种相对较新的语言。 | Scala 有更好的文档，因为它是在 Clojure 之前开发的。 |
| 代码是用有限数量的单词开发的。 | Scala 中单词和词汇的使用非常广泛。 |
| Clojure 不是一种容易阅读的语言。 | 与 Clojure 相比，Scala 可读性更强。 |
| Clojure 语言需要一个编辑器。 | Scala 语言中根本不需要编辑器。 |
| 字节码并不像看起来那么有效。 | 可以使用低位码的有效字节码。 |
| Clojure 需要一些时间来编译，因为编译只有在加载完整代码时才会发生。 | Scala 编译时没有时间延迟，因为代码是逐行编译的。 |
| Clojure 很慢。 | Scala 比 Clojure 快。 |
| 微服务开发是高效的。 | 微服务开发效率不高。 |
| Clojure 的并发性不好。但是它提供了一个并发库 core.async | Scala 提供了一个名为 Akka 的并发库。 |
| 代码是线程安全的。 | 该代码不是线程安全的。 |
| 并发支持来自 Go lang。 | 并发支持是受 Erlang 的启发。 |
| Clojure 中的功能更多。 | Scala 没有这个功能。 |
| 并发库更加灵活，反应更快。 | 并发库不灵活，也不被动。 |
| 可以修改运行代码。 | 不能修改运行代码。 |
| 代码不能重复使用。 | 代码可以重复使用。 |
| 图书馆的存储并不容易。 | 这些库没有太多内存，因为它是轻量级的。 |
| 不使用类也能支持多态性。 | 多态性需要类。 |
| 没有 Apache/Spark 支持，因此机器学习在 Clojure 中很困难。 | Scala 支持 Apache/Spark，我们也可以进行机器学习。 |
| 制造商为 iPhones 提供终身支持。 | 制造商为 iPhones 提供终身支持。 |
| 只有一种代码风格，因此用户不会感到困惑。 | 有许多代码风格会让用户感到困惑。 |
| 任何用户都可以在社区中添加 Clojure 补丁，在 Rich 批准后，它将被实现。 | 任何用户通过社区添加 Scala 补丁都不容易。 |
| 数据结构是使用宏实现的。 | 数据结构实现不需要宏。 |
| 这种语言不太通用。这很简单也很容易。 | 语言更一般化。 |
| 代码作为 lisp 的另一面使用，宏系统功能强大。 | Scala 更像是一种脚本语言，语法简洁。 |
| Clojure 中的模式匹配不好，数据结构是持久性的。 | 这种模式在 Scala 中匹配得很好，数据结构也不是持久的。 |

### 结论

Clojure 和 Scala 都是开源编程语言，Scala 更受欢迎，被许多人使用。两种语言都使用数据结构。Scala 可以作为更好的 Java，人们选择 Scala 作为 Java。两种语言都使用 Java 虚拟机运行，使用相同的字节码，可以获得相同的性能。
