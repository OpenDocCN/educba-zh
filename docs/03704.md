## 机器循环介绍

对于中央处理器(CPU ),机器周期被认为是在计算机处理器中执行机器指令的一系列步骤。机器周期被认为是接收到执行指令所需的步骤列表。机器周期包括执行机器指令所需的四个处理周期。这四个过程包括指令的读取、机器语言的解释、代码的执行和结果的存储。处理器以连续的方式重复执行所有这四个过程，以便可以执行指令。

### 什么是机器周期？

机器周期也称为处理器周期。中央处理器(CPU)是执行机器周期的主要部件，因为它是计算机系统的主要部件。机器周期的主要组成部分是中央处理器和存储器。机器周期被认为是四个不同过程的序列，它们以重复的方式执行一条指令。这四个过程包括:a)获取 b)解码 c)执行 d)存储。在第一次进程获取中，控制单元在主存储器中请求执行第一条指令，该指令作为某个特定地址存储在存储器中，该地址可以容易地被程序计数器(控制单元)识别。得到指令后，下一步是解码输入的指令。CPU 包含用于解码指令寄存器中的指令的控制单元。寄存器是内存中的一个小组件，它执行操作并加速操作的处理。寄存器存储从 CPU 主存储器中取出的指令。指令的解码包括根据指令操作码将操作数字段分解成小的组成部分。操作码是机器语言指令的一部分，它给出关于指令需要执行什么操作的信息。之后，指令由计算机处理器执行。程序计数器用于跟踪指令序列，并在指令执行时跟踪指令地址，以及队列中的下一条指令用于执行。当指令的执行完成时，输出被存储在存储器中以备后用，这样，如果需要的话，可以很容易地取出结果。

### 机器周期中涉及的步骤

以下部分简要讨论了这四个步骤:

#### 1.获取过程

在这个过程中，取出的程序计数器值给出了 CPU 需要执行的指令的地址。从需要由处理器执行的存储器中取出指令。计算机的主存储器存储所有需要处理器执行的指令。控制单元是负责从存储器中取出指令的部件。所有指令都存储在主存中的某个地址位置，可以使用名为指令计数器的组件对其进行跟踪。

#### 2.解码过程

在此过程中，提取的指令被解码以用于指令的执行。解码过程使用指令寄存器内容来解码需要应用于指令的操作类型，并通知 ALU(算术逻辑单元),以便可以对解码的操作数执行该操作。在 CPU 架构中，控制单元负责解码和翻译指令。在解码和翻译指令后，信号被送到 ALU 执行解码的指令。

#### 3.执行流程

在这个过程中，解码后的指令最终由计算机处理器执行。CPU 负责对输入数据(操作数)执行指令。对于计算机指令来说，主要有两部分组成，即操作码和操作数。算术逻辑单元(ALU)是根据操作数(即输入数据)上的操作码执行指令的组件。对于不同的操作，操作码的值是不同的。根据操作码，ALU 对指令执行操作，然后指令被处理器执行。

#### 4.存储过程

在这个过程中，执行结果存储在内存中。一旦算术逻辑单元对数据进行操作，操作的结果和处理的数据被发送到主存储器，用于结果的恢复。存储器(RAM)是这个过程中的主要组件，因为结果被存储起来供以后使用。为了在屏幕上显示输出，结果从主存储器中取出，或者可以根据需要用于一些其他操作。并且在使用后删除存储的数据，并且对于下一个指令再次重复整个过程。所有这些进程对处理器都至关重要，因为处理器的处理速度和性能取决于这四个进程。

### 为什么我们需要机器周期？

要求机器周期告诉指令执行的完整流程，以便可以更好地理解指令执行的内部，如果有任何要求，可以进一步改进。机器循环是执行指令的连续步骤，如果不这样，指令将不能正确执行，从而产生错误的结果。处理器执行机器指令需要机器周期。

#### 机器周期的重要性

机器周期对于计算机处理器执行机器指令很重要，因为设备性能是基于机器周期的。如果指令执行不当，处理器的性能可能会降低。设备的处理速度和性能是通过它在给定时间跨度内可以执行的指令数量来计算的。

### 结论

机器周期有四个过程，即获取过程、解码过程、执行过程和存储过程。所有这些过程都是处理器执行指令所必需的。通过机器周期，可以计算处理器的性能并选择器件。
