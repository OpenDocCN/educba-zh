## Logstash 替代方案介绍

Logstash 是一种基于过滤器/管道模式编译、排序和创建日志或事件的方法。它允许实时集中和解释来自多个出口的日志和事件。Logstash 是用 JRuby JVM 编程语言编写的，所以可以在多个平台上运行 Logstash。它从几乎任何来源收集各种类型的信息，如记录、传输、数据包、事件、时间戳数据等。

### Logstash 的替代品

让我们讨论 Logstash 的一些替代方案。

<small>Hadoop、数据科学、统计学&其他</small>

#### 1.卡夫卡

Apache Kafka 是一个分布式社区事件流应用程序，每天管理数万亿个事件。Kafka 最初构建为一个消息队列，是一个分布式提交日志抽象。自 2011 年由 LinkedIn 开发并开放以来，Kafka 已经从一个消息队列迅速发展成为一个完整的事件流媒体网络。汇合是由 Apache Kafka 的最初设计师开发的，为 Kafka 提供了最完整的交付汇合框架。Kafka 的融合界面通过附加的社交和商业功能扩展了运营商和开发者在开发中的广播体验。

#### 2.Splunk

Splunk 是一个管理计算机数据和其他广泛数据形式并提供洞察的程序。运行 Web 服务器、IOT 计算机和移动应用程序日志的 CPU 从系统中创建这些数据。这些数据不得提供给最终消费者，也没有商业价值。然而，对设备的理解、控制和输出优化是非常重要的。Splunk 可以读取这些非结构化、半结构化或偶尔结构化的知识。它在读取数据后对这些数据进行扫描、标记、报告和仪表板。

#### 3.灰色日志

Graylog 是一个日志处理框架，用于存储、索引和分析几乎所有来源的结构化和非结构化数据。今天，大多数系统位于采用微服务架构的各种机器上，具有多个微服务。因此，用户每次从每个微服务获取不同设备上的日志会花费大量时间。为了防止这种情况，我们有一个名为 Graylog 的应用程序，它通过在一个位置设置所有微服务的日志来工作。

#### 4.弹性搜索

Elasticsearch 是一个面向所有数据类型的开源搜索和分析引擎，包括文本、数字、地理空间、结构化和非结构化数据。基于 Apache Lucene，Elasticsearch N.V .于 2010 年首次发布。(弹性，现在已知)。Elasticsearch 是 Elastic Stack 的一个已知组件，这是一个用于数据摄取、增强、存储、分析和查看的开源工具集，这要归功于它的简单、REST APIs、分布式设计、可伸缩性和速度。Elastic Stack 现在包含大量轻量级传输代理，称为 Beats，用于向 Elasticsearch 提供数据，通常称为 ELKStack。

#### 5.卡夫卡

Apache Kafka 是用 Scala 和 Java 开发的，是一个开源的分布式流队列框架。Apache Kafka 在队列中按顺序用于发布和订阅消息。Kafka 具有高性能、可靠性和复制功能，是一个快速、可伸缩和健壮的发布订阅者消息传递系统，具有容错功能。

#### 6\. Filebeat

Filebeat 提供了一种传输和集中日志和数据的简单方法，因此当您有多个服务器、虚拟机和日志容器时，SSH 是不可接受的。Filebeat 的其他优势包括处理大量文件、支持加密和有效管理背压的能力。Filebeat 基本上是一个日志代理，它创建日志文件并将其发送到 Logstash 以进行进一步处理，或者发送到 elasticsearch 以在计算机上进行索引。

#### 7.Stackify

Stackify 表示每个人都正常操作。我们的目标是为开发人员提供他们需要的资源，以增强故障排除和跟踪他们生成的应用程序和服务器。传统的 IT 操作工具是为网络和设备管理人员设计的。开发人员通常花 30%的时间处理 IT 运营，以修复应用程序。开发者实际上没有工具来有效地支持他们的软件。

#### 8.水道

Apache Flume 是一个机制服务/工具/数据摄取，用于从不同的源收集大量的流数据，并将其传送到一个集中的数据存储，如日志文件和事件等。Flume 是一个具有高度可靠性、分布性和配置性的工具。它主要用于将流信息(日志数据)从不同的 web 服务器复制到 HDFS。
