## ASP.Net 数据绑定简介

同样，ASP.Net 也支持数据绑定的概念。通过数据绑定，您可以将任何服务器控件绑定到各种属性、表达式、集合甚至方法。还可以灵活地选择数据源，可以是数据库、静态文件、enum 或任何其他源。因此，ASP.Net 数据绑定提供了一种简单方便的方法来创建服务器控件和应用程序数据之间的链接。

让我们理解图像中使用的术语。

*   **绑定源**–保存业务数据的应用数据变量。
*   **绑定目标**–数据绑定到的服务器控件或 UI 元素。
*   **依赖对象**–与绑定目标相关联的对象。
*   **依赖属性**–数据绑定到的依赖对象的属性或特性。
*   **源对象**–保存业务数据的对象。
*   **源属性**–值为实际数据的源对象的属性或特性。

因此，如果您想将 UI 上输入控件的 text 属性绑定到雇员姓名，那么 TextBox 就是目标/依赖对象。文本是目标属性。员工是业务对象，姓名是财产。

绑定的方向决定了数据在源和目标之间的来回传播。在单向绑定中，业务数据的任何变化都会反映在 UI 元素中，但反之亦然。当业务数据是只读的，并且 UI 控件本质上是信息性的时，这很有帮助。

双向绑定确保源或目标对象中的数据更改自动同步。这就创建了完全交互式的 UI 表单，表单字段中的更改会在后端变量中即时更新。

在单向源绑定中，UI 元素数据中的任何更改都会在业务对象中更新，但反之亦然。这在基于其他用户输入计算数据并需要频繁重新评估的情况下很有帮助。

### 什么是数据绑定？

*   在我们继续之前，让我们先了解一下数据绑定的一般概念。数据绑定是将应用程序用户界面与业务数据连接起来的过程。现在您可能想知道这与在 UI 上显示业务数据是一样的。不，不是的。数据改了怎么办？它会在用户界面上自动更新吗？
*   因此，当数据绑定正确完成时，用户界面上的元素会自动反映业务数据的变化。假设您的 UI 元素一直在观察分配给它们的数据变量/集合。一旦数据发生任何变化，UI 元素就会被指示相应地反映这种变化。
*   这种现象的一个真实例子是我们智能手机中的计算器。它们被设计成一旦用户键入任何值就给出结果。请注意下面的截图，结果是如何实时产生和显示的。而不需要给出“等于”命令。

### 如何在 ASP.Net 创建数据绑定？

下面是一个简单的例子来演示 ASP.Net 的数据绑定。我们将创建一个简单的 WPF 应用程序，在用户界面上有三个控件(文本块、滑块和进度条)。两个控件的值将绑定到第三个控件中的数据。

第一步:打开 Visual Studio，**新建一个项目。**

<noscript><img class="alignnone wp-image-217472 size-full" src="../Images/ab6c4d9f799dcb01116a525987d28396.png" alt="click create after confrigure" width="407" height="295" srcset="https://cdn.educba.com/academy/wp-content/uploads/2019/09/click-create-after-confrigration.png 407w, https://cdn.educba.com/academy/wp-content/uploads/2019/09/click-create-after-confrigration-300x217.png 300w" sizes="(max-width: 407px) 100vw, 407px" data-original-src="https://cdn.educba.com/academy/wp-content/uploads/2019/09/click-create-after-confrigration.png"/></noscript>

<noscript><img class="alignnone wp-image-217474 size-full" src="../Images/6fc32cd76408cf0ffc20c08f5b890bd1.png" alt="a spell appication created (Data Binding in ASP.NET)" width="608" height="445" srcset="https://cdn.educba.com/academy/wp-content/uploads/2019/09/a-spell-appication-created.png 608w, https://cdn.educba.com/academy/wp-content/uploads/2019/09/a-spell-appication-created-300x220.png 300w" sizes="(max-width: 608px) 100vw, 608px" data-original-src="https://cdn.educba.com/academy/wp-content/uploads/2019/09/a-spell-appication-created.png"/></noscript>

**第五步:**现在添加图中带下划线的**代码**绑定 TextBlock 的 Text 属性。

`Text="{Binding Value, ElementName=slider}"`

同样，对于 ProgressBar 的 Value 属性。

`Value="{Binding Value, ElementName=slider}"`

这将属性绑定到 Slider 的 Value 属性。

![code added to the properties (Data Binding in ASP.NET)](../Images/e3f316545025864c7aed864d0f258569.png)

<noscript><img class="alignnone wp-image-217478 size-full" src="../Images/e3f316545025864c7aed864d0f258569.png" alt="code added to the properties (Data Binding in ASP.NET)" width="486" height="352" srcset="https://cdn.educba.com/academy/wp-content/uploads/2019/09/code-added-to-the-properties.png 486w, https://cdn.educba.com/academy/wp-content/uploads/2019/09/code-added-to-the-properties-300x217.png 300w" sizes="(max-width: 486px) 100vw, 486px" data-original-src="https://cdn.educba.com/academy/wp-content/uploads/2019/09/code-added-to-the-properties.png"/></noscript>

第六步:运行应用程序。你会看到滑块控件现在更新了文本块和进度条的值。移动滑块以查看实际效果。

让我们理解例子中使用的术语。

*   **绑定源**–UI 中的滑块控件。
*   **绑定目标**–text block 和 ProgressBar
*   **依赖对象**–创建的 TextBlock 和 ProgressBar 对象。
*   **依赖属性**–Text block 的 Text 属性和 ProgressBar 的 Value 属性。
*   **源对象**–用 x: Name="slider "创建的 Slider 对象。
*   **Source 属性**–滑块的 Value 属性。

**第七步:**最后的**代码**应该如下:

```
<Window x:Class="DataBindingInASP.Net.MainWindow"

xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
xmlns:d="http://schemas.microsoft.com/expression/blend/2008"
xmlns:mc="http://schemas.openxmlformats.org/markup-compatibility/2006"
xmlns:local="clr-namespace:DataBindingInASP.Net"
mc:Ignorable="d"
Title="MainWindow" Height="450" Width="800">
<Grid>
<TextBlock HorizontalAlignment="Left" Margin="100,100,0,0" TextWrapping="Wrap" VerticalAlignment="Top"
Text="{Binding Value, ElementName=slider}" />
<Slider x:Name="slider" HorizontalAlignment="Left" Margin="100,200,0,0" VerticalAlignment="Top" Maximum="100"
Width="250" Height="25"/>
<ProgressBar HorizontalAlignment="Left" Height="25" Margin="100,300,0,0" VerticalAlignment="Top" Width="250"
Value="{Binding Value, ElementName=slider}"/>
</Grid>
</Window>
```
