## XHTML 和 HTML 的区别

在 Web 技术中，我们都使用 html 作为前端技术，它代表超文本标记语言，包含预定义的标签、元素和具有元素行为的属性。它还结合了被称为 css 的表示技术，它将与 html 相协调，并提供更有吸引力的网页。XHTML 被称为 EXtensibe 超文本标记语言也是标记语言之一，它几乎与 HTML 完全相同。与 html 相比，它在网页中更困难，因为 XHTML 主要被定义为 xml 也与 html 代码配对。在这个主题中，我们将学习 XHTML 和 HTML。

### XHTML 和 HTML 的主要区别

下面提到了主要差异:

每种基于网络的技术都有一些相似的不同之处。这两种标记之间也有一些关键的区别，languages.Html 通常遵循一些规则来声明标记和元素，这些规则将与预定义的根元素行为及其功能相适应。但是在 XHTML 中，对于在 HTML 中分配和声明标签有一些严格的规则和规定。

Html 和 xhtml 都是基于网络的技术，可以用来编写网页上的代码。但是 html 完全基于基于 SGML(标准通用标记语言)的语言，并且它遵循 DTD(文档类型定义)模型，它完全基于文档结构而不是特征。xhtml 是 html 中基于 xml 的特性，它得到了 html 和 xml 标准的确认。与 html 相比，代码会更加复杂，而且它已经指定了一些严格的规则来开发网页。

当我们使用 xhtml 时，它被开发为减少了浏览器特定的标签，但在 html 代码中，它可能会根据用户浏览器的不同而有所变化。与 xhtml 相比，标签和元素也增加了。我们已经知道 xml 是一种标记语言，我们都在网页请求和服务器响应中使用过它。它有一些格式良好的文档模式。我们可以使用 xml 来描述我们在 web 文档中编写的任何 web 数据，但是我们使用 html 来在 web 浏览器中显示我们在 web 文档中设计和记录的任何数据。

Xhtml 网页用于读取所有支持 xml 的设备，也包含在支持 xml 语言的浏览器中。xhtml 元素将与其他受支持的元素嵌套使用。它与开始和结束标记系统 xhtml 标记元素具有相同的基本规则，总是使用小写字母，而且它必须有一个根元素。属性名必须是小写字母，它们的值应该用引号括起来，xhtml 支持 dtd 类型文档中的强制元素。
