## 渗透测试简介面试问题和答案

渗透测试也称为笔测试。这是一种用于测试系统或 web 应用程序安全级别的测试。它用于了解系统功能的弱点或漏洞，也有助于获得目标系统风险评估的完整细节。这是一个包含在完整的系统安全审计中的过程。渗透测试有两种类型，即[白盒测试](https://www.educba.com/white-box-testing/)或[黑盒测试](https://www.educba.com/black-box-testing/)。渗透测试将决定系统安全性的强度。根据要测试的应用程序的类型，有不同的工具来执行这种渗透测试。

### 第 1 部分-渗透测试面试问题(基础)

第一部分包括基本的渗透测试面试问题和答案。

#### Q1。什么是渗透测试，它有什么用？

**答:**
渗透测试也称为 Pen 测试，是一种对 web 应用或系统的网络攻击，目的可能是好的，也可能是坏的。就不良意图而言，它是一种对系统的网络攻击，以窃取某种安全、机密和敏感的信息。就良好意图而言，它是一种检查系统对漏洞和外部攻击的优势和劣势以及它可以处理的安全级别的强度。

#### Q2。渗透测试的优势是什么？

**答案:**
这是面试中常见的渗透测试面试问题。在系统上执行渗透测试的优势是

1.  它将有助于检测系统或 web 应用程序的安全威胁和漏洞。
2.  这将有助于监测必要的标准，以逃避一些。
3.  如果通过渗透到应用程序中而将大量流量转移到网络，这有助于减少应用程序的停机时间。
4.  它保护组织的机密和安全信息，并维护品牌形象或价值。
5.  保护应用程序以避免巨大的经济损失是很重要的。
6.  更加关注业务连续性。
7.  保持客户之间的信任。

#### Q3。渗透测试有哪些不同的阶段？

**答:**
对目标系统或 web 应用进行渗透测试有规划侦察、扫描、获取访问、维护访问、分析和配置等不同阶段:

1.  **计划和侦察**:在这一阶段，对要实现的目标进行分析和测试，并收集信息。
2.  **扫描:**在这个阶段，使用任何一种扫描工具来测试目标系统在入侵者入侵情况下的响应能力。
3.  **获取访问权限:**在此阶段，将执行渗透或入侵攻击，攻击 web 应用程序以暴露系统可能的漏洞。
4.  **维护访问权限:**在此阶段，将仔细维护获得的访问权限，以识别系统的漏洞和弱点。
5.  **分析和配置:**在此阶段，从维护的访问中获得的结果也将用于配置 Web 应用防火墙设置。

#### Q4。Scrum 的需求是什么？

**回答:**
下面是 Scrum 的几个需求列表，但不是全部:

1.  它需要用户故事来描述需求，并跟踪分配给团队成员的用户故事的完成状态，而用例是更老的概念。
2.  名称是必需的，因为它以单行概述的形式描述了一个句子，以给出用户故事的简单解释。

#### Q5。有哪些不同的渗透测试方法？

**答案:**
不同的渗透测试方法有外部测试、内部测试、盲测、双盲测试、定向测试。外部测试是对公众可见的互联网网站、电子邮件应用程序和 DNS 服务器等进行测试的一种形式。内部测试是一种通过网络钓鱼或内部攻击的形式渗透到系统内部应用程序的测试。盲测试是一种以实时可能性的形式渗透到基于其名称的应用程序中的形式。双盲测试是一种测试形式，其中甚至连应用程序的名称也是未知的，甚至安全专业人员也不知道在特定目标上执行的任何想法，而目标测试是一种安全专业人员和测试人员以互相瞄准的形式一起执行测试的形式。

现在让我们来看看高级渗透测试面试问题。

#### Q6。什么是跨站脚本(XSS)？

**答:**
[跨站点脚本是一种攻击](https://www.educba.com/what-is-cross-site-scripting/)，以注入 [web 应用或系统](https://www.educba.com/how-to-build-web-applications-using-mongodb/)的形式出现。在这种情况下，不同类型的恶意脚本被注入到脆弱的系统中，以在系统管理员不知情的情况下获取机密信息或入侵系统。

#### Q7。什么是入侵检测？

**答:**
入侵者检测机制将通过扫描应用程序文件系统中的记录形式的现有文件来帮助检测可能发生的攻击。这将有助于组织及早发现对其系统应用程序的攻击。
