### Tableau 与 Power BI 和 QlikView 的主要区别

我们来讨论一下 Tableau 与 Power BI 和 QlikView 的顶级对比:

1.  Tableau 的使用不需要任何预先的技术知识。简单的拖放界面使这成为可能。与此相反，为了有效地使用 Power BI，需要了解 Excel 和 DAX(数据分析和表达)。QlikView 很难操作，因为必须掌握脚本知识并熟悉数据建模。
2.  Tableau 是一个纯粹的可视化工具。如果唯一的目的是为分析生成可视化，那么 Tableau 最适合这个目的。Power BI 不仅仅是一个可视化工具，因为它还提供了出色的预测建模和 BI 功能。QlikView 的独特之处在于其深入功能，让我们将数据发现的见解融入可视化分析中。
3.  Tableau 为高级分析提供了某些内置功能，如时间序列建模。它还支持与外部工具的集成。然而，在高级分析方面，Power BI 超越了其他 BI 工具。另一方面，QlikView 在此参数上无法与这两种工具相匹配。
4.  在性能方面，Tableau 的简单用户界面支持快速开发。Power BI 虽然相对较新，但它提供了无缝选项来生成各种高级分析，因此就性能而言，BI 工具已迅速成为最受欢迎的工具。QlikView 在 BI 方面高于 Tableau，但低于 Power BI，在可视化方面则相反。
5.  在数据连接和提取的上下文中，Tableau 使用了 cubing 技术并拥有自己的数据仓库。Power BI 不需要额外的模型，因此可以与数据建立直接连接。QlikView 根据内存技术原理处理数据。
6.  Tableau 根据用户要求提供各种选项。它拥有庞大的用户群，因此这导致它提出了各种各样的选择。定价因版本而异，而且定价结构很复杂。Power BI 仅提供两种选择，即。Power BI Pro 和 Power BI Premium。Power BI 的定价相对较低。QlikView 相对来说是一个老工具，它主要为小型企业和大型企业提供两种选择。某些版本的价格是固定的，而其他版本的价格则根据销售情况而变化。
7.  Tableau 有效地集成了像 R 这样的外部统计包，可视化可以集成到工具中。然而，Power BI 是集成的最佳工具，也留下了 Tableau。它集成了许多外部资源，如 MS Excel、XML、JSON、SQL、Google Analytics、Marketo、脸书、Hadoop、R、Spark、Python 等。与 Tableau 和 Power BI 相比，QlikView 在集成方面较弱。
8.  Tableau 最适合在其领域工作中需要 BI 工具的用户，例如研究人员。Power BI 非常适合仪表板开发，因为它集成了高级统计和可视化分析功能的几个特性。QlikView 允许共享分析，其数据发现功能最适合大型企业的内部团队。
9.  谈到每种工具的优势，Tableau 提供了比任何其他 BI 工具都出色的视觉功能。Power BI 除了提供各种功能外，价格也相对较低。QlikView 有一个发展良好的社区，因此获得了很高的评价。

### 结论

Tableau、Power BI 和 QlikView 都是功能强大的 BI 工具。他们每个人都擅长某些参数。这些工具会根据竞争对手的工具评估和用户需求不断更新。每个工具都有一个独特的销售主张，可以吸引相似的用户。
