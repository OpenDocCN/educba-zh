## 吉拉出口介绍

吉拉用于在整个开发生命周期中跟踪整个项目。它向用户提供不同的特征；出口是吉拉提供的功能之一。在吉拉，我们可以根据需要将问题导出为其他文件格式，如 Excel、PDF 或 CSV，以便使用过滤器搜索相关问题。我们知道，导出具有不同的特性，这是一个受欢迎的选项，它支持根据我们的需求执行各种操作。比如有时候我们需要共享报表，或者我们可以说当时信息是另一种格式的；我们可以使用导出选项。

### 吉拉出口概述

每当我们从事出口业务时，一大堆社会事务会成为你一天中的重要部分。在日常的单口相声或数周的回顾中，报告是提供最新信息和展示聚会以及我们在事情进展中的同谋的核心。理想情况下，我们接受这些报告应该以我们想要的方式显示我们想要显示的数据。虽然吉拉社区有公开决策(董事会指定和标准仪表板报告)，但这些决策是有限的，不通用。Atlassian 明白无数的客户会要求更多。

<small>网页开发、编程语言、软件测试&其他</small>

虽然最好的决定是从 Atlassian Marketplace 购买一个值得注意的应用程序，在吉拉制作可移动的框架，但许多聚会都更适合像 Microsoft Excel 和 Google Sheets 这样的普通评估表制作商。记住，Atlassian 制造了与 Microsoft Excel 和 Google Sheets 的混合，因此客户可以在他们最熟悉的阶段制作他们的吉拉数据的图表和表格。

通过交换问题，您可以将信息以 Excel 或 PDF 报告的形式提供给外部同谋。将数据放在 Excel 中对于评估是有价值的，并且可能是你的考官所期望的，这样他们可以继续前进。交换问题有助于消除与 SLA 和审查目的的一致性。最后，问题交换背后的最后一个合法支持是开发另一个系统，而不失去对来自前者的信息的归纳。

传达吉拉议题是一项简单的任务。吉拉客户可以从吉拉的三项直接进展中兑换积分。所有需要做的事情是在负责的飞行员中搜索问题，点击项目按钮，并选择正确的安排。

### 吉拉出口需求

客户表示，他们发出吉拉问题，主要是因为他们需要独特地对待数据，这与吉拉提供的不同。因此，首先，我们将展示促使个人从吉拉发出这些信息的普通情况。稍后，我们将检查交易问题及其项目的各种方式。

有时候我们需要和外部的人分享信息，分享他们想要的东西；我们可以根据他们的要求导出问题报告。为了分析信息，我们需要相同格式的数据；这时，我们可以使用导出选项，它还可以帮助管理人员跟踪项目，并可用于审计。有时我们需要将问题转移到不同的实例中。这时候，最好的选择就是导出数据。
