## 应用程序监控工具简介

APM 确保提供符合特定标准的客户服务，并识别与应用程序相关的问题。可以使用各种类别跟踪或监控应用程序输出:加载时间、应用程序响应时间等。随着技术的使用，这些技术现在变得越来越复杂和分散。为了提高最终用户的满意度，跟踪应用程序的性能也很重要。应用程序监控包括 web 请求、CPU、事务和内存使用、应用程序错误、应用程序性能监控。在本主题中，我们将了解应用监控工具。

### 应用程序监控工具列表

以下是应用程序监控工具的列表

<small>网页开发、编程语言、软件测试&其他</small>

#### 1.应用动力学

AppDynamics 为大型公司提供 SaaS APM 解决方案和现场替代方案。AppDynamics 自称是一个应用智能论坛，它跟踪产品性能，然后就应用成功对业务运营的影响提供见解。AppDynamics 让您全面了解应用程序输出如何影响您的业务，从数据收集到分析以及随后从数据中获得的知识。

#### 2.Stackify 回描

Retrace 是为开发人员设计的，是一款廉价的 SaaS APM 工具。它旨在使开发人员能够提高 QA 性能，并通过代码级的非常详细的事务“追溯”他们应用程序中的开发问题。Prefix 通过使用一个可自由访问的工作站级 APM 工具来帮助开发人员编写和检查他们的代码。对于各种规模的开发团队来说，Retrace 的目标是易于使用且价格合理。

#### 3.Dynatrace

第一个自学应用程序管理工具是 DynaTrace，以前称为 Compuware APM。通过其处理程序，应用程序及其组件可以在拓扑结构中自动可视化。这使 DynaTrace 成为一种成功的方法。不过，有一点要记住，学习需要时间。因此，您可能需要等到有足够的数据点时，才能避免看到假阳性项目。

#### 4.河床钢铁中心

另一个企业级 APM 解决方案是 Riverbed 的 SteelCentral。为了从其他奇异解获得同样详细的结果，需要河床的几个组成部分。SteelCentral 的响应、内部应用程序和门户网站都是获得其他许多网站的全局视图所必需的。当你意识到虚拟设备和物理设备都参与其中时，我发现这种对消费者的分析非常有用。直观使用:GUI 不直观，其设计的不同元素在外观和功能上不同于其他界面部分。

#### 5.SCOM

开箱即用的 APM 方法，SCOM 是好的。SCOM 在 APM 环境中提供了巨大的价值，因为总体工作是创建适当的管理包。然而，体面的治疗和营养标准对保持其效用是重要的。要获得直观的仪表板，要么需要大量的工作，要么需要第三方插件。

#### 6.戴尔 Foglight

从 box research 的仪表板提供用户体验跟踪、数据库和应用程序的交叉连接映射。Quest 为 APMs 的需求提供了一个强大的框架，但是代码可能会令人沮丧，并且很难找到您想要的信息。

#### 7.珍妮弗·索菲特

JenniferSoft APM 解决方案提供了其他标准的 APM 功能，可以在墙上显示拓扑视图和实时仪表板。它还计划成为一个低开销的解决方案，但没有指定。这意味着他们建造它是为了尽可能少的占用资源。

#### 8.纳斯特尔

对于深度 APM 审查和探索，Nastel 提供了另一种开箱即用的解决方案。但是像大多数其他应用程序性能管理供应商一样，Nastel 是一个基于中间件的企业事务平台，并专注于中间件事务。

#### 9.国际商用机器公司

半个多世纪以来，IBM 一直是企业级解决方案的主力。您正在为您的应用程序性能管理提供您所期望的东西。IBM 提供了一个健壮的 SaaS 包，包含几个级别，所以您可以想走多远就走多远。数据库、代码堆栈跟踪和自动挂起事务解决在最低级别上是可能的。

#### 10.智能熊

在明确针对 REST 和 SOAP API 的 AlertSite 产品中，SmartBear 吸收了 Lucierna 的 APM。这种价格限制使它成为 APM 利基市场的一种商品——以小市场为目标。然而，SmartBear 即将扩展这一产品，为应用程序管理的供应商创造一个重要的参与者。

#### 11 .想法精确

IDERA 的深度监控能力提高了它的声誉。Precise 也不例外:IDERA 通过使用深度数据库结构扩展了 precision，以包括真正的 APM 解决方案。此外，IDERA 以提供直观的仪表板而闻名，该仪表板提供快速的洞察力，并且通过使用这些仪表板是当今可用的最好的 APM 监控工具之一。
