## 介绍如何避免影子 IT 的陷阱并提高性能

公司的 IT 基础架构由硬件、服务器、操作系统、软件和应用程序组成。硬件部门可能由系统和网络管理员管理，而软件应用的[项目经理](https://www.educba.com/project-manager-jobs/)可能负责具体部门和项目。每个部门都有单独的负责人来管理运营，只有在 IT 部门批准的情况下，才能添加额外的硬件、软件应用程序或设备。

然而，随着 web 的出现以及一些软件服务提供商提供了使用软件和应用程序的选项，包括免费下载或访问云，许多员工可能出于各种目的在订阅的基础上下载或访问软件。在 IT 管理员不知情的情况下安装或使用的此类应用程序构成了影子 IT 的一部分。

随着云和[软件即服务(SaaS)](https://www.educba.com/what-is-software-as-a-service-saas/) 的日益流行，许多经理可能倾向于走使用影子 IT 的简单路线，而不是在组织内创建一张让 IT 部门响应、评估和实施的票证。

### 使用 Shadow IT 的相关危险和一些步骤。

以下是与使用影子 IT 相关的危险，以及一些有助于提高企业绩效的步骤:

#### 1.缺乏现代化和执行缓慢

如果越来越多的经理和部门使用影子 it，这可能反映了组织内 IT 部门的效率低下。或者 IT 团队本身可能对部门提出的问题单(议题或问题)反应缓慢。在某些情况下，交付解决方案可能需要几周甚至几个月的时间。这可能会导致更多的员工寻求云服务来完成工作。

营销云应用程序，如用于文件共享的 [Netsuite](https://www.netsuite.com/portal/home.shtml) 、 [Salesforce](https://www.salesforce.com/in/) 、 [DropBox](https://www.dropbox.com/?landing=dbv2) 、 [Google Docs](https://www.google.com/docs/about/) 、用于工作的 [Yammer](https://www.yammer.com/) 、 [Asana](https://asana.com/) 和项目协作，已经动态发展，以提供可以在智能手机上访问的基于网络和移动的服务。

指责 It 部门实施缓慢或缺乏创新精神很容易，但在每个部门盲目增加更多基于云的应用程序孤岛肯定不是解决这个问题的答案。每个部门主管或领导应与相关 IT 经理或领导进行公开对话，以解决问题。如果每个部门都使用不在本部门之外协作的影子 IT，这可能会对组织的效率产生更大的影响。此外，与 IT 部门主管进行对话将有助于高层管理人员了解更多投资的需求，也许是为了获得人才或更好的工具来创建新的软件解决方案。

简而言之，由于缺乏 IT 计划的集中化，组织孤岛成倍增加，导致部门之间缺乏控制和协作。这是企业需要解决的问题，尽管使用影子 IT 会带来一些短期好处——降低 IT 成本、提高灵活性、快速完成任务和应用程序的实施。

#### 2.影子 IT 增加了安全风险

由于没有集中控制和影子 IT 的部署越来越多，组织面临的安全威胁越来越大。甚至像 Dropbox 这样的文件共享应用程序也可能涉及公司的专有数据或知识产权(IP ),这些数据或知识产权可能会在公司外部共享，或者落入他人之手。

这种应用程序孤岛不在内部防火墙的范围内，因此容易受到黑客攻击。如果他们的客户数据在没有适当控制的情况下被共享，并且在出现安全违规的情况下，监管机构会因为与数据合规性相关的问题而对其处以高额罚款，监管限制可能会发生，从而导致消费者不信任和满意度下降。因此，各部门的经理和团队领导应意识到使用第三方软件增加的风险，并告知 IT 部门此类应用的使用情况。

在金融、医疗服务、投资和银行业等一些行业，未能遵守法规可能代价高昂，并限制组织的发展能力。它使组织面临漏洞和潜在的数据丢失。

同样，一些员工可能倾向于使用容易被猜到的密码，或者在多个应用程序中使用相同的密码，从而使他们面临威胁。当[网络罪犯](https://www.educba.com/cyber-security-tools/)设法进入一个这样的应用程序时，他们就更容易入侵整个网络。

根据 IBM 的一项安全研究，如果网络犯罪分子攻击第三方云应用程序，他们可以窃取有价值的公司数据和凭据，直接访问公司的网络。此外，它们可能很难追踪，因为它们来自第三方系统，而不是直接进入公司网络。

#### 3.重复的应用程序和缺乏内部支持

当部门依赖更多的云应用程序时，可能会导致不同的组拥有不同的管理员(admins)而导致应用程序重复。这导致成本增加和缺乏协作。如果各部门常用的应用程序是在一个集体计划下实现的，这是可以避免的。当太多的云应用以分散的方式使用时，如果服务提供商不提供及时的支持，他们可能会面临问题。此外，内部团队可能缺乏解决相关问题的技能。因此，最好与 it 部门进行对话，并获得有关应用程序的建议，这些应用程序最好与现有基础架构相集成，并具有更好的服务支持。

#### 4.审查现有工具和政策

一开始，有人提到影子 it 使用的增加反映了内部 IT 部门缺乏效率。改进现有 IT 系统的一种方法是找出员工的痛点和系统中的漏洞，促使他们寻找基于云的替代方案。如果可以在内部开发一个更简单的替代方案，可以在内部管理，这可能是管理未经批准的应用程序和工具的第一步。

#### 5.有效的沟通比维持治安更重要

部门和关键团队成员选择影子 IT 应用程序的动机非常强烈。管理层不应该因为他们使用 it 或通过限制而惩罚他们，而是应该鼓励 IT 部门和相关的非 IT 部门之间的对话。可以让他们意识到使用云服务所涉及的安全威胁、协作问题，并要求他们对这类应用的使用保持透明。影子 IT 应该从影子中出现，并为 IT 和非 IT 部门所接受。

也许，投入一个小团队来研究各部门的云应用程序需求，并致力于最佳的云应用程序，可能是将云与他们的内部系统集成的正确方法。首席信息官(CIO)的角色是充当应用程序和用户之间的中间人。

从长远来看，对云服务的使用施加更严格的规则可能会导致效率降低和员工之间缺乏信任。据报道，一些公司已经对 Evernote、Dropbox 和 iPhones 采取禁令，以阻止云服务的使用，但这种服务可以大幅提高员工的生产力也是事实。因此，这个问题的解决方案不在于有效的监管，而在于更多的内部沟通和对话。

#### 6.IT 部门需要监控出站流量

有防火墙来保护入站流量，但需要监控出站流量，以找出寻求云应用程序的员工。防火墙可以自动识别出站流量，并记录被监控的站点。这可以帮助检测使用此类应用程序的人，并与他们进行对话。

跟踪员工使用的云应用程序还使组织能够评估与他们相关的威胁。如果内部 IT 部门无法全面评估威胁，则有 CipherCloud 等组织，其风险情报实验室已经评估了数百万个云应用程序是否符合 HIPAA、PCI 和 EU Safe Harbor 等法规。

不能排除员工将云应用用于个人用途以及使用外部应用的相关风险。当员工从事此类工作时，可能会严重影响员工的工作效率和公司业绩。

同样，我们发现许多员工使用个人电子邮件 id 来注册云应用程序，并在移动设备和其他设备上上传数据和更新。这很方便，而且[确实提高了生产率](https://www.educba.com/improve-productivity-in-office/)；然而，与此人离开组织并与竞争对手共享此类数据相关的风险。如果没有适当的控制，离开组织的员工仍然可以访问云应用程序的数据，因为登录是使用他们的个人 id 而不是公司 id 创建的。

#### 7.软件升级会导致系统故障

对于云应用程序，升级的责任在于服务提供商。然而，这种升级可能会对组织内部系统的功能或集成产生影响。因此，管理与降级相关的变更是一项艰巨的任务，需要第三方的支持。

如果 IT 部门参与云应用程序的安装和管理，则可以避免此类问题，因为他们可以与服务提供商合作，确保顺利集成和协作。分析师说，它可以防止因软件升级而导致的任何故障。

#### 8.建立一个框架，将工具带入内部

在不禁止外部工具和应用的情况下，如果建立了部署技术的正确框架，就可以提高生产力，这也将促进组织内部的创新。实施混合云是解决这个问题的一种方法，它为各部门的工作组应用程序提供了一个理想的平台。该框架将使员工拥有构建业务所需解决方案的工具。

随着企业的全球化，员工在工作中旅行，在旅行中工作，允许使用可以与台式机、笔记本电脑和移动设备集成的最佳技术是有意义的。首席信息官(CIO)和首席技术官(CTO)需要对云应用的使用进行控制，即使他们没有限制云应用的使用来抑制创新。IT 部门应该营造一个共同创新的环境，而不是像过去那样各自为政。

现在，微软发布的云应用安全可以帮助 IT 团队跟踪员工使用的云应用，这在以前可能没有被发现。它不仅能识别应用，还能提供风险评分、实时风险评估和分析。它还将使 It 管理员能够仅授权选定的云应用程序，设置控制和数据共享策略，并根据公司的要求对其进行定制。

IT 专业人士正在与云应用提供商密切合作，以使他们的产品安全并被企业接受。IBM Cloud Security Enforcer 将帮助服务提供商评估基于云的服务中的风险和威胁。它正与 DropBox 和其他领先的不受监管的供应商密切合作。

### 结论

思科和 NTT 通信公司最近的一项研究显示，影子 IT 的使用在整个行业都在增加。在 NTT 的调查中，约 77%的决策者使用过由第三方服务提供商设计的云应用程序，他们预计这种应用程序的使用将会增加。调查中的许多受访者不知道基于云的数据存储在哪个国家。
