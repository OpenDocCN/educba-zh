## Unix 简介

由贝尔实验室的 Dennis Ritchie、Ken Thompson、Brian Kernighan、Joe Ossanna 和 Douglas Mcllroy 于 1970 年开发的 Unix 操作系统，通过培养类似 POSIX 的互操作性原则和遵循类似单一目的执行、仅在文本上操作的标准接口、核心内核卷绕 Unix 系统从而允许系统和其他进程同时工作，并在很大程度上实现互操作性，以此能力而闻名。

在系统间日益增长的不兼容性中，像 POSIX 和单一 Unix 规范这样的互操作性原则发挥了很大作用。

系统的基本原理如下:

1.  单一目的执行
2.  一个很大程度上标准化的操作文本的界面
3.  围绕核心内核卷绕 Unix 系统，这允许系统和各种其他进程
4.  很大程度上可互操作

### 了解 Unix

*   **Unix 文件:**从一个角度来看，文件是存储在磁盘中的信息或数据的集合。除了 DB 透视图之外，几乎所有数据都存储为它的文件。LS 命令用于显示目录中关联的文件，

例如:drwxr-xr-x 3 root root 4096 2018 年 4 月 4 日 Test.txt

-rw-r–r–1 根根 3028 2018 年 4 月 4 日 Sample.conf

| **字段** | **描述** |
| drwxr-xr-x | 与文件相关的模式和访问详细信息 |
| Three | 文件中的链接 |
| 根 | 文件的所有者 |
| 根 | 文件组名称 |
| Four thousand and ninety-six | 文件的总字节数 |
| 2018 年 4 月 4 日 | 月份和年份的详细信息 |
| 高级配置和电源接口 | 路径名/文件名 |

1.  只允许阅读
2.  只允许写
3.  执行许可
4.  **–:**无权限

文件模式，为了识别 drwxr-xr-x 文件模式，可以将其分为四个关键组，

1.  第一组: d
2.  第二组: rwx
3.  第三组: r-x
4.  第四组 : r-x

*   **Unix 目录:**目录对于 [windows 操作系统](https://www.educba.com/windows-operators/)中的文件夹来说非常熟悉。在大多数情况下，每个目录都是其文件的集合。目录主要分为两种类型，

1.  根目录:它可能是所有其他目录的父目录
2.  **子目录:**任何位于根目录下的目录都称为子目录。用户可以重命名该目录。

### Unix 能做什么？

*   对于不能接受任何停机机会的服务器，可以将其更新为非活动状态，而无需将其拖入重新启动过程。
*   对于真正的瘦客户机，它允许从网络重新启动整个操作系统。
*   允许多个人同时登录多台主机。

### 优势

*   允许使用受保护的内存进行多任务处理，因此这允许并发用户同时运行多个程序，而不会出现任何系统崩溃。
*   大量的程序可以用少量的内存轻松执行。这清楚地显示了 Unix 内存系统所展示的效率。
*   每个试图登录 UNIX 系统的用户都需要使用有效的密码和帐户进行身份验证。
*   大量的命令允许很好地完成特定的任务。
*   非常便携的操作系统，可以跨多种系统使用
*   非常优化的程序开发环境。

### 结论

最后，UNIX 无疑是在市场上长期稳定运行的高性能操作系统之一，满足了各种市场需求和用户。毫无疑问，访问的并发性和安全性是系统的重要用途之一。尽管经历了一个漫长的技术和发展时代，但对这种开源操作系统的需求在市场上仍然是突出和稳定的。
