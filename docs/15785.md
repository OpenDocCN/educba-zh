### 第 1 部分—数据分析师技术面试问题(基础)

第一部分包括基本的面试问题和答案:

#### Q1。说到数据挖掘和数据分析，有什么区别？

**答案:**

| **数据挖掘** | **数据分析** |
| 在预先建立的数据库中识别模式的技术被称为模式识别。 | 它用于以一种有用的方式组织和排序非结构化数据。 |
| 数据挖掘是在干净且记录良好的数据上执行的。 | 因为数据分析需要数据清理，所以没有以文档化的方式提供。 |
| 结果很难解读。 | 结果很容易理解。 |
| 它最常用于机器学习，即利用算法来识别模式。 | 它用于从原始数据中提取信息，原始数据在被分析之前必须首先被清理和组织。 |
| 从海量数据集中提取信息的技术被称为数据挖掘。 | 公司使用数据分析来更深入地研究这些信息，以便了解更多信息。检查、清理、转换和建模数据都是数据分析的一部分。 |
| 数据分析中的操作之一是数据挖掘。 | 对数据进行分析、清理、操作和建模以识别有用信息、提供结论和辅助决策的过程称为数据分析。 |
| Rapid Miner, Oracle Data Mining, IBM SPSS Modeler, Knime, Python, Orange these are the some data mining tools that used to perform the mining operations. | 数据分析工具是收集和分析有关公司、其客户和竞争对手的数据的软件和程序，目的是改进流程和发现洞察力，以便做出数据驱动的决策。 |

#### Q2。什么是数据分析方法？

**答案:**

数据分析是对数据的收集、清理、解释、转换和建模，以便获得业务洞察力并向用户提供报告。过程中涉及的众多工序。信息是从各种来源收集的，并在被清理和处理之前保存下来。在此步骤中，所有缺失值和异常值都将被剔除。下一步是评估已经收集的数据。为了改进一个模型，需要运行多次。然后对模型进行验证，看它是否符合业务需求，并将模型付诸实践，并将结果报告分发给各个利益相关者。

#### Q3。你在分析数据时遇到过哪些困难？

**答案:**

列出你在分析和清理数据时遇到的所有困难。

以下是数据分析项目中遇到的一些最常见的问题:

*   数据质量很差，有几个值缺失或不正确。
*   不切实际的时间表和企业利益相关者的期望。
*   混合/整合来自众多来源的数据非常困难，尤其是在没有标准参数和规范的情况下。
*   工具和数据架构选择不足，无法及时满足分析目标。

#### Q4。数据分析的类型是什么？

**答案:**

一些数据分析如下:

*   单变量分析
*   双变量分析
*   多变量分析

#### Q5。在数据分析中，使用了哪些数据验证程序？

**答案:**

以下是数据验证方法的示例:

*   **表单级验证:**在这种方法中，验证在用户完成表单并保存数据后执行。
*   **字段级验证:**为了避免人为交互导致的错误，当用户输入数据时，在每个字段中执行验证。
*   **搜索标准的验证:**这种形式的验证与用户相关，因为它在一定程度上匹配用户正在寻找的内容。其目的是确保及时返回结果。
*   **数据保存验证:**这种类型的验证在文件或数据库记录保存时进行。当有许多数据输入表单时，通常会这样做。

### 第 2 部分—数据分析师技术面试问题(高级)

现在让我们来看看高级面试问题:

#### Q6。数据分析项目需要哪些步骤？

**答案:**

以下是数据分析项目的基本步骤:

*   认可行业
*   获取信息
*   检查和净化数据
*   验证信息
*   实现数据集并跟踪它们
*   做预测
*   重复

#### Q7。你是怎么准备数据的？

**答案:**

因为数据准备是数据分析如此重要的一部分，面试官可能会好奇你打算如何在处理和分析之前清理和转换原始数据。我们应该讨论它将使用的模型，以及它计算背后的逻辑推理，以回应这样的数据分析师。此外，它还应考虑到讨论在数据分析中以更大的可扩展性和更快的数据利用来帮助职业成就的行动。

#### Q8。数据分析的目的是什么？

**答案:**

数据分析至关重要，因为它有助于企业和组织提高业务绩效，并创建利用数据进行分析以做出更好的业务决策的公司。组织可以更深入地了解市场、消费者、他们的产品或服务等等，这可以帮助他们实现更好的公司增长。

#### Q9。数据分析师面临的最普遍的问题是什么？

**答案:**

以下是数据分析师面临的一些最普遍的问题:

*   典型的拼写错误
*   重复的条目
*   缺少的值
*   非法的价值观
*   不同的值表示
*   确定重叠区域

#### Q10。举一个数据清洗的例子。

**答案:**

数据清理，也称为数据清洗、数据清理或数据争论，是发现并根据需要更改、替换或删除错误、不完整、不准确、不相关或缺失数据的行为。数据科学的这一重要组成部分保证了数据的准确性、一致性和可用性。
