## MongoDB 和 Oracle 的区别

MongoDB 来自 MongoDB Inc .公司，该公司以其 NoSQL 数据库(其中不需要 SQL)而闻名，它在其系统中处理文档，而 Oracle 公司的 Oracle 是一个关系数据库管理系统。MongoDB 和 Oracle 都可以在所有主要使用的操作系统中访问，如 Windows、Linux、Unix 等。Oracle 系统由数据库、表和数据组成，而 MongoDB 由字段中的文档组成。

### MongoDB

*   MongoDB 是由 MongoDB Inc(是一家美国软件公司)**设计开发的。它是在 GNU Affero 通用公共许可证和 Apache 许可证的组合和协调下发布的。MongoDB 于 2009 年 2 月发布，最新的稳定版本发布于 2018 年 6 月。**
*   通常，一个 MongoDB 服务器中有多个数据库。MongoDB 文档不支持 SQL，它支持高级的、丰富的和专门的查询语言。MongoDB 数据库将数据存储在一个称为集合的区域中，而不是存储在表中。这些是相当于 RDBMS 表的粗略数据。
*   MongoDB 是用 C++、C 和 JavaScript 编程语言编写的。MongoDB 可以方便地在以下操作系统中运行:Windows Vista 和更高版本、Linux、OS X 10.7 和更高版本、Solaris 和 FreeBSD。

### 神谕

*   在将 Oracle 数据库引入市场后，它将关系模型扩展到了对象关系模型。因此它使得在关系数据库中存储非常复杂的商业模型成为可能。
*   甲骨文的历史演变非常有趣。Oracle 数据库是由 Larry Ellison、Bob Miner 和由 Larry Ellison 领导的 Ed Oates 三个朋友组成的团队于 1977 年在 SDL 的一家软件开发实验室开发的。Oracle 数据库是广泛使用和可信的关系数据库引擎之一。
*   一个 Oracle 数据库是用汇编语言、C 和 [C++编程语言](https://www.educba.com/c-programming-language-basics/)编写的。Oracle 数据库可以在所有主流平台上运行，包括 Windows、UNIX、Linux 和 Mac OS。Oracle 数据库通常用于运行在线事务处理(OLTP)、数据仓库(DW)应用程序和混合(OLTP & DW)数据库工作负载。

### MongoDB 和 Oracle 的主要区别

两者都是市场上的热门选择；让我们讨论一些主要的区别:

*   MongoDB 是最著名的面向文档的数据库之一，而 Oracle 数据库是一个多模型的数据库管理系统，它大量使用 RDBMS 来构建企业应用程序。
*   在 MongoDB 中，数据以文档和字段的形式存储在一个集合中。但是在 Oracle 数据库中，数据是以传统的 RDBM 方式存储的，即以行和列的形式存储在平板电脑中。
*   Mongo 数据库为用户定义的 Map/Reduce 方法提供了一些 API，而 Oracle 数据库不支持 MapReduce。
*   在 MongoDB 中，分区方法(即在不同节点上存储不同数据的方法)是分片的，而在 Oracle 数据库中是水平分区的。
*   在 Oracle 数据库中，还提供了可选的 Oracle 分区。
*   MongoDB 不支持 SQL，但 Oracle DB 支持 SQL。
*   MongoDB 是一个免费的开源软件，而 Oracle 数据库是商业化的(市场上有一个受限的免费版本)。
*   MongoDB 的实现语言是用 C++、C 和 JavaScript 编程语言编写的。Oracle 数据库也是如此，包括汇编语言、C 和 C++编程语言。
*   MongoDB 的服务器端脚本可以通过使用 JavaScript 编程语言来实现。通过使用 PL/SQL 编程语言，Oracle 数据库也可以实现同样的功能。
*   在 MongoDB 中，它只使用一个辅助数据库模型，即键值存储，而在 Oracle DB 中，它使用四个辅助数据库模型，包括键值存储，其余三个是文档存储、图形 DBMS 信息和 RDF 存储信息。

### 结论

在竞争中，与 Oracle DB 相比，Mango DB 处于第一位，因为 MongoDB 在迁移过程中更容易处理，因为它本质上是无模式的。

所有规模的组织都可以采用 MongoDB，因为它使开发人员能够更快地开发应用程序，处理高度多样化的数据类型，并且更有效地管理这些应用程序。Oracle 数据库并不适合所有类型的组织。非常适合大规模的企业级应用。我希望现在您已经对 MongoDB 和 Oracle 有了更公平的认识。请继续关注我们的博客，获取更多类似的文章。
