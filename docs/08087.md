## GitHub OpenCV 简介

GitHub OpenCV 的关键见解是基于计算机的视觉、机器学习算法和图像级处理。甚至像实时操作这样的过程也可以通过 OpenCV 系统来实现。作为一个基于开源的系统，该系统的最初版本是 1.0。商业和学术水平的项目都可以通过 OpenCV 的方式获得，因为它是在 BSD 下授权的。

在与 OpenCV 相关联的 GitHub 系统中，可以非常高效地实现多核处理。C 和 C++是这些应用中使用的主要语言，因为这些语言可以灵活地实现多级核心处理。使用这些基于 GitHub 的 OpenCV 系统的一个关键优势是可以灵活地向系统中添加新的项目和代码。

### 它是如何工作的？

其中一些关键过程可以借助基于 GitHub 的 OpenCV 系统来完成。这些是在需要时激活系统的各种相关值。非常重要的是，确保为一个问题创建一个 PR。我们需要严格确保针对一个问题提出的 PR 不超过一个。这可以大大避免提出重复的 PR 请求，这将为我们节省大量的处理时间。它们如下:边缘检测、特征匹配和提取特征、单应过程和几何级的变换、级联分类器和允许检测面部、对象分类、对象跟踪。

下面给出了如何将 OpenCV 项目的代码贡献给 GitHub 的步骤:

*   确保为处理的每个问题至少提出一个与拉动相关的请求。
*   需要正确评估位于底部的分支；我们可能需要确保选择了正确且必要的基本分支。
*   文档和测试有望从这里开始被包含进来。之后，所有需要的文档和测试都可以添加到这里。
*   在进入提交流程之前，确保 oops 提交被清除并被进一步处理。

### 拉取请求

*   这个过程的关键是创建拉取请求；这需要紧密地关联、评估和决定，其中一些关键操作与拉请求相关联。
*   首先，在考虑执行推送请求之前，我们需要确保这个请求的构建状态。尤其是在本地系统中。所以在用户的本地系统中，我们需要确保拉请求被完美地执行，并且 PR 很好地通过拉。接下来需要给出有意义的标题；必须添加足够的信息，提交的详细信息非常接近预期，然后，如果这是为特定 PR 打开的，为什么需要提交和链接相关的问题页面。
*   非常重要的是，确保为一个问题创建一个 PR。我们需要严格确保针对一个问题提出的 PR 不超过一个。这可以大大避免提出重复的 PR 请求，这将为我们节省大量的处理时间。
*   哎呀，提交在拉请求中不受欢迎。确保遵循编码风格指南，更重要的是，我们需要确保选择准确的目标分支，并将其作为拉请求的首选。此外，一定要为每一个新的功能添加地址，或者甚至为与测试数据相关的已修复的错误添加地址。此外，图像或视频将在更大程度上影响拉取请求。最后，如果您支持优化，或者补丁应该对基本功能的整体性能产生影响，请确保进行整体性能检查。了解如何编写 PerfTests 网页以实施出色的检查，以及如何使用 Perf Tests 网页正确运行整体性能评估。
