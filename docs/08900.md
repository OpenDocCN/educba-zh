## Java 中的非访问修饰符简介

非访问修饰符是 Java 7 中引入的关键字，用来通知 JVM 类的行为、方法或变量等。这有助于引入额外的功能，例如用于指示变量不能初始化两次的 final 关键字。总共引入了 7 个非访问修饰符。

1.  静态
2.  最后的
3.  摘要
4.  同步的
5.  短暂的
6.  strictfp
7.  当地的

#### 2.抽象非访问修饰符

![Abstract Non-Access Modifier](../Images/46f57b84e19914652f3a8a503b315315.png "Abstract Non-Access Modifier")

<noscript><img class="alignnone wp-image-318702 size-full" title="Abstract Non-Access Modifier" src="../Images/46f57b84e19914652f3a8a503b315315.png" alt="Abstract Non-Access Modifier" width="452" height="95" srcset="https://cdn.educba.com/academy/wp-content/uploads/2020/03/Abstract-Non-Access-Modifier.png 452w, https://cdn.educba.com/academy/wp-content/uploads/2020/03/Abstract-Non-Access-Modifier-300x63.png 300w" sizes="(max-width: 452px) 100vw, 452px" data-original-src="https://cdn.educba.com/academy/wp-content/uploads/2020/03/Abstract-Non-Access-Modifier.png"/></noscript>

*   **抽象类**:一个类被声明为抽象，表示这个类不能被实例化，也就是说这个类不能形成对象，但是可以被继承。尽管如此，这个类有一个构造函数，将在它的子类的构造函数中被调用。它可以包含抽象方法和最终方法，其中抽象方法将在子类中被覆盖。

**代码:**

```
public abstract class MyActivity{
public MyActivity(){
}
public final String myFun1(){
}
}
```

*   **抽象方法**:抽象方法是没有任何定义的方法。它只包含方法的签名，并且意味着这些需要在子类中被覆盖。

**例子:**公共抽象 void fun 1()；

**代码:**

```
abstract class Electronics
{
abstract void display();
abstract void display(String msg);
}
class Computers extends Electronics
{
@Override
void display() {
System.out.println("Abstract method is called");
}
@Override
void display(String txt) {
System.out.println(txt);
}
}
public class AbstractDemo {
public static void main(String[] args) {
Computers obj=new Computers();
obj.display();
obj.display("Method with arguments");
}
}
```

#### 3.同步非访问修饰符

![Synchronized Non-Access Modifier](../Images/6302b63b219c4f10819bca2d5d4369be.png "Synchronized Non-Access Modifier")

<noscript><img class="alignnone wp-image-318707 size-full" title="Synchronized Non-Access Modifier" src="../Images/6302b63b219c4f10819bca2d5d4369be.png" alt="Synchronized Non-Access Modifier" width="328" height="105" srcset="https://cdn.educba.com/academy/wp-content/uploads/2020/03/Synchronized-Non-Access-Modifier-1.png 328w, https://cdn.educba.com/academy/wp-content/uploads/2020/03/Synchronized-Non-Access-Modifier-1-300x96.png 300w" sizes="(max-width: 328px) 100vw, 328px" data-original-src="https://cdn.educba.com/academy/wp-content/uploads/2020/03/Synchronized-Non-Access-Modifier-1.png"/></noscript>

该关键字有助于防止多个线程同时访问一个方法，从而同步程序流，并使用多线程功能产生所需的结果。

**代码:**

```
class Person1
{
public synchronized void sendFun(String txt)
{
System.out.println("Sending message\t" + txt );
try
{
Thread.sleep(1000);
}
catch (Exception e)
{
System.out.println("Thread interrupted.");
}
System.out.println("\n" + txt + "Sent");
}
}
class DemoThread extends Thread
{
private String txt;
Person1  person;
DemoThread(String m,  Person1 obj)
{
txt = m;
person = obj;
}
public void run()
{
synchronized(person)
{
person.sendFun(txt);
}
}
}
public class HelloWorld
{
public static void main(String args[])
{
Person1 snd = new Person1();
DemoThread S1 =
new DemoThread( " Hi " , snd );
DemoThread S2 =
new DemoThread( " Bye " , snd );
S1.start();
S2.start();
// wait for threads to end
try
{
S1.join();
S2.join();
}
catch(Exception e)
{
System.out.println("Interrupted");
}
}
}
```

#### 4.静态非访问修饰符

![Static Non-Access Modifier](../Images/79a9f74ca4f6917f619ebad86673a8a6.png)

<noscript><img class="alignnone size-full wp-image-318711" src="../Images/79a9f74ca4f6917f619ebad86673a8a6.png" alt="Static Non-Access Modifier" width="448" height="340" srcset="https://cdn.educba.com/academy/wp-content/uploads/2020/03/Static-Non-Access-Modifier.png 448w, https://cdn.educba.com/academy/wp-content/uploads/2020/03/Static-Non-Access-Modifier-300x228.png 300w" sizes="(max-width: 448px) 100vw, 448px" data-original-src="https://cdn.educba.com/academy/wp-content/uploads/2020/03/Static-Non-Access-Modifier.png"/></noscript>

这个变量用于内存管理，也是加载类时引用的第一个变量。这些成员被视为一个类；因此，不能使用对象调用它们；相反，类名被用来引用它们。

*   **静态变量:**如果一个变量被声明为静态的，那么只有该变量的一个副本被创建并在所有对象之间共享。因此，一个对象对变量所做的任何更改都会反映在其他对象中。因此，在类级别保存值的变量被声明为静态的。
*   **静态类** : Static 关键字只能用于嵌套类。
*   **静态方法**:由于静态方法被类名引用，因此只能访问静态成员变量和其他静态方法。此外，不能使用 this 或 super 指针引用这些方法。main 方法是最常见的[静态方法的例子，当它的类被加载时，它总是被加载](https://www.educba.com/static-method-in-python/)。
*   静态块:这是一个在类被加载时用来执行某些操作的块。因为它是静态的，所以只能使用该类的静态成员。

**代码:**

```
public class Demo
{
// static variable
static int x = 10;
static int y;
//static class
public static class DemoInnerClass{
static int z=10;
}
// static block
static {
System.out.println("Static block initialized.");
y = x + 4;
}
//static method
public static void main(String[] args)
{
System.out.println("from main");
System.out.println("Value of x : "+x);
System.out.println("Value of y : "+y);
System.out.println("Value of z : "+DemoInnerClass.z);
}
}
```

<noscript><img class="alignnone size-full wp-image-318717" src="../Images/f3b5b8baa294d5978df350527321b811.png" alt="Strictfp Non-Access Modifier" width="457" height="95" srcset="https://cdn.educba.com/academy/wp-content/uploads/2020/03/Strictfp-Non-Access-Modifier.png 457w, https://cdn.educba.com/academy/wp-content/uploads/2020/03/Strictfp-Non-Access-Modifier-300x62.png 300w" sizes="(max-width: 457px) 100vw, 457px" data-original-src="https://cdn.educba.com/academy/wp-content/uploads/2020/03/Strictfp-Non-Access-Modifier.png"/></noscript>

*   **Strictfp 类/方法:**该关键字用于确保浮点数运算的结果在每个平台上都产生相同的结果。该关键字不能用于抽象方法、变量或构造函数，因为它们不需要包含操作。

**代码:**

```
public class HelloWorld
{
public strictfp double calSum()
{
double n1 = 10e+07;
double n2 = 9e+08;
return (n1+n2);
}
public static strictfp void main(String[] args)
{
HelloWorld t = new HelloWorld ();
System.out.println("Result is -" + t.calSum());
}
}
```

#### 7.瞬时非访问修饰符

当通过网络将数据从一端传输到另一端时，为了成功接收数据，必须对数据进行序列化，这意味着在发送之前先将其转换为字节流，然后在接收端再将其转换回来。为了告诉 JVM 哪些成员不需要进行序列化，而不是在传输过程中丢失，需要使用一个瞬态修饰符。

**语法**:

```
private transient member1;
```

**代码:**

```
import java.io.*;
class Demo implements Serializable
{
int x = 10;
transient int y = 30;
transient static int z = 40;
transient final int d = 50;
public static void main(String[] args) throws Exception
{
Demo input = new Demo();
FileOutputStream tos = new FileOutputStream("abc.txt");
ObjectOutputStream tin = new ObjectOutputStream(tos);
tin.writeObject(input);
FileInputStream fis = new FileInputStream("abc.txt");  ObjectInputStream ois = new ObjectInputStream(fis);
Demo output = (Demo)ois.readObject();
System.out.println("x = " + output.x);
System.out.println("y = " + output.y);
System.out.println("z = " + output.z);
System.out.println("d = " + output.d);
}
}
```
