## JSON 和 SOAP 的区别

Web 服务是使用 JSON 或 SOAP 创建的。SOAP，简单对象访问协议，是发送由 W3C 联盟管理的消息的标准协议。SOAP 有许多规则和几个安全特性，这增加了它的复杂性。它具有内置的合规性和授权。由于它需要更多的带宽，应用程序可能会很慢。JSON 是为克服 SOAP 的问题而开发的一种架构风格。开发人员可以用自己的方式实现通信，JSON 的标准不会阻止他们这样做。JSON 是轻量级的，也比 SOAP 快。

### JSON

JSON 是 JavaScript 对象符号的首字母缩写。它本质上是轻量级的，用于数据交换。JSON 最重要的优点是人类很容易理解，即读和写。任何机器都很容易解析和生成。这是基于 1999 年 12 月的标准 ECMA-262 第三版，它是 JavaScript 编程语言的子集。

#### 构建 JSON 的结构:

*   JSON 中维护的格式是键/值对的集合。在不同的语言中，它被实现为对象、记录、结构、字典、哈希表、键列表或关联数组。
*   它可以是有序的值列表。在不同的语言中，它被实现为数组、向量、列表或序列。

现在来看浏览器和服务器之间的数据交换，我们都知道数据只能是文本。

如前所述，JSON 是文本，它可以从任何 JavaScript 对象转换成 JSON，然后 JSON 可以被发送到服务器。类似地，我们也可以将从服务器收到的任何 JSON 转换成 JavaScript 对象。

这是一个非常简单的方法，通过它我们可以发送和接收消息，并在服务器和浏览器之间建立通信，而不需要复杂的解析和翻译。

我还想提醒您的是，JavaScript 还提供了内置函数，可以将 JSON 格式的字符串转换为本地 JavaScript 对象:

#### JSON.parse()

因此，如果我们从服务器接收 JSON 格式的数据，可以很容易地将其转换成 JavaScript 对象来使用。

### 肥皂

以下是 SOAP 协议的几个特征。

*   SOAP 是一种通信协议，它被设计为通过互联网进行通信。
*   通过扩展 HTTP，SOAP 可以用于 XML 消息传递。
*   SOAP 提供了通过。网络服务。
*   SOAP 能够交换完整的文档，或者它可以调用远程过程。
*   SOAP 甚至可以用于消息广播。
*   SOAP 是独立于语言的，它可以在任何平台上运行，即平台无关。
*   使用 SOAP，可以定义发送什么信息以及如何发送。
*   它可以将客户端的应用程序连接到远程服务，甚至调用远程方法。

如上所述，SOAP 消息是独立于平台和语言的，因为它们完全是以 XML 格式编写的。

SOAP 的基本结构是用 XML 编写的，因此它包含以下元素和其他元素:-

*   **信封**—这是一个强制元素。它用来表示信息的开始和结束。
*   **标题**—它是一个可选元素。它可以包含任意数量的可选消息属性，用于在中间点或端点进行处理。
*   **Body**—它也是一个强制元素。它基本上包含了需要发送的 XML 数据。
*   **故障**–它是一个可选元素。该元素的作用是在故障发生时抛出错误消息，并处理消息。

### JSON 和 SOAP 的主要区别

两者都是市场上的热门选择；让我们讨论一些主要的区别:

*   SOAP 是一种协议，意味着一组规则。JSON 是一个对象。
*   SOAP 可以使用 JSON 进行通信，但是反过来是完全不可能的。
*   SOAP 使用 XML 格式，而 JSON 使用键值对。
*   可以用 SOAP 来声明错误消息，但是用 JSON 就不可能了。
*   JSON 与 SOAP 的比较。

### 结论

所以经过讨论，我们可以得出这样的结论:SOAP 和 JSON 是完全不同的东西，没有可比性。在我们的讨论中，我们遇到了 JSON 和 SOAP 的一些基本事实，我们发现一个是对象，另一个是协议。JSON 使用键值对，而 SOAP 使用 XML。SOAP 可以使用 JSON，但反过来就不行了。我也试着用例子来解释，因此上面的观点证明了 JSON 和 SOAP 是两回事，没有可比性。
