## Web 应用程序安全性介绍

以下文章概述了 Web 应用程序的安全性。我们现在生活在一个网络世界。每天，在各个领域，如银行、学校、商业、世界顶级机构和研究中心，都有数不清的交易在网上进行。因此，正在处理的数据非常安全并且通信可靠是极其重要的。因此，确保网络安全变得非常重要。

### 什么是 Web 应用安全？

Web 应用程序安全是信息安全的一个分支，它处理 web 应用程序、web 服务和网站的安全。它是一种具体应用于 web 或 internet 层面的应用安全。

<small>网页开发、编程语言、软件测试&其他</small>

Web 安全非常重要，因为 web 应用程序会因为糟糕的编码或对应用程序输入和输出的不正确处理而受到攻击。常见的网络安全攻击有跨站点脚本(XSS)和 [SQL 注入](https://www.educba.com/what-is-SQL-Injection/)。

除了 XSS、SQL 注入，其他类型的 web 安全攻击还有任意代码执行、路径泄露、内存损坏、远程文件包含、[缓冲区溢出](https://www.educba.com/what-is-buffer-overflow/)、本地文件包含等。Web 安全完全基于人员和流程。因此，在网站上线之前，开发者[使用适当的编码](https://www.educba.com/uses-of-coding/)标准并对任何此类网络安全威胁进行健全性检查是极其重要的。

事实上，安全检查必须在开发的早期应用，并在软件开发生命周期的每个阶段持续应用。开发人员需要在网络安全和安全编码实践方面接受良好的培训。一次性测试应用程序肯定是无效的。web 安全攻击的持续回归需要在每个阶段实施。

### 标准化网络安全

*   OWASP(开放 web 应用程序安全项目)是 Web 应用程序安全的标准团体。
*   它提供了 web 应用程序安全领域的完整文档、工具、技术和方法。
*   OWASP 是关于 web 应用程序安全性最佳实践的公正信息源之一。

### OWASP 最大的网络安全风险

以下是 OWASP 上报告的主要网络安全风险:

#### 1.SQL 注入

这种类型的注入攻击可能会执行恶意和不正确的 SQL 查询，从而控制 web 服务器数据库。攻击者可以使用 SQL 语句绕过应用程序安全措施。例如，他们可以验证或授权网页或网站，并绕过 SQL 语句获取 SQL 数据库的内容。这种攻击可能发生在使用 SQL、MYSQL、Oracle 等的网站上。，比如数据库。根据 OWASP 2017 文档，这是最普遍和最危险的安全攻击。

#### 2.跨站点脚本(XSS)

这使得攻击者能够将客户端脚本注入到 web 应用程序和其他用户查看的网页中。此外，跨站点脚本漏洞可用于绕过同源策略等策略。截至 2007 年，XSS 占所有网络安全攻击的 84%。根据数据的敏感性，XSS 可能是对网站的小攻击，也可能是大威胁。

攻击者将恶意数据放入发送到客户端浏览器的内容中。当数据被传送到客户端时，看起来组合的数据来自可信的服务器本身，并且具有客户端的所有权限集。因此，攻击者现在可以获得对敏感页面内容、会话 cookies 和各种其他信息的更高访问权限和特权。

#### 3.身份验证和会话管理中断

这种攻击允许捕获或绕过网页或应用程序上的身份验证。这是网站开发人员遵循的一个较弱的标准，会导致一些问题，例如:

*   可预测的登录凭据。
*   存储时未正确保护用户登录凭据。
*   会话 id 在 URL 中公开。
*   密码和会话 id 不通过加密的 URL 发送。
*   会话值不会在特定时间后超时。

为了防止这些攻击，开发人员应该小心维护适当的标准，例如保护密码并在密码通过时对其进行适当的哈希处理，不暴露会话 id，在特定时间后使会话超时，以及在成功登录尝试后重新创建会话 id。

#### 4.修复损坏的身份验证

*   密码长度应保持在至少 8 个字符。
*   密码应该很复杂，使用户无法预测。这应该利用适当的密码设置规则，如字母数字、特殊字符和数字大/小写组合。
*   身份验证失败不应表明身份验证数据的哪一部分不正确。相反，错误响应在某种程度上应该是通用的。例如无效凭证，而不是显示用户名或密码，这是完全不正确的。

### 安全错误配置

*   这是使网站易受攻击的不良做法之一，例如，应用服务器配置向用户返回[完整堆栈跟踪](https://www.educba.com/what-is-full-stack/)，使攻击者知道缺陷在哪里，并据此攻击网站。
*   重要是，要实施更强大的应用程序体系结构，并定期运行安全扫描来防止这种情况。

### 结论

非常重要的是，每个网站都要遵循适当的标准，保持适当的编码技术，拥有强大的应用程序架构，定期运行扫描而不会失败，并在更大程度上避免网络安全攻击。
