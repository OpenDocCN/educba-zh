## 吉拉问题类型介绍

我们知道，吉拉是用来管理整个开发生命周期的项目。吉拉还提供了管理问题的功能。换句话说，我们可以说吉拉工具将工作划分为诸如任务、史诗、萌芽、请求或任何不同类型的工作等主题。在吉拉，我们有一些默认发行类型。在吉拉，我们可以按照我们的要求记录问题中的子任务，它提供了在项目开发阶段跟踪所有问题的工具，这意味着从开始到结束。

### 什么是问题类型？

问题类型以独一无二的方式识别各种工作，并帮助您区分、安排和报告您在整个吉拉站点的合作情况。此外，他们可以帮助你的团队将更多的结构整合到你的功能循环中。各种问题类型可帮助您搜索和整理小组的工作，确定最终工作的进展，甚至评估您的小组解决问题的能力或完成更大任务的速度。每个吉拉项目都附带默认的问题类型，以满足您的活动和小组的要求。此外，您可以修改您的问题类型，以匹配项目的一些技术和您需要的管理人员。

<small>网页开发、编程语言、软件测试&其他</small>

吉拉经理人公开发行股票的能力偶尔会出现，并导致不良后果。当问题类型被滥用时，这将导致标准吉拉有用性不起作用。吉拉现在配备了许多默认问题类型，通常是编程改进的坚实匹配。如果在你的协会中这不是真的，它应该让你认为你的工作方法会使新的定制问题类型合法化。

吉拉软件附带五种标准问题类型，以便在类似的吉拉项目中，问题可以有不同的字段、工作流程或两者兼有。例如，一个 Bug 问题类型可能有明确的畸形区域，如“重现步骤”和“预期结果”尽管如此，这两个字段在任务问题类型的屏幕上没有位置。

### 吉拉发行类型的分类

下面给出了提到的分类:

#### 1.子任务

这是已创建问题的子任务；在单个问题中，我们可以根据我们的需求创建多个子任务，并将它们标记为已解决。子任务问题，可以帮助你的小组将一个共同的问题分解成更小的块。假设你的小组有一项需要许多个人来处理的任务，或者认为你的小组低估了他们工作的程度或复杂性，这可能是有用的。子任务可以独立于它们相关的标准问题进行描述和评估。他们可以帮助你的小组从现在开始更好地理解和评价比较工作；更多信息，我们可以看下面的截图。

### 吉拉发行类型计划

问题方案用于确定在指定项目下可以使用哪种特定类型的问题。此外，它还负责指示在发布问题时将在 JIRA 的 UI 中引入问题类型的请求。当企业包括在 JIRA 中时，发行类型共谋产生。

通常，有以下两种问题类型:

#### 1.默认问题类型方案

这是第一个问题方案，每当我们创建新问题时，它都会自动添加到该方案中。默认的发行方案，我们可以按照我们的要求编辑。默认发行类型是新发行的默认方案；默认的问题类型方案；我们可以看到下面的截图了解更多信息。

### 吉拉软件发行类型

以下是吉拉软件问题的类型:

*   **Bug:** 这只不过是现有功能中的缺陷，我们称之为 Bug。
*   史诗:史诗就是大用户故事，我们把它分成小部分，称之为故事。完成整部史诗的标准时间是一个月。换句话说，我们可以说史诗可以转化为用户故事。
*   这只不过是一件作品。
*   任务:这是分配给团队成员的工作。
*   故事:这只是我们需要在指定项目中完成的任务列表。
