### 通过以下方式实施流程组

当我们使用' aggregate '时，aggregate 函数应用于整个表(整个表)。尽管这个过程通过对整个表应用函数使工作变得简单，但通常的方法是仅对行组应用“聚合”。

使用分组依据计数的活动用户:

**代码:**

**SQL:**

`SELECT
is_active,
COUNT(id) AS total
FROM
auth_user
GROUP BY
is_active`

**Python:**

`queryset = (User.objects
.values(
'is_active',
)
.annotate(
total=Count('id'),
)
.order_by()
)`

**输出:**

如果我们在注释之前没有调用值，那么将不会产生聚合结果。因此，在按功能分组时，顺序是需要记住的最重要的事情之一。

**在查询集中使用过滤器:**

当一个特定或精确的列或行需要计数时，我们应该使用“过滤器”。在过滤查询中应用聚合的过程中，我们可以在查询集中的任何地方使用“filter”。

**代码:**

**SQL:**

`SELECT
is_active,
COUNT(id) AS total
FROM
auth_user
WHERE
is_staff = True
GROUP BY
is_active`

**Python:**

`queryset = (User.objects
.values(
'is_active',
)
.filter(
is_staff=True,
)
.annotate(
total=Count('id'),
)
)`

类似于 filter，在查询中的任意位置使用“order_by”对查询集进行排序。为了方便起见，我们可以同时使用“分组依据”和“聚合”字段。

**代码:**

**SQL:**

`SELECT
is_active,
COUNT(id) AS total
FROM
auth_user
WHERE
is_staff = True
GROUP BY
is_active
ORDER BY
is_active,
total`

**Python:**

`queryset = (User.objects
.values(
'is_active',
)
.filter(
is_staff=True,
)
.annotate(
total=Count('id'),
)
.order_by(
'is_staff',
'total',
)
)`

### 分组依据中的条件聚合

要聚合组的一部分，我们应该使用条件聚合。

**举例:**

按注册代码的年份统计正式员工和非正式员工的数量。

**代码:**

`SELECT
EXTRACT(‘year’ FROM date_jointed),
COUNT(id) FILTER (
WHERE is_employee = True
) AS employee_users,
COUNT(id) FILTER (
WHERE is_employee = False
) AS non_employee_users
FROM
authentic_user
GROUP BY
EXTRACT(‘year’ FROM date_joined)
From Django.db.models import F, Q
(User.objects
.values(‘date_joined__year’)
.annotate(
employee_users=(
Count(‘id’, filter=Q(is_employee=True))
),
non_employee_users=(
Count(‘id’, filter=Q(is_employee=False))
),
))`

### Having 子句

在 SQL WHERE 中，关键字和聚合函数不应一起使用。相反，HAVING 子句被添加到 SQL 中。

**语法:**

`SELECT column_name(s)
FROM table_name
WHERE condition
GROUP BY column_name(s)
HAVING condition
ORDER BY column_name(s);`

### Having GROUP BY 的用法

为了从聚合函数的结果中过滤出所需的数据，使用了 HAVING 子句。

**样本案例:**

**代码:**

`SELECT Employees.LastName, COUNT(Orders.OrderID) AS NumberOfOrders
FROM (Orders
INNER JOIN Employees ON Orders.EmployeeID = Employees.EmployeeID)
GROUP BY LastName
HAVING COUNT(Orders.OrderID) > 10;`

**分组依据–不同:**

在计数函数中，有时希望只对不同的事件进行计数。

**样本:**

计算每个用户活动状态下不同姓氏的数量。

**Note:** Use of distinct **=** True in the call is to Count.

**代码:**

`SELECT
is_active,
COUNT(id) AS total,
COUNT(DISTINCT last_name) AS unique_names
FROM
auth_user
GROUP BY
is_active
(User.objects
.values(‘is_active’)
.annotate(
total=Count(‘id’),
unique_names=Count(‘last_name’ distinct=True),
))`
