## GitHub 黑客工具简介

GitHub 向道德黑客提供顶级的黑客工具，通过模拟恶意攻击和评估系统中的安全漏洞来测试任何 Web 应用程序或系统或数据库的安全特性。它提供了一个机会，可以在系统被专业黑客入侵之前纠正设计或代码中的缺陷。

IT 安全专业人员使用这些工具来评估组织的安全准备情况，并事先获得他们的批准和行动计划。在其库中，GitHub 为用户提供了大量的黑客工具，以便在系统的实施阶段以及实施后的持续阶段对系统进行安全测试。

### GitHub 黑客工具概述

GitHub 库中有几个黑客工具，让我们在这里讨论一些工具的细节

#### 1.高斯

Gau 的全面扩展就是获取所有的网址。Gau 是 GitHub 平台中可用的开源工具，该工具对于获取景观中使用的 URL 非常有用。这些信息将有助于通过跟踪网站中访问的所有 HTTP 和 HTTPS 站点来调查应用程序的子域。

在所访问的站点中观察到的任何异常都将用于堵塞应用程序中的漏洞，以便将来在应用程序的执行过程中不会有任何安全威胁。这个工具的代码是用 go 语言开发的，安装在 Kali Linux 操作系统中。用户应该在他们的系统中安装 go 语言环境和 Kali Linux 操作系统，这样该工具才能工作。Gau 拥有良好的用户界面。

#### 2.达尔福克斯

该工具是一个强大的跨站点脚本(XSS)扫描器，黑客将这些脚本注入客户端代码中，以 a .从 web 应用程序中窃取重要信息，b .向服务器发送恶意请求。这个 XSS 扫描器基于 DOM 解析器概念工作，它可以处理所有类型的 XSS 攻击。, 1.持久(已存储)，2。非持久性(反射)，3。文档对象模型(DOM)

除了处理 XSS 攻击，这个工具还能够处理 SQL 注入，服务器端模板注入。存在许多测试和分析功能，其高性能水平和友好的操作是该工具的独特卖点(USP)。这个工具还有助于持续集成和持续交付。

#### 3.dnsx

该工具提供了一个强大的域名系统工具包，可以在 DNS 环境中执行多项调查。用户可以发出任何数量的 DNS 检查，包括通配符过滤，以验证系统的健康状况并发现任何异常。除了 DNS 检查和解析之外，它还提供了控制和打印提取信息的功能。

它的一些特点是

它有助于域名系统跟踪
它还支持探测 DNS 状态代码
操作简单且易于处理
它使用最少的请求来管理通配符查询。
如果搜索的子域数量超过某个阈值，它会自动切换到该系统中的通配符，从而大大节省操作时间

#### 4.Ffuf

Ffuf 代表“起毛比你傻子还快”。它是一个开源的 Web fuzzing 工具，使用一个自动化的过程，通过向应用程序随机注入多个数据组合来发现软件黑客，直到其中一个组合暴露出程序中存在的漏洞。通过这样做，它消除了许多可能为黑客打开应用程序的漏洞。

这个工具在职业黑客中非常受欢迎，它可以在应用程序被专业黑客利用之前发现程序中令人惊讶的错误。除了发现漏洞之外，这个工具还被用来挖掘黑客放置的隐藏的子域、文件和文件夹。

这个工具的一个优点是，它是市场上最快的网络模糊工具，有几个有用的内置选项，并有办法根据需要过滤掉响应。这个工具是用 Go 语言开发的，它的模块化结构使它能够通过合理的努力与现有的体系结构集成。如果没有正确使用线程和过滤器，这个工具可能会产生误导性的结果。

#### 5.核心

Nuclei 能够使用模板向多个主机发送请求，以扫描系统中是否存在恶意内容。快速扫描和零假阳性结果是该工具的独特功能。它使用数据序列化标记语言 YAML，更侧重于数据而不是文件。

YAML 为 Nuclei 进行任何种类的安全检查提供了极大的灵活性。它能够扫描各种系统组件，如 HTTP，DNS，文件，TCP 等。该工具使用用户提供的模板扫描恶意内容。GitHub 的存储库中有数百个这样的模板，主要由 IT 安全专业人员提供。

#### 6.子 finder

这个工具专门扫描网站寻找有效的子域名并列出它们。它在被动模式下使用几个在线资源来发现子域。该工具基于模块化架构，针对最佳速度进行了微调，非常符合用户的期望。效率、速度和被动的操作方式是它的优点。

#### 7.最终报告

这是一个开源的智能工具，用于调查网站的恶意内容。该工具用 Python3 编写，有助于从目标网站或使用开源代码的系统收集数据。它在短时间内从多个网站或社交媒体获取数据，并以用户想要的方式呈现给他们。

爬行、头分析、SSL 证书检查和个人身份信息是该工具的主要活动。

### 其他 GitHub 黑客工具

Webscreenshot.py、Turbolist3r、Nmap Automator、DXAS、BoomER、Aquatone、Powershell-RAT、DumpsterFire、CMSeek、Sniffair 等。，是 GitHub 资源库中可用的其他工具。

### 结论

GitHub 的知识库中有大量这样的道德黑客工具。用户可以选择适合他们需求的工具，并在上线之前使用它们来检查安全漏洞并进行纠正。实施后，他们可以定期扫描他们的系统中是否存在任何不满意的内容，并持续纠正它们。
