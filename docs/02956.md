## AWS vs Azure vs Google Cloud 的区别

AWS 被称为亚马逊网络服务。它是一个云计算平台。AWS 平台早在 2002 年推出，但在 2006 年重新推出。亚马逊提供不同的服务，即基础设施即服务(IaaS)、平台即服务(PaaS)和软件即服务(SaaS)。这是一个正在出售的云。AWS 在计算机能力等基础设施方面提供不同的产品；在这种情况下，AWS 提供由他们配置的虚拟机以及定制的机器。

客户还可以选择尺寸、功率、内存等。例如，它还提供临时存储，并提供像公共云和私有云这样的网络。在虚拟私有云中，用户可以隔离云中的网络。可以创建子网、专用 IP 地址和网络网关、路由表。Azure 是微软的产品，主要指微软 Azure。也是云计算服务平台。它由微软创建，最初于 2010 年发布。它支持 Linux 和 Microsoft Windows 操作系统。它用于构建、部署、测试和管理应用程序。它还提供不同的服务 IaaS、PaaS 和 SaaS。

### AWS、Azure 和谷歌云的主要区别

AWS vs Azure vs Google Cloud 都是市场上的热门选择；让我们讨论一下 AWS、Azure 和 Google Cloud 之间的一些主要区别

1.  AWS 提供虚拟服务器和虚拟机的实例。Azure 提供虚拟硬盘，Google Cloud 提供虚拟机实例。
2.  AWS 实例可以通过按需、预订和现货购买以下任何型号。azure 实例可以从按需和短期承诺的任何模型中购买。对于谷歌云来说，它可以按需提供和持续使用。
3.  AWS 按小时向用户收费。它使用现收现付模式。Azure 按每分钟向客户收费。谷歌云按分钟向客户收费。
4.  在 AWS 中，ECS 被用作码头管理。Azure 也在使用容器服务。谷歌容器引擎正被用于谷歌云中的 docker 管理。
5.  在 AWS 中，冰川被用于存档存储。在 Azure、归档存储和 google cloud 中，cold line 用于归档存储。
6.  在 AWS 中，亚马逊云搜索正被用于搜索服务。在 Azure 中，正在使用 Azure 搜索。对于谷歌云来说，它并不存在。
7.  在 AWS 中，亚马逊 Kinesis 被用于分析。对于 Azure，正在使用 Azure 流分析。对于谷歌云平台，云数据流和云数据准备正被用于分析。
8.  在 AWS 中，AWS ops work 和 config 用于自动化。对于 Azure，正在使用 Azure automation。对于谷歌云，计算引擎管理与木偶，厨师等。
9.  在 AWS 中，AWS 云 HSM 用于合规性。Azure 信任中心正被用于 Azure 和谷歌云平台的合规性安全。

在 AWS 中，AWS 密钥管理服务用于安全凭证。对于 Azure， [Azure key vault](https://www.educba.com/what-is-azure/) 和 google cloud 的 google cloud platform security 正被用于安全凭证。
