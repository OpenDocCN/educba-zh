## 什么是肥皂？

以下文章概述了什么是 SOAP？它为基于 XML 的数据传输定义了一套标准的规则规范。它是一个 web 服务，用于在客户端和服务器之间使用通用语言 XML 进行通信。它允许我们编写自己的应用程序编程接口(API)。它使得将我们的数据从一个应用程序传递到另一个应用程序成为可能。API 接收从客户端到服务器的请求；它通过 HTTP、SMTP 等互联网协议从服务器向客户端发回响应。它是一种标准化协议，使用其他协议(如 HTTP 和 SMTP)发送消息。

SOAP 规范是官方的 web 标准，由万维网联盟(W3C)维护和开发。这是一个正式的协议；它具有严格的规则和高级安全特性，如内置的 ACID 遵从性和授权。更高的复杂性，它需要更多的带宽和资源，这可能导致更慢的页面加载时间。符合 SOAP web 服务规范的 web 服务是 SOAP web 服务。SOAP 代表——简单对象访问协议——可能是这两种模型中比较有名的。

万维网联盟(W3c)定义了一些 SOAP 规则。

*   SOAP: 简单对象访问协议
*   **WSDL:** Web 服务描述语言
*   **UDDI:** 通用描述、发现和集成

**了解:**

它非常依赖 XML，并与模式一起定义了一个非常强类型的消息传递框架。客户机和服务器之间的所有信息交换都采用一种通用格式:XML。它有一个定义好的结构，名为 SOAP messages。其结构由信封、报头和主体报头组成，包括关于消息、认证、复杂类型、路由信息等的信息。主体包含请求的实际信息，这意味着要发送到服务器。

### 它是如何让工作变得如此简单的？

Web 服务是创建、开发和分发 web 服务的一项非常简单的工作。这类似于开发者在当前的 web 开发中所做的事情。趋势是挑选越来越多的工作细节，并使事情自动化，以便节省时间。它的工具包使得开发者很容易忽略这里描述的加密&封装的严格过程。开发人员可以使用的 SOAP 实现的列表长得惊人。

我们精心挑选了三个最受欢迎的工具:

*   面向 Java 的 Apache SOAP
*   SOAP: Lite for Perl
*   微软。网

您使用哪个工具包并不重要；编写、开发、部署和使用 SOAP web 服务的基本过程是相同的。通过访问 http://www.soaplite.com/或 http://www.soapware.org/.，可以获得所有已知的流行 SOAP web 服务实现和工具包的全面和最新列表。所有流行的计算机编程语言(如 C、Perl、PHP 和 Python、Java、C#、C++)都有 SOAP 工具包。

### 你能用它做什么？

我们将把 Soap web 服务应用到现实世界中。我们将看到一个部署 SOAP 和 WSDL 的例子。我们将开发 InfoShare Web 服务，这是一组简单的对等 Web 服务，用于共享应用程序用户信息。InfoShare 应用程序帮助我们共享使用这些应用程序的用户的数据。

客户端的保密性非常重要，因为要实现[数字签名](https://www.educba.com/digital-signature-algorithm/) (DS)并维护一个位于中心位置的用户信息注册表。这些 web 服务是在小型 web 服务接口中开发的。这些接口可以用 Soap web 服务支持的任何语言开发。

### 优势

### 范围

SOAP 协议已经有几年历史了。在 XML 协议版本 1.0 和 SOAP 1.2 之间应该有一个小的升级，因为万维网联盟致力于使用 SOAP，因为它们之间应该有向后兼容性&这将是它们操作的基础。如果有人有兴趣了解关于 XML 协议的讨论，可以参考[https://www.w3.org/2000/xp](https://www.w3.org/2000/xp/)&订阅相同内容。

### 我们为什么需要它？

它为基于 XML 的数据传输定义了一套标准的规则规范。它实现不同的传输协议，如 HTTP & SMTP。标准协议 HTTP 简化了 SOAP 模型穿越防火墙和代理的过程，无需对 SOAP 协议进行任何修改。由于冗长的 XML 格式，SOAP 可能比 CORBA 这样的中间件应用程序要慢。
