### 第 1 部分–APEX 面试问题(基础)

这第一部分包括基本的面试问题和答案。

#### 1.什么是 APEX，它是用来做什么的？

**答案:**
APEX 是一种面向对象的编程语言，是一种强类型的专有语言，由 Salesforce 开发。它将用于开发 Salesforce 中的产品，以执行事务控制语句和流执行操作。APEX 的语法类似于 Java，可用于 web 服务请求。

#### 2.APEX 有什么特点？

**回答:**
APEX 的不同特性是强大而严格的数据集成，可同时执行多个查询和语句，强类型，直接引用对象模式来提供值，易于使用的语法，源自 java 类编程语言，易于测试，可提供更轻松的执行和测试套件创建。

让我们进入下一个 APEX 面试问题。

#### 3.APEX 有哪些应用？

**答:**
APEX 不同类型的应用是创建不同类型的服务，如电子邮件、日程安排、触发等。创建集成多个系统和不同服务的不同 web 服务，对多个模式对象和定制的验证规则执行验证，为不可用的功能创建定制的业务流程。

#### 4.APEX 的架构是什么？

**回答** :
这些是面试中常见的 APEX 面试问题。APEX 架构将涉及应用服务器、数据存储，其中网络将连接到互联网，最终用户和开发者用户将涉及不同的 web 服务器请求以及编译错误和验证。平台应用服务器将源代码编译成指令序列，并将由 Apex 解释器进行解释。最终用户将执行触发器。对应用服务器平台的 web 服务请求不会有延迟。

#### 5.什么是 APEX 语法？

**答案:**
APEX 语法有不同的特性，比如用变量声明来存储内存的不同值。查询将类似于 SOQL，它将用于执行查询，循环语句用于执行操作中的迭代，流控制语句可用于控制流执行是开始还是停止执行过程，DML 语句可用于通过执行查询来操纵数据。

### 第 2 部分–APEX 面试问题(高级)

现在让我们来看看高级面试问题。

#### 6.APEX 开发环境包含什么？

**答:**
sales force APEX 开发环境具有不同的特性和流程，开发一个应用程序并根据实体的要求成功设置和编辑。代码可以在本地开发者版本或 Salesforce 的沙箱中开发。根据标准，代码通常在沙盒中开发，然后部署到生产环境中。将在代码开发期间执行的不同操作是开发代码及其编译过程、调试代码、测试代码和应用程序、执行 SOQL 查询执行及其效率、颜色编码、自动对齐和自动完成构建过程。在执行这些操作之前，在 Salesforce 中执行代码的主要步骤将包括登录到 Salesforce 沙盒或控制台的操作。

#### 7.APEX 中有哪些不同的数据类型？

**答:**
APEX 语言中可用的不同类型的数据类型有原始数据类型(整数、长整型、双精度、日期、日期时间、ID 或布尔型和字符串)、集合(列表、映射和集合)和枚举类、接口和对象。原始数据类型 Integer 可以是 32 位数的任何值，其范围类似于 Java 编程语言的范围。布尔数据类型将有一个 true 和 false 值。日期数据类型只能存储日期值，而不能存储时间。原始数据类型 Long 可以是 64 位数的任何值，其范围类似于 Java 编程语言的范围。字符串数据类型是一组将在单引号内初始化的字符。数据类型 Blob 也存在，它存储一组二进制数据。枚举是一种抽象的数据类型。

#### 8.APEX 有哪些不同的系列？

**答案:**
APEX 中不同的集合数据结构是列表、映射、集合。列表中可以存储任何类型的数据，例如原语、集合、不同类型的复杂对象或模式对象。列表中有不同类型的方法可用于执行不同的操作，例如检索列表的大小、清除内容、获取列表的详细信息，以及更新、删除和添加记录或值。集合和地图也存在相同的功能，但是功能不同。

#### 9.APEX 中的 SOQL 是什么？

**答案:**
以上是高级 APEX 面试中问到的问题。SOQL 通常被称为 Salesforce 对象查询语言，设计和开发用于 Salesforce 数据库。

#### 10.APEX 中有哪些安全功能？

**答:**
APEX 的不同安全规则在共享数据或运行代码时强制执行，以保护应用功能和代码。也有不同的共享安全特征要实施，并且有不同的安全级别。
