## Perl 简介

Perl 是一种编程语言，可用于执行许多不同的任务。典型的易于使用的 Perl 应用程序是从文本文件中提取信息、打印报告和转换文本文件。然而 Perl 为相当复杂的问题提供了许多方法，包括机器编程。Perl 的执行往往比相应的 C 程序占用更多的 CPU 时间。另一方面，计算机会变得越来越快，用 Perl 而不是 C 写东西会节省时间。

### 什么是 Perl？

要理解 Perl，首先要理解什么是脚本——它是由另一个程序处理或解释的程序(或步骤)。如果我们执行一个程序，那么相应的系统执行命令；同样，在脚本中，相应的系统对程序没有任何控制，但是一个不同的程序工作并执行脚本。

Perl 代表实用提取和报告语言。

它是一种脚本语言，语法类似于 C/C++。Web 开发人员用它来为 web 服务器创建脚本。它非常擅长解析文本，所以程序员经常用它来阅读和搜索文本文件。作为一种语言，Perl 在幕后完成了大部分工作(当它们在服务器计算机上运行时)

### 理解 Perl

*   **Perl 变量—**变量有两种类型的值，分别是整数和字符串。整数意味着 1，2，3……..-10(任何+ve 或–ve 值)。字符串是字符序列(用“”或“”表示)。
*   **Perl 操作符–**有两种类型，即–算术或赋值操作符。算术运算符包括+、–*、/、%、**(表示指数)，而赋值运算符包括=、+=、-=、*=、/=、%=、**=
*   **条件语句—**条件语句有助于执行更复杂的场景。If 条件语句–

`If (comparison){
Statement;
Statement;
…………..}`

*   **Loops—**Loop 语句将多次执行特定的场景(根据给定值的要求)。

`While loop – while (comparison){statement; statement; }
For var (setup; cond; inc) {statement; statement; ……}`

### 使用 Perl

使用 Perl 作为您的第一编程语言在用户社区中有不同的意见。支持他们观点的理由可能有很多。在这里，我们将看到他们反馈的可能原因

*   根据大多数开发人员的说法，Perl 的组合与 Python、Ruby 或 Scala 不兼容。
*   奇怪的语法
*   简单的代码结构和复杂的工作一样；只是你处决他们的方式。

Perl 的学习曲线更浅(意味着容易学习)和长(你可以深入挖掘以了解更多关于它的可能性)。另外，它是一种解释型语言；用户可以编写多个程序并测试它们，而不需要或只需要很少的编译步骤；Perl 的这种行为有助于实验这种语言，还可以快速轻松地进行测试和调试。

下面我们列出了一些更有价值的要点——

*   正规表达式
*   混杂
*   内存管理
*   引用和复杂结构
*   模块化程序设计

### 优势

*   有了 Perl archive CPAN，安装不同用户编写的模块变得很容易。
*   它填补了传统 shell 编程与常规编译编程语言(如 [C 和 C++](https://www.educba.com/c-vs-c-plus-plus/) )之间的空白。
*   很适合编写代码简单、行数少的程序，因为它有利于文本操作。
*   这是一个高度可移植的脚本平台。
*   大多数平台都提供费用和许可，如大多数 UNIX 变体/MS-DOS/Windows NT/Windows 95 series/OS/2 和 Macintosh。
*   简单高效——我们将看到比较 C 和 Perl 编写的语法的方法。
*   在 C 语言中，我们将编写类似这样的内容——

`void main()
{
printf("Hello World!");
}`

使用 Perl

`print 'Hello World!'`
