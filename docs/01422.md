## HBase 架构简介

HBase 是一个开源的分布式键值数据存储系统和面向列的数据库，具有高写入输出和低延迟随机读取性能。通过使用 HBase，我们可以执行在线实时分析。HBase 架构具有很强的随机可读性。在 HBase 中，数据在物理上被分割成区域。单个区域服务器托管每个区域，一个或多个区域负责每个区域服务器。HBase 架构由主从服务器组成。集群 HBase 有一个名为 HMaster 的主节点和几个名为 HRegion Server (HRegion Server)的区域服务器。每个区域服务器中有多个区域。

HBase 架构中有主要元素:HMaster 和 Region Server。区域 HBase 保存数据。

#### 1.主人

HMaster 节点是轻量级的，用于将区域分配给服务器区域。

Hmaster 的一些主要职责是:

*   首先，执行一些管理任务，包括加载、平衡、创建数据、更新、删除等。

根据客户端应用程序的指示，负责模式的更改或元数据的修改

*   HMaster 处理 HBase 表上的大量 DDL 工作。

HMaster 接口公开的一些方法主要有。面向元数据的方法。

*   表格(创建、删除、启用、禁用、删除表格)
*   ColumnFamily(添加列，修改列)
*   区域(移动、分配)

客户端与 HMaster 和 ZooKeeper 进行双向通信。它直接联系 HRegion 服务器进行读写操作。HMaster 将区域分配给区域中的服务器，并依次检查区域服务器的健康状态。

区域服务器是处理客户读、写、更新和删除请求的工作节点。区域服务器是轻量级的；它在集群 Hadoop 的所有节点上运行。区域服务器的主要任务是保存区域中的数据并执行客户请求。HBase 区域服务器的另一个重要任务是，当 HBase 表在插入数据后变得过大时，使用自动分片方法通过动态分配 h base 表来执行负载平衡。

HMaster 可以联系多个 HRegion 服务器，并执行以下功能:

*   管理和区域托管
*   自动分割区域
*   处理读写请求
*   直接客户沟通

#### 3.HDFS

HDFS 代表 Hadoop 分布式文件系统。它将每个文件存储在几个块中，并在 Hadoop 集群中复制块以保持容错。HDFS 提供高容错能力，并使用低成本材料。使用廉价的商用硬件向群集中添加节点并进行处理和保存，将为客户带来比现有硬件更好的结果。HDFS 联系 HBase 的组件，以分布式方式保存大量数据。

#### 4.动物园管理员

Zookeeper 是一个开源项目。HMaster 和 HRegionServers 向 ZooKeeper 注册。它提供各种服务，如维护配置信息、命名、提供分布式同步等。分布式同步是在节点之间提供协调服务以访问正在运行的应用程序的过程。它有代表区域服务器的临时节点。主服务器使用这些节点来搜索可用的服务器。

这些节点还用于跟踪网络分区和服务器故障。Zookeeper 是客户端区域服务器之间的交互媒介。如果客户机想要与区域服务器通信，zookeeper 就是通信媒介。

### HBase 架构中搜索如何初始化？

如你所知，元表的位置是由 Zookeeper 保存的。每当客户联系或请求 HBase 时，程序如下。

客户从动物园管理员那里了解如何放置它们的元表。然后，客户端从元表中请求适当的行键来访问区域服务器位置。有了元表位置，客户就可以缓存这些信息。在该区域被移动或移位之前，客户不得查阅元表。然后，将再次请求元服务器，并更新缓存。与往常一样，客户不会浪费时间在元服务器上查找区域服务器位置，因此节省了时间并加快了搜索过程。

### 特征

很容易从源和目的地与 Hadoop 集成。

支持分布式存储，如 HDFS。

它有一个随机存取功能，使用内部哈希表存储数据，以便在 HDFS 文件中更快地搜索。

### HBase 架构的优点和缺点

以下是优点和缺点:

#### HBase 架构的优势

以下是提到的优点:

*   这些可以存储大型数据集。
*   我们可以共享数据库。
*   千兆字节到千兆字节的成本效益
*   通过复制和故障实现高可用性

#### HBase 架构的缺点

下面是提到的缺点:

*   SQL 结构不支持
*   不支持交易
*   仅在键排序时
*   集群内存问题

HBase 是 apache 中面向 NoSQL 列的分布式数据库之一。与 Hadoop 或 Hive 相比，HBase 的性能更好，因为它检索的记录更少。因此，在本文中，我们讨论了 HBase 架构及其重要组件。
