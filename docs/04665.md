#### Q1。解释 oracle financials 中不同的弹性域？

**答案:**

*   **资产:**地点弹性域、资产键弹性域、类别弹性域
*   **总帐:**科目相关弹性域
*   **服务:**项目弹性域
*   **库存:**系统项目弹性域、项目类别弹性域、项目目录弹性域、销售订单弹性域
*   **应收款:**销售税和地点弹性域、地区弹性域

#### Q2。动态插入意味着什么？

**回答:**
这是面试中问的基本 Oracle Apps 技术面试问题。这是 Oracle applications 的一个常见功能，它允许不同类型的用户利用任何窗口/表单输入新的帐户组合。在该特征不可用或被禁用的情况下，用户不能在任何表单或窗口的帮助下创建不同的账户组合。他们有一套固定的组合形式，你可以直接输入不同的代码组合，这是唯一的选择，以防动态插入不可用。

#### Q3。解释在 oracle applications 中，值集是什么意思？

**答:**
任何弹性域段都可以用于显示录入的批准值。所有实际批准值的列表也包含在值集中。以下是与值集相关的一些特征:

*   不同的弹性域可以共享一个值集。例如，值集中包含的所有地区办事处的名称可以是多个弹性域的一部分。
*   不同值集的值范围是有限的。有些限制几乎可以忽略不计，而有些则允许某些值。
*   不同的值集可以有不同的值集。
*   同一弹性域的多个其它段可以使用相同的值集。即使特定弹性域有不同的结构，也可以存在一个公用值集。SRS 或标准请求提交有许多共享相同值集的附加报表参数。

#### Q4。解释会计期状态？

**答案:**

*   **从未打开:**这意味着在此状态下不能输入或过账日记账。
*   **未来可输入:**此状态意味着日记帐可以输入，但不能过帐。“帐套”窗口中显示的固定数字表示将来可输入的期间。
*   **打开:**在打开期间，可以将日记帐输入并过账到任意数量的时间线上。在开放无限期时要特别小心，因为这可能会减慢整个发布过程，并且对于试图进入日志的新用户来说也可能会有点麻烦。
*   **已关闭:**此状态意味着在此期间不能过帐日记帐，如果要过帐日记帐，必须重新打开这些期间。每当您完成年度/月份/季度处理后，应手动关闭这些期间。
*   **永久关闭:**这些是永远不能再打开的时期。清除和存档的数据可以用此状态来描述。

#### Q5。描述该领域的作用，“未来时期”？

**答:**
该值告诉用户可以输入多少个周期的日志条目。在这种情况下，前提条件是未来期间应处于打开状态。还应注意确保用户不会无意中在错误的时间段输入条目。

#### Q6。解释留存收益账户？

**答:**
它用于定义净收入的百分比，该百分比不是股息的一部分，而是由公司保存的，该公司计划将相同的金额再投资于其业务或任何其他与公司相关的财务交易，如支付债务等。股东权益在资产负债表上记录这一点。

#### Q7。解释交易日历的用途？

**答:**
定义为负责应用平均余额交易处理的人。工作日的数量也与可选的交易日历一起标记。
