## CMD 和 BAT 的区别

CMD vs BAT 在这一点上，BAT 是微软用于批处理文件的第一个扩展。这个扩展可以在任何微软系统中使用，包括最早的系统和最新的版本。BAT 是在 DOS、OS/2 和 Microsoft Windows 中使用的批处理文件。CMD 文件是批处理文件的较新版本。较旧的系统不知道 CMD 文件，将部分运行它们，这将对保存为 CMD 的批处理文件抛出一个错误。CMD 是系统中的命令提示符，编程语言称为 DOS 批处理语言。现在 CMD 被很多人使用。

### **CMD 和 BAT 的主要区别**

让我们来讨论一些主要的关键区别:

*   CMD 改进了脚本，因为它是为 Windows NT 开发的。BAT 脚本是为 DOS 命令提示符开发的旧版本。
*   CMD 文件不能在 16 位环境中工作，而 BAT 可以在所有环境中工作。
*   CMD 文件使用最新版本的微软语言，而 BAT 使用旧版本的微软语言。
*   CMD 向后兼容，BAT 不向后兼容。
*   CMD 在大多数 command.com 脚本中运行，而 BAT 本身不会在 command.com 脚本中运行，除非被要求这样做。
*   CMD 不允许在非 NT 系统中执行。BAT 脚本可以在所有系统中运行。
*   要运行 CMD 脚本，我们必须输入 filename.cmd，而要运行 BAT 脚本，只需输入文件名就足够了。当 CMD 和 BAT 文件具有相同的文件名并且保存在相同的目录中时，就会发生这种情况。
*   CMD 提供了运行文件的扩展。BAT 文件运行程序不需要任何扩展名。
*   与 BAT 相比，CMD 更快、更稳定。
*   由于错误变量，两个脚本的执行是不同的。在 CMD 中，误差变量总是重置为零。错误变量保持不变，除非 BAT 脚本中显示错误。
*   CMD 是作为与 cmd.exe 的交互而创建的，与 command.com 没有任何兼容性，尽管命令在 command.com 运行，而 BAT 是在 command.com 环境中创建的。
*   CMD 文件在执行前保存在内存中，但 BAT 文件不保存在内存中，而是逐行执行。
*   我们可以在执行后编辑一次 CMD 文件，这些文件不会产生任何问题。但是在执行后编辑 BAT 文件会给脚本带来问题，它可能根本无法运行。
*   在 CMD 中按 tab 键完成代码，而在 BAT 中代码执行不会以这种方式发生。
*   CMD 中的文件名确实很长，但是 BAT 中的文件名并不长。
*   CMD 只在 Windows-NT 系统中使用，而 BAT 可以在 DOS 和 Windows 中使用。

### 表格的比较

让我们讨论一下 CMD 与 BAT 之间最常见的比较:

| **CMD** | **蝙蝠** |
| 即使命令受到正确或成功的命令扩展的影响，错误级别变量也会改变。 | 当错误显示且可见时，错误级别变量会发生变化。 |
| cmd 在 CMD 解释程序中执行。 | BAT 运行在 NTDVM 环境中，该环境最初是 16 位的，随着新版本的发布得到了升级。 |
| cmd 扩展名在 cmd.exe 是只读的。 | 蝙蝠扩展可以在 command.com 和 cmd.exe 看到。 |
| 当使用 CMD 调用错误级别时，错误级别被重置。 | 当使用 BAT 调用错误级别时，执行前面的命令。 |
| CMD 是后来发展起来的，有很多高级功能。 | BAT 没有太多的高级功能，现在正在使用 CMD 来实现这些功能。 |
| 在 win9x 中，CMD 不被视为可执行文件类型。Win2k 和 NT 执行 CMD 文件时不会引发任何错误。 | win9x 中的文件是用 BAT 执行的。此外，它还执行其他 windows 版本的文件。 |
| 几乎所有版本的 windows 都支持 CMD 文件。 | 新版 windows 不支持 BAT 文件。 |
| Cmd.exe 直接执行 CMD 文件。 | Command.com 将英美烟草公司的文件发送到 cmd.exe 执行。 |
| CMD 脚本在命令提示符下编写。任何类型的脚本都可以在命令提示符下运行，并且扩展名为。cmd 文件。 | 微软的批处理使用 BAT 文件，存储在。bat 格式。 |
| 命令和实用程序以及编程语言使得 CMD 独一无二。 | 该程序只是一个批处理脚本，称为批处理文件。 |
| 文本文件名在命令提示符下给出，有助于编辑。cmd 文件。 | 任何文本编辑器如记事本或 MS word 都可以用来打开。bat 文件进行编辑。 |
| 文件不是按顺序操作的，非常需要用户干预来支持文件。 | 文件是按顺序操作的，因此不需要用户干预。用户可以直接管理文件。 |
| CMD 是命令提示符的同义词，尽管命令提示符按照指示运行许多程序。 | BAT 文件并不总是使用命令提示符执行，它们有自己的执行方式。 |
| 几乎所有程序都使用 CMD 文件。为了检查软件，版本和许多其他目的，它被使用。 | BAT 文件有助于启动 Windows 系统中的大程序，也有助于维护。 |
| CMD 有一个运行脚本的编程语言，并且是一个命令解释器。 | BAT 是一种有解释器的脚本语言。 |
| CMD 不运行命令序列，因此我们可以说 CMD 比 BAT 更安全。 | 当 BAT 文件被打开时，它运行命令序列，因此容易受到安全攻击。 |
| CMD 在 Android 中有一个应用程序，可以查看、安装和运行 CMD 脚本。 | BAT 文件不能在 Android 中使用，因为它只适用于 windows。 |

### 结论

用户可以交替使用 CMD 和 BAT 来实现重复性任务的自动化。两者有很多相似之处，可以认为是一种简单的编程语言。命令可以带参数执行，也可以不带参数执行。简单的脚本很容易在两个程序中执行，命令也非常相似。
