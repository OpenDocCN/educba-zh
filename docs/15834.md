## Unix Shell 简介

UNIX Shell 是一组允许多个用户执行多项任务的计算机操作系统。shell 是与内核交互的媒介，内核是 Unix 系统的命令行界面(CLI)。它基本上是 Unix 系统的命令行界面。它的工作方式是收集用户的输入，根据用户的输入执行程序，然后在程序执行后显示输出。

在本主题中，我们将了解什么是 Unix shell。

### 它是如何工作的？

*   它需要你的输入。
*   它根据你的输入执行程序。
*   一旦执行完毕，它就会显示输出。

### 了解 Unix Shell

它是内核的额外一层。

内核:

*   帮助控制所有计算机操作。
*   它协调所有要执行的实用程序或程序。
*   它管理实用程序的资源消耗。它管理所有的系统进程。

因此，与内核相关联为您提供了一个平台来交互和执行实用程序和程序。

### 外壳的类型

Unix 中有两种类型的 shells:

*   **Bourne shell** :在 Bourne 类型的 shell 中，默认提示是 **$** 符号。

以下是 Bourne shell 的 4 个子类别:

1.  伯恩·谢尔
2.  Korn shell (ksh)
3.  伯恩再次炮轰(bash)
4.  POSIX 外壳(sh)

*   **C shell** 在 C 型 shell 中，默认提示是%字符。

以下是 C shell 的两个子类别:

1.  c 外壳(csh)
2.  TENEX/TOPS C 壳牌公司(tcsh)

最初的 Unix shell 是由 Leslie Stephen R. Bourne 爵士在 20 世纪 70 年代中期编写的，当时他在新泽西的美国电话电报公司·贝尔实验室。Bourne shell 是 Unix 上出现的主要 shell。因此，它被称为“贝壳”。在大多数 Unix 操作系统版本中，Bourne shell 有时会放在/bin/sh 中。因此，它是编写脚本的备选 shell。

### 使用

*   常见任务由 it 部门自动完成。例如，如果您想要备份您的主目录并将其上传到服务器上，您可以编写一个脚本并计划每天或每周运行它。
*   您可以通过编写脚本来自动化您在新系统上编写的软件设置。
*   通过编写脚本并从调用它，无论何时登录到系统，您都可以做更多的事情。shrc/。bashrc/。zshrc(这取决于源路径)。
*   它节省时间，因为它让我们可以有效地做重复性的工作。如果你有一个相同的脚本，你不必一次又一次地输入命令。
*   您可以编写一个脚本来完成不同的任务，如一起杀死或启动多个程序，或者启用/禁用某些功能。

### 优势

*   **任务自动化:**它有助于自动化频繁执行的任务，这意味着您可以编写一个脚本来执行每天的任务或需要执行多个命令的任务。举个例子，

这很容易-

1.  每天在云上上传一个文件或文件夹。
2.  压缩每天增长的日志文件。

*   **合并多个命令:**将多个命令合并成一个命令。执行单个命令更容易，因为您不必记住命令的顺序或它们的执行顺序。
*   **更容易开发:**用 C/C++在一个程序中编写相同的脚本是可能的，但是比 C/C++程序更容易编写和调试 shell 脚本，尤其是对于系统管理任务。
*   **透明性:**由于脚本是文本文件，所以更容易看到和检查脚本正在执行什么操作，但是在 C/C++程序中，您需要访问源代码文件。
*   **可移植性:**如果有 shell，那么在其他 Unix 或类似 Unix 的操作系统上执行脚本会更容易。
