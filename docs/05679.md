## NoSQL 数据建模简介

NoSQL 数据建模被定义为允许我们保留许多数据的 NoSQL 数据库。可以从任何位置和设备访问这些数据，但是我们无法决定哪种数据建模技术适合我们的需求。此外，NoSQL 数据模型已被构建用于构建高效和快速的查询，在这种情况下，它可以在一秒钟内创建数百万个查询，因为所有数据都包含在一个表中；因此，连接和交叉验证不会大量执行。

### 什么是 NoSQL 数据建模？

NoSQL 数据建模技术在诸如逻辑到物理映射、规范化和反规范化等训练有素的技术中发挥了重要作用；目前，由于缺乏系统的理论，数据建模在实践和理论方面都没有得到有效的研究；它有不同的类型，可以以键-值形式保留数据，这是一种键-值存储，可以根据要求创建查询，如果要求很高，这也很有帮助，另一种类型允许以文档形式存储信息，下一种类型允许以列的形式存储数据，这种数据建模还允许以图形形式存储数据，其中数据之间有关系。

### NoSQL 数据建模的类型

以下是 NoSQL 数据建模的类型:

#### 1.键值存储

这种类型的数据建模是为高生产需求构建的，但这是最常见的数据模型，在这种模型中，它可以使用带有指针的键值来保留数据。尽管指针是唯一的，并且可以与一段数据绑定，但它允许使用空字符串作为值键。

#### 2.基于文档的存储

SQL 不需要保留信息；它不需要知道文档存储在哪里，因为 SQL、XML 和 JSON 是绑定在一起的，这会降低查询速度。

数据已存储在此存储的表中，因此不需要交叉引用；它存储在一个类似于键值存储的文档中。

#### 3.基于列的存储

这种数据模型能够将信息保留在列中，而不是通常在 SQL 中发生的行中；在这种数据模型中，通常，数据被保留在列中，列被进一步分组为族，族被进一步划分为更多的列，以便可以生成列的嵌套。

#### 4.基于图形的存储

这个数据模型是任意两个数据之间有关系的网络数据模型；可以为我们想要在图上构成的任何数据创建这种类型的数据模型。可以利用关系和节点，其中数据是节点，节点之间会产生关系。

### NoSQL 数据建模基本原则

有一些基本原则，

*   它有一个原则，他们有一个很好的设计；因此，不允许反规范化和数据复制。
*   由于数据重复，它一直将内容复制到各种文档中来克服它。
*   并且由于反规格化，用户可以在特定数据模式下修改特定数据。

### NoSQL 数据建模技术

有三种建模技术:

#### 1.概念技术

*   **反规范化:**这是一种非常简单的技术，可用于将数据复制到多个表中以进行澄清；它可以将所有需要查询的数据集中在一个地方。
*   **聚合:**在这种技术中，用户可以形成复杂的实体结构，在这种结构中，通过保持一对一的关系，可以克服聚合，在这种关系中，大多数数据模型都有一些标准技术。
*   **应用程序端连接:**NoSQL 不辅助连接，因为它们是面向问题的，其中连接在设计时就已经完成，这可以与关系数据库相比较。

#### 2.通用建模技术

*   **可枚举键:**在 NoSQL 数据建模中，未分级的键值是起作用的，因为该方法可以统一在专用服务器上；需要通过排序有序键的特征来添加一些配置。
*   **降维:**可以用来克服多维数据到键值模型的映射。
*   索引表:这种技术仅用于存储目的，而不是在内部支持它们。
*   **组合键索引:**当使用有序键时，这种常用技术很有用。
*   **反向搜索-直接聚合:**这种技术已经被用于满足一组特定的标准。

#### 3.层次建模技术

*   **树聚集:**这种技术适用于单个文档，当用一个记录处理该文档并且可以访问一个记录时，可以对该文档进行结构化。
*   邻接表:这是一种简单的技术，在这种技术中，节点被建模为具有直接前身的个性化数组记录。
*   **物化路径:**这种技术是一种反规格化，可以用来避免重复的树结构遍历。
*   **嵌套集合:**这种技术已经在关系数据库的树状结构中使用，它可以与 NoSQL 和键值或文档数据库相关。
*   **嵌套文档展平:**这种技术已经被用于将复杂的结构映射到简单的文档。
