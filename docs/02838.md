## Hadoop 职业介绍

Hadoop 不仅仅是大数据世界中的一个框架。它有一个广泛的生态系统，拥有相关技术的保护伞。出于同样的原因，Hadoop 领域的职业前景也很好。如果你对 Hadoop 基础有很好的理解，这将是你在 Hadoop 领域取得伟大事业的基础。

### 从事 Hadoop 职业所需的教育

像许多新兴的数据技术一样，Hadoop 不需要任何特定的教育背景。大约一半的 Hadoop 开发人员来自非计算机科学背景，如统计学或物理学。所以很明显，如果你准备好学习基础知识，背景并不是进入 Hadoop 世界的障碍。有很好的涵盖 Hadoop 的在线课程 eduCBA 的课程是最好的例子——master-Apache-Hadoop

<small>Hadoop、数据科学、统计学&其他</small>

此外，如果您希望深入了解 Hadoop 集群管理的特定领域或 Hive 材料中关于每个特定主题的数据建模，如在线课程和教科书，大多数情况下 Hadoop 集群将设置在 AWS 或 Azure 等云供应商中。因此，熟悉你选择的任何云供应商都会有很大帮助。AWS 的 Hadoop 服务叫做 EMR。

热门专业包括:

*   spark–可扩展的内存数据处理引擎
*   h base—[在 HDFS 之上的](https://www.educba.com/what-is-nosql-database/)没有 SQL 数据库
*   波束流第一进场数据处理
*   pig–数据转换(ETL)脚本
*   hive–数据仓库
*   Mahout，Spark ml lib–Hadoop 上的可扩展机器学习
*   Apache Drill–Hadoop 上的 SQL 引擎
*   Flume，sq OOP–数据摄取服务
*   Solr & Lucene–搜索和索引

### Hadoop 中的职业道路

根据 2017 年堆栈溢出调查结果，Hadoop 在大数据领域最受欢迎和喜爱的框架中处于领先地位([调查链接](https://insights.stackoverflow.com/survey/2017#technology))。这是可能的，因为来自不同 IT 视角的人发现 Hadoop 是一条潜在的职业道路，并希望转换。

无论你目前的角色是什么，在 Hadoop 世界中都会有一个容易适应的职业转换。一些流行的例子——

*   **软件开发人员(程序员):** Hadoop 数据开发人员，处理不同的 Hadoop 抽象 SDK，从数据中获取价值。
*   **数据分析师:**原来你精通 SQL。在 Hadoop 中，在 SQL 引擎上工作的巨大机会，如 [Hive 或 Impala](https://www.educba.com/hive-vs-impala/)
*   **业务分析师:**试图利用大量收集的数据变得更有利可图的组织和业务分析师的角色在这方面至关重要。
*   **ETL 开发人员:**如果你是一名传统的 ETL 开发人员，你可以使用 Spark 工具轻松转移到 Hadoop ETL。
*   **测试人员:**Hadoop 世界对测试人员的需求巨大。通过理解 Hadoop 和数据分析的基础，任何测试人员都可以转换到这个角色。
*   **BI/DW 职业:**可以轻松切换到 Hadoop 数据架构到数据建模。
*   **高级 IT 专业人员:**凭借对数据领域和现有挑战的深刻理解，高级专业人员可以通过了解 Hadoop 如何解决这些挑战而成为顾问。
*   有像数据工程师或大数据工程师这样的普通角色，他们主要负责在云供应商的基础上实施解决方案。通过了解云提供的数据组件，这将是一个很有前途的角色。

### 就业岗位

Hadoop 生态系统提供了多种职业道路。

*   MapReduce 开发人员:这基本上是一个 Java 开发人员的角色，他们也了解 Hadoop 系统如何在内部工作。有一个像 Hive 或 Pig 这样的抽象可用，但 MapReduce 作业对于高性能系统是必要的。MapReduce 开发人员是一个对系统了如指掌的人，报酬非常高。
*   **Hadoop 管理员:**这些人负责保持 Hadoop 集群的健康和运行。这可能包括典型的 [Hadoop 管理员](https://www.educba.com/hadoop-administrator/)任务，如定期系统健康检查，但大多数任务是理解 Hadoop 系统架构所必需的。
*   **Devops:** 在 Hadoop 集群中部署新的系统组件和其他与开发相关的变更。这个角色的职责变化很大，取决于组织的文化。
*   **数据开发者:**Hadoop 之上的数据处理。这是 Hadoop 生态系统中最受欢迎的角色之一。具有 SQL 或分析背景的人最适合这些角色。主要从事 Hadoop 的高级抽象，如 Hive 或 Pig。
*   **数据安全管理员:**数据是最有价值的资产，保护数据是最重要的。安全管理员了解系统的局限性，确保采用行业标准策略和最佳实践来保护数据。
*   **Data Visualizer:** 处理下一代可视化工具，这些工具支持动态数据切片和聚合以及内存中的数据缓存
*   **ETL 开发人员:**使用 Hadoop 生态系统工具转换数据以提高数据质量或根据业务逻辑进行转换。 [ETL 过程可能是](https://www.educba.com/etl-process/)流或批处理。
*   **系统架构师:**以经济高效的方式设计考虑数据可用性和持久性的高性能系统。严重依赖硬件提供商。
*   **Data Architect:** Apart from traditional Logical/Physical design of data, a lot of things like column encoding, denormalization, partitioning design etc. will be a responsibility of data architect.

### 职业前景

[Hadoop 生态系统正变得越来越多样化](https://www.educba.com/hadoop-ecosystem/)以满足业务需求的变化。随着生成的数据呈指数级增长，越来越多的组织成为数据驱动型组织，Hadoop 系统的相关性只会增加。

一些值得注意的趋势:

*   从批量处理转变为使用火花和光束的流优先数据处理方法
*   使用 Spark ML 将更多实时机器学习模型应用于实时数据。
*   将 SQL 引擎从数据存储中分离出来，就像在 S3 上使用 Presto 在数据湖上进行即席分析一样。
*   像 [AWS Redshift](https://www.educba.com/what-is-aws-redshift/) 这样的柱状 MPP 数据库可以实现快速数据访问

由于大数据处理的一个基本方面在于容错分布式和水平可扩展系统，Hadoop 很好地实现了这一点，因此 Hadoop 将继续作为数据处理的领先生态系统。
