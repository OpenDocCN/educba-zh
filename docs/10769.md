## Maven 相位简介

Maven 是一个构建工具，也可以用于许多其他目的，比如依赖管理、单元测试、项目文档和部署。在这里，我们将了解 maven 构建生命周期的不同阶段。如果一个人知道一些构建项目和执行阶段所必需的命令。他可以很容易地构建任何 maven 项目，因为 POM 可以确保获得所有其他依赖项，并且输出期望的结果。在执行任何阶段时，必须记住唯一指定的阶段没有运行，而是执行指定阶段之前的所有阶段和提到的阶段。

### 构建生命周期和阶段

Maven 为我们提供了三个内置的构建生命周期。这些构建生命周期是默认的、干净的和现场的。它们中的每一个都有一些用途和用法，并且还包含许多构建阶段，通过这些阶段来定义它。默认的构建生命周期有助于处理和维护项目的部署。第二个构建生命周期，即清理构建生命周期有助于清理项目，而站点构建生命周期负责项目站点的文档创建。每个内置的构建生命周期都由不同的阶段组成。默认的生命周期由 23 个阶段组成，清理生命周期由 3 个阶段组成，站点生命周期由 4 个阶段组成。

例如，默认的构建生命周期由以下一些阶段组成，这些阶段在生命周期中充当不同的阶段。

*   **验证阶段:**该阶段确保所有必要的东西都存在，并且项目在每个要求的格式上都是正确的。
*   **编译阶段:**在这个阶段，对项目的源代码进行编译。
*   **测试阶段:**在这个阶段，在任何单元测试框架的帮助下，对源代码进行一些基本的测试用例的测试，这些测试用例不包括其中的打包或部署。
*   **打包阶段:**将编译好的代码打包成可分发的格式，比如 JAR。
*   **验证阶段:**为了确保质量得到维护，在这个阶段对当前源代码进行集成测试。
*   **安装阶段:**经常需要在某个其他项目中添加某个项目的依赖项。此阶段确保正确安装项目包，以便通过添加当前正在构建的项目的依赖项来使其他项目可用。
*   **部署阶段:**在这个阶段，构建完成，当前项目被复制到远程存储库。这使得构建的项目可供其他项目作为依赖项或开发人员添加。

三个构建生命周期的所有阶段都是按顺序执行的。在默认的构建生命周期中，主要验证项目的正确性，并确保正确满足所有要求，之后编译源代码，然后进行单元测试，接着进行集成测试，并在本地存储库上安装打包的项目，之后通过将项目部署在远程存储库上，其他项目和开发人员可以使用该项目。

### Maven 命令

我们可以根据自己的方便性和必要性选择希望运行的任何阶段，并使用下面的格式从命令行发出命令。

#### 1.mvn 包

上述命令将为您创建一个 jar 或 war 文件，这取决于您使用的项目类型和语言。如果您的应用程序是一个 java 项目，那么将生成 java 档案，即 jar，对于 web 应用程序，将创建 web 档案，即 wars。根据您的目的，您可以启动不同的命令并运行不同的阶段，例如为了执行单元测试，您将启动 mvn 测试命令等等。每当您执行某个命令时，该命令前面的所有阶段也将执行。

例如，无论何时启动 mvn verify 命令，序列中在它之前的所有阶段(验证、编译、测试和打包)都将被执行。因此，每当您想要执行某个阶段时，只需选择您想要执行构建生命周期的最后一个阶段，所有之前的阶段也会被执行。

请注意，某些阶段不能从命令行调用。

#### 2.mvn 干净部署

每当您想要清理项目并将您的项目或工件部署到可以进一步共享的存储库中时，您可以使用这个命令。当您的项目由子项目组成并且包含多个项目时，也可以使用此命令。在这种情况下，每个子项目都被清理和部署，也就是说，清理和部署构建周期是为每个子项目执行的。这被称为 maven 支持的多模块结构。

### 插件目标

*   通过将生命周期的每个阶段分解成多个插件目标，构建生命周期或任务被进一步分解成粒度级别，这些插件目标是负责管理和构建工件/项目的小任务。
*   一个插件可以绑定到零个、一个或多个构建阶段。
*   只要它不属于任何阶段，那么就可以通过直接调用它在生命周期构建过程之外单独执行。
*   从 Maven 2.0.5 开始，POM 决定属于同一阶段的多个插件的执行顺序。

### 结论–Maven 阶段

Maven 有三个内置的构建生命周期，名为 default、clean 和 deploy。每个构建生命周期都由不同的阶段组成，这些阶段甚至被称为按顺序执行并执行某些特定任务的阶段。每个阶段进一步由一个或多个目标组成，这些目标有助于管理和构建项目。
