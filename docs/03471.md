## QlikView 与 Qlik Sense 的区别

我们先来谈谈 QlikView，这是一款用于分析解决方案的工具。它是由成立于 1993 年的 Qlik 公司开发的。这些公司为我们提供并帮助我们建立商业分析平台。QlikView 是一款工具，让我们能够设计和开发交互式分析应用程序及其仪表盘。QlikView 工具的主要优势在于它可以动态生成数据视图，帮助我们压缩数据并将它们保存在内存中。另一方面，我们有 Qlik Sense，它也是 Qlik 的另一个产品，用于数据集成和业务分析，是一个商业智能工具，为我们提供了许多功能，还提供了许多高级的可视化和分析服务，简而言之，它帮助我们进行可视化和数据分析，借助于此，我们还可以集成来自各种来源的数据。在本教程的下一节中，我们将详细了解这两种工具的主要区别，以便更好地理解和判断何时使用什么。

### 主要差异

现在，我们将更深入地了解这两种工具的主要区别。为了更好地理解，我们将详细讨论它们，见下文；

**QlikView**

1)如果我们讨论 QlikView 的架构细节，那么我们有以下几点:它基本上由前端和后端部分组成，让我们详细讨论每一个部分；

**a)前端:**它是 QlikView 的重要组成部分，帮助我们从浏览器访问 QlikView 文档。该组件包含 QlikView server，可帮助企业用户直接访问 QlikView server 上已有的报告。它为用户提供了一个基于网络的用户界面。如果商业用户希望与转帖互动，那么他们可以利用前端组件，分析他们的数据，并做出决定。通过使用前端组件，业务用户也可以与其他用户共享数据，这可以在线或离线进行。因此，QlikView server 基本上维护前端和后端组件之间的通信，即客户端-服务器通信。QlikView server 中的所有文档都将保存在。qve 扩展名，但我们可以将它们作为独立的文档存储在 windows 操作系统中。

b)现在后端组件由名为 QlikView Publisher 和 QlikView desktop 的两部分组成。这是一个 windows 环境，支持从数据源加载和转换数据的功能。我们可以创建带扩展名的文件。qvw，然后这些文件传递到 QlikView 服务器进行进一步处理，这里我们使用 QlikView publisher，它帮助我们在各种 QlikView 服务器和用户之间分发文件，它还负责为我们处理安全性和授权。

对于 Qlik 感觉

2)另一方面，我们有 Qlik 感觉，它进一步包括下面提到的不同成分；

**a) Qlik Sense 管理控制台:**这是一个管理组合，帮助我们从控制台集中管理 Qlik Sense。它基本上为我们提供了以下功能，这些功能可以在 Qlik Sense 仪表板中集中管理；

1)安全管理

2)任务管理

3)审计

4)数据监控

**b) Qlik Sense hub:** 这是 Qlik Sense 的另一个组件，它帮助我们对数据执行操作，通过它我们可以创建数据可视化，这是用户可以与软件交互并能够为其应用程序创建仪表板和分析数据的部分。Qlik Sense hub 是使用 CSS3、HTML 5、JavaScript 等开发设计的。

**c) Qlik Sense 存储库:**这是 Qlik Sense 的另一个组件，它帮助我们存储所有数据和信息，这些信息基本上与我们现有的其他 Qlik Sense 服务的管理和配置有关。设计和可视化数据对用户更友好

**对照表**

让我们用这两个工具来看一下表格，以便更清楚地了解情况；

| **QlikView** | **Qlik Sense** |
| 这是 Qlik 公司开发的工具。 | 另一方面，Qlik Sense 也由 Qlik 开发 |
| 这个工具被技术用户使用，因为它需要通过脚本操作数据 | 另一方面，非技术用户也可以使用它 |
| QlikView 工具没有为我们提供可视化的数据准备 | 但是在 Qlik Sense 的帮助下，我们可以实现可视化数据准备 |
| QlikView 也不提供数据挖掘分析 | 但是 Qlik Sense 为我们提供了数据挖掘和分析功能 |
| 就 QlikView 而言，我们没有图表建议和人工智能支持的见解 | Qlik Sense 还为我们提供图表建议和人工智能支持的见解。 |
| 在 QlikView 中，我们不能使用自然处理语言 | 另一方面，在 Qlik Sense 的情况下，我们也可以利用自然处理语言。 |
| 它还为我们提供内存数据处理，并帮助我们将数据存储在报告本身中 | 另一方面，我们有由各种组件组成的快速感觉，帮助用户创建数据并可视化它们。 |
| 它为我们提供了透明度、一致性，这是各个级别和管理层所需要的。 | 组件，如 Qlik Sense Hub、Qlik Sense 管理控制台 |
| 它还帮助我们整合来自各种来源的数据，并帮助他们可视化这些数据 | 它帮助我们进行数据监控、数据审计、应用程序管理和数据连接。安全性和任务管理。 |

### 结论

在这篇文章中，我们已经看到了这两个工具之间的主要区别，可以根据需要使用，如果我们想要更多基于人工智能的任务，那么我们可以选择 Qlik Sense，因为它支持更多的任务。另一方面，QlikView 更适合技术用户，但易于学习和操作。
