## Redis 备选方案介绍

让我们先试着理解 Redis 是什么，然后是它的替代品。Redis 是远程词典服务器的缩写。它是一个内存中的键值数据存储，也是开源的。它速度极快，可以用于缓存会话管理、高性能数据库和消息代理。它是一个键值存储，这个键值存储可以用作读写数据的存储库。

#### 1.MongoDB

MongoDB 是一个面向文档的存储平台，允许用户像存储文档一样在 JSON 中存储[对象，而不是结构不同的表，从而提供了一个动态灵活的模式。因为与单个对象相关的所有数据都可以存储在其数据库中，所以 MongoDB 对于拥有来自多个来源的数据集的项目特别有用。](https://www.educba.com/what-is-json/)

*   否-SQL
*   易用性
*   快速的
*   高性能
*   自由的
*   开源
*   易于扩展

#### 2.Memcached

它是分布式内存对象的高性能缓存框架。 [Memcached 是一个键值存储](https://www.educba.com/what-is-memcached/)内存中的小块任意数据，即字符串、来自数据库调用的对象、API 调用或页面渲染性能。它旨在通过减少数据库的负载来提升动态 web 应用程序。

**Memcached 的特性**

*   快速对象缓存
*   高性能
*   稳定的
*   成熟的
*   分布式缓存系统
*   改进的响应时间和吞吐量
*   非常适合缓存 HTML

RabbitMQ 是一个传统的消息代理，实现了多种消息协议。为了实现高级消息队列协议(AMQP)，最初开发了 RabbitMQ。AMQP 在生产者、经纪人和消费者的帮助下标准化消息传递。

**rabbit MQ 的特点**

*   消息的吞吐量是每秒一百万条
*   更好的路由系统
*   持久的
*   这种方法工作迅速，并具有合理的度量/监控
*   易于配置
*   分布式且可靠

ArangoDB 数据库是一个开源的 NoSQL 应用程序，具有可扩展的数据模型，用于图形、地图、文档和键值。使用类似 SQL 的查询语言或 JavaScript 扩展构建高性能框架

#### 11.SAP Hana

SAP Hana 是一个关系数据库管理系统，具有内存和面向列的特性。它利用了内存中的特性，能够在短时间内处理大量的实时数据。内存处理引擎允许它处理存储在 RAM 中的数据，而不是从磁盘中检索数据。

**SAP Hana 的特性**

*   易于与其他 SAP 工具集成
*   非常快
*   内存缓存
*   易于部署
