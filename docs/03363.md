## 雷兹诉蒙戈布案简介

最流行的数据存储解决方案变成了 Redis 和 MongoDB。他们的诉求是为用户提供一个简单高效的结构。这两种方法都非常适合连接数据库在可伸缩性和简单性方面表现不佳的情况。MongoDB 是一个面向信息存储文档的解决方案，支持服务器或云安装。MongoDB 中的数据库是以 JSON 文件的形式提供的，对于许多应用程序来说，这是一种有效而简单的方式。然而，Redis 是用户服务器、高性能集群或云信息的关键价值存储解决方案。MongoDB 和 Redis 都可以很好地与 Java、Python 等语言和框架一起进行编程。

**什么是 MongoDB？**

<small>Hadoop、数据科学、统计学&其他</small>

如前所述，MongoDB 是一个优秀的面向文档的数据库。它有很多优点，但没有缺点。与此同时，MongoDB 满足管理大范围信息的极其复杂的标准，并能相当快地操纵数据处理。MongoDB 以用户友好的方式为 JSON 文档的输入和输出存储句柄和使用二进制数据。此外，命令行中的 MongoDB 接口简单，用户友好，易于使用。MongoDB 在二进制映像中操作，其中每种类型的 BSON 都有每种类型的值的标识整数和字符串。BSON 的最大容量是 16 兆字节。

**什么是惯用语？**

Redis 是一个远程字典服务器，它实现了键值存储原则。这是开源 NoSQL 数据库的一个很好的例子。Redis 通过内存存储和永久设备(如 SSD 和 HDD)的快照，以及保留在虚拟内存中的定时数据存储机制，提供高性能的数据读取和保存。除了这些显著的特点，Redis 还提供了一个传输系统，它可以在一个集群中的多台计算机上进行复制，支持十几种编程语言，并可以在许多操作系统中运行。Redis 支持可以连接到键的数据类型的值。这些数据类型有助于管理字符串、集合、列表、排序集合、哈希表和超级日志等信息。

### Redis 和 MongoDB 的主要区别

让我们从以下几点来讨论 Redis 与 MongoDB 之间的一些关键差异:

**1。** Redis 是位于用户、云或集群终端的键值存储解决方案，而 MongoDB 是基于文档的存储解决方案，可以安装在云中或用户终端的服务器上。

**2。**由于与 MongoDB 数据库相比，Redis 可以更快地管理数据库，因此我们可以说，在较高的延迟率方面，Redis 比 MongoDB 快得多，但在较低的延迟率方面，情况就完全不同了。当谈到更低的延迟率时，MongoDB 默认击败 Redis。

**3。MongoDB 甚至比 Redis 工作得更好，尽管它仍然有一个计算机内存。但是 MongoDB 一旦连接上 CPU，性能就慢慢变慢了。**

**4。** MongoDB 在 Redis 的物理机器上支持额外的扩展方法和 RAM，在扩展场景上明显不如 Redis。

**5。** Redis 附带了一个内存引擎，让工具可以轻松地在较高的工作负载设置下运行，而 MongoDB 附带了一个 Percona 引擎，让工具成为一个内存存储引擎。

**6。** Redis 可与现有数据库一起用作备用数据库。另一方面，MongoDB 被用作组织中唯一的数据库。

7 .**。** Redis 的可伸缩性不如 MongoDB，但同样高度灵活，它比 Redis 更具可伸缩性。
