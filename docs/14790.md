## SVG 和 Canvas 的区别

SVG 缩写为可缩放矢量图形。它是一种基于矢量的图形，使用基于 XML 的图形格式，为交互提供支持。SVG 图像比位图图像好得多。在 SVG 图像中，矢量图像由一组固定的形状组成，在缩放这些图像时，它会保留图像的形状。Canvas 是一个 HTML 元素，用于在网页上绘制图形。它被称为位图，具有用于在其上绘图的即时模式图形应用编程接口。元素 canvas 用作图形的容器。在 Canvas 中，我们需要脚本来绘制图形。

**让我们更详细地研究一下 SVG 和 Canvas:**

<small>网页开发、编程语言、软件测试&其他</small>

*   W3C 开发了 SVG。它最初发布于 2001 年。文件扩展名是。svg 和. svgz .其互联网媒体类型为 image/svg+xml，统一类型标识符为 public.svg-image。位图图像由一组固定的像素组成，在缩放位图时，它会向我们显示图像的像素。SVG 图像可以借助 Inkscape、adobe illustrator、adobe flash 等矢量图形编辑器生成。
*   由于 Internet Explorer 等浏览器缺乏对矢量图像的支持，SVG 在 web 上的使用受到了限制。Konqueror 是 2004 年第一个支持 SVG 图像的浏览器。之后，慢慢地，谷歌宣布支持网页内容上的矢量图像。SVG 现在也支持本地浏览器、插件和移动浏览器。
*   Canvas 非常具有交互性，它可以响应用户与任何触摸事件、关键字和鼠标的交互。Canvas 允许选择保存图像。Canvas 在一次处理多个元素时非常有效，并且在画布上绘制的对象可以是动画。
*   Canvas 主要依赖于分辨率，并且它具有复杂的可视化，因为它有时在绘制大区域时会很慢。有几种策略可用于绘制不同的形状，如路径、方框、圆形、文本和添加图像。

### SVG 和 Canvas 的主要区别

两者都是市场上的热门选择；让我们讨论一些主要的区别:

*   SVG 不依赖于分辨率，这意味着它与分辨率无关。如果我们把图像放大，它不会变形。画布依赖于分辨率。如果图像被放大，它将开始反射图像的像素。
*   SVG 指的是基于形状，而 Canvas 指的是基于像素。
*   SVG 最适合具有大渲染区域的应用程序，如 google maps。Canvas 的文本渲染能力很差。
*   如果 SVG 很复杂，那么它的渲染速度会很慢，因为任何大量使用文档对象模型(DOM)的东西都会变得很慢。Canvas 提供了高性能元素，最适合于渲染更快的图形，如图像编辑，这是一种需要像素操作的应用程序。
*   SVG 在浏览器中功能强大，因为它是基于矢量的，并提供高质量的体验。它可以与多媒体、音频和视频配对。Canvas 主要用于绘制形状、图表和复杂的照片构图。
*   SVG 可以通过脚本和 CSS 修改。画布只能通过脚本修改。
*   SVG 图像不能以其他格式保存。在 Canvas 中，您可以将结果图像存储在。巴新和。jpg 格式。
*   SVG 主要推荐用于全屏用户界面。大屏幕不推荐使用画布。
*   如果对象有大量小元素，SVG 文件的大小会增长得更快。对于画布图像，文件的大小不会增加太多。
*   SVG 更适合元素或项目较少的应用程序。Canvas 主要更适合成千上万的对象和小心的操作。
*   SVG 图形主要使用数学函数和公式开发，需要在源文件中存储较少的数据。在 Canvas 中，要绘制图形，需要开发许多策略。
*   在 SVG 中，事件模型或用户交互是抽象的。对于 Canvas，事件模型或用户交互是细粒度的。
*   SVG 提供了更好的可伸缩性，因为它可以在任何分辨率下以高质量打印。Canvas 的可扩展性很差，因为它不适合打印更高的分辨率。
*   SVG 为更大的表面或更少的对象提供了更好的性能。Canvas 为较小的表面或大量对象提供了更好的性能。
*   SVG 语法很容易理解，但是无法读取图形对象。Canvas 语法非常简单，易于阅读。

### 结论

SVG 和 Canvas 都用于创建或开发图像和形状。开发人员根据需求同时使用 SVG 和 canvas 来解决他们的目的，例如 SVG 不用于创建游戏等动态应用程序，Canvas 不用于其较差的文本渲染和缺乏动画。SVG 和 Canvas 都用于在 web 上创建丰富的图形，但是它们有本质的不同。

SVG 主要依靠文件，而 canvas 主要使用脚本。SVG 被认为更容易访问，因为它们支持文本，canvas 依赖于 Javascript。所以，浏览器不支持 SVG 但仍然可以显示文本的事件。如果 javascript 已被禁用，则设备无法解释 Javascript 输出。因此，总是有必要根据需求及其用途来选择技术。
