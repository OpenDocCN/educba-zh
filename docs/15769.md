## 安全测试面试问题介绍

安全测试是一个过程，在此过程中，执行测试以检测保护数据和维护预期功能的安全机制中的任何缺陷。机密性、认证、授权、可用性、完整性和不可否认性是安全性的关键要素。如果你正在寻找一份安全测试的工作，那么你需要准备它的面试。在这里，我们将看到顶级安全测试面试问题及其答案，它们将帮助您在面试中取得成功。

在这篇 2022 年安全测试面试问题的文章中，我们将介绍 11 个最重要和最常见的安全测试面试问题。这些面试问题分为以下两部分:

### 第 1 部分-安全测试面试问题(基础)

第一部分包括基本的面试问题和答案:

#### Q1。什么是安全测试？

**答案:**

安全测试是通过执行测试用例来揭示信息系统安全机制缺陷的过程。测试人员扮演着攻击者和游戏场的重要角色，可以发现系统中与安全机制相关的缺陷。安全测试的目标是发现任何应用程序或系统中的漏洞，并保护它们的数据免受攻击者的攻击。

#### Q2。什么是 SQL 注入？

**答案:**

SQL 注入是一种代码注入技术，用于攻击数据驱动的系统，在这种系统中，恶意 SQL 语句被插入到条目字段中以供执行。它通常被认为是网站的攻击媒介，但也可以用来攻击任何类型的 sql 数据库。SQL 注入攻击允许攻击者伪造身份、篡改现有数据、导致否认问题(如使交易无效或改变余额)、允许完全泄露系统上的所有数据、破坏数据或使其不可用，以及成为数据库服务器的管理员。

#### Q3。什么是脆弱性？

**答案:**

脆弱性是任何系统的弱点。利用该漏洞，漏洞或攻击者可以很容易地攻击系统。可以通过执行安全测试来避免这种情况。如果不时地执行安全测试，那么出现漏洞的机会就会减少。为了保护系统免受漏洞或任何攻击，安全测试是必须的。

#### Q4。什么是入侵检测？

**答案:**

入侵检测是一种帮助确定可能的攻击并处理它们的系统。它从各种系统和来源收集信息。然后，它分析这些信息，并找到攻击系统的可能方式。它还检查异常活动。检查系统数据是否被更改。

#### Q5。安全测试的属性是什么？

**答案:**

安全测试有 7 个属性，即认证、授权、完整性、不可否认性、机密性、可用性和弹性。

*   **认证:**在认证中，检查用户的身份以提供对系统的访问。
*   **授权:**在授权中，检查用户访问资源的权限。
*   **完整性:**完整性确保系统的数据不被更改。
*   不可否认性:不可否认性是确保某人不能否认他们所做的行为。
*   **保密性:**它确保信息只对经过身份验证的用户保密。
*   **可用性:**它确保系统、应用和数据在用户需要时可供他们使用。
*   **弹性:**是指实体在不利的网络事件下持续交付预期成果的能力。

#### Q6。什么是 NIDS？

**答案:**

NIDS 代表网络入侵检测系统，用于分析整个子集上的通过流量，并与已知攻击进行匹配。如果发现任何漏洞，系统会自动向管理部门发送警报信息。此外，它还能够与防火墙等其他系统协同工作，以帮助更好地防范已知的攻击源。它可以从小型计算机到大型计算机。

### 第 2 部分-安全测试面试问题(高级)

现在让我们来看看高级面试问题:

#### Q7。什么是 cookie，并解释它的类型？

**答案:**

cookie 是从 web 服务器接收的一条信息，存储在 web 浏览器中，以后可以随时读取。Cookies 包含基于密码的信息、自动填充信息等。

有两种类型的 cookie 会话 cookie 和持久 cookie。

*   **会话 Cookie:** 会话 Cookie 是临时，仅在该会话中持续。
*   **持久 Cookie:** 持久 Cookie 存储在硬盘上，直到过期或被手动删除。

#### Q8。列出安全测试方法。

**答案:**

安全测试中使用白盒测试、黑盒测试和灰盒测试方法。

*   **白盒测试:**在这种测试方法中，所有的信息都提供给测试人员。
*   **黑盒测试:**在这种方法中，测试者不提供信息；他们可以在实时环境中测试系统。
*   **灰盒测试:**这种方法是白盒测试和黑盒测试的结合。它向测试人员提供部分信息，其余的由他们自己测试。

#### Q9。什么是 SSL？列出它的组件。

**答案:**

SSL 代表安全套接字层。它用于创建客户端和服务器之间的安全连接。SSL 记录协议、加密算法、握手协议和更改密码规范是 SSL 的组成部分。

#### Q10。什么是肥皂？

**答案:**

SOAP 代表简单对象访问协议，一种基于 XML 的协议。应用程序使用 SOAP 通过 HTTP 交换信息。

#### Q11。什么是 WSDL？

**答案:**

WSDL 代表 Web 服务描述语言。它是一种 XML 格式的语言，描述了 web 服务以及如何访问它们。
