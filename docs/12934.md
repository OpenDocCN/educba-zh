## CLI 简介

CLI 可以扩展为“命令行界面”，它只是一个用户界面，通过接受命令形式的输入，而不是点击和键入形式的输入，允许在系统上执行任何功能操作。这在传统的磁盘操作系统(DOS)中很常见，在 DOS 的命令提示符界面中输入命令。最近，该 CLI 被软件或系统管理员用于配置计算机系统，因为图形用户界面(GUI)中的类似动作会占用大量时间和手动动作。

### CLI 的目的是什么？

当基于图形用户界面的操作系统没有在早期的计算机中开发出来时，计算机使用一些基于命令的操作系统，如 MS-DOS、Apple-DOS、Unix 等。因此，人类与这些操作系统的任何交互都是通过一些命令来实现的，这些命令随后由解释器来解释，解释器充当人类命令和操作系统机器语言之间的中间件。

涉及 CLI 的一些最常见的日常任务是访问目录、创建新目录、创建文件、向文件写入文本、删除文件、显示文件夹中的文件等。

然而，即使在操作系统的现代 GUI 世界中，我们也有通过使用鼠标执行点击、访问网站等来与操作系统进行交互的最具交互性和最简单的方式。许多地方仍在使用 CLI 来执行一些基本任务。最常见和最频繁使用 CLI 的是程序员，他们使用该工具向他们的应用程序添加/安装组件。他们还使用 CLI 根据场景在服务器上自动执行一些与系统相关的配置，这样他们就不需要实际在服务器上工作。他们通常会保存一个命令文件或 PowerShell 脚本，以便在服务器上运行 CLI 命令并执行所需的操作/配置。

### 我们为什么使用它？

如今，众所周知，计算机已经从命令行工具发展到更高级、更复杂的图形用户界面，这些界面非常简单易用，任何用户都可以使用它们来操作计算机并完成工作。

然而，在计算机系统的配置中，CLI 仍然有着重要的用途。目前，软件程序员或系统管理员大多使用 CLI 来执行一些关键任务，如果使用 GUI，这些任务将耗费大量时间和精力。

为了更好和更详细地理解这一点，让我们分别看一下程序员和系统管理员使用 CLI 的情况。

### 系统管理员使用 CLI

这里的系统管理员指的是在 IT 世界中负责基础设施的人/团队。管理员可以是一个大型组织的整个团队，负责照看组织中运行的所有计算机系统，并维护每个系统的正常运行。系统管理员也可以指作为技术工程师为用户解决普通用户无法自行解决的系统相关问题的个人。这些系统管理员广泛使用 CLI 来检查系统的操作系统级配置，并直接在操作系统级执行一些任务，否则需要为 GUI 下载额外的软件。

除了这些任务之外，大型组织中的系统管理员还需要监控所有系统上安装的最新软件或操作系统更新，通过自动化在每个系统上定期执行一些任务，从而消除日常手动运行这些任务的需要。另一个主要用途是，只需在计算机上运行一些命令，就可以远程更改配置，而不需要在计算机所在的同一位置手动执行该任务。

所有这些都减少了使用 CLI 所需的大量时间和精力，并且还确保了即使在人工工作最少的大型组织结构中，也可以跟踪[安全风险并加以避免。](https://www.educba.com/security-risk-analysis/)

### 应用程序开发人员/程序员使用 CLI

应用程序开发人员也在日常工作中广泛使用 CLI 工具。如果一个普通用户通过 GUI 打开一个应用程序，他将花费很少的时间导航到图标并点击它。

现在想象一下，一个开发人员在一天中必须多次打开一个应用程序，每次他都必须花费大量的时间来定位图标并单击它来打开它。尽管如此，有人认为应用程序快捷方式可以放在桌面上，很容易访问。现在再想象一下，同一个程序员必须在一天中多次打开多个这样的应用程序。

为每个这样的程序创建一个快捷方式会使桌面变得笨拙，并且再次找到正确的图标会耗费时间。因此，程序员在计算机上打开应用程序或路径的理想方式是通过命令行或 CLI。程序员大多记得打开他们经常使用的应用程序的确切命令，也更喜欢通过命令直接打开目录路径。

程序员的另一个主要用途是为他们正在开发的应用程序编写代码。大多数情况下，程序员需要将对某些第三方库的引用添加到他们的应用程序中，以便与外部世界进行交互，这些外部世界可能包括连接到同一台计算机的打印机、存储应用程序数据的数据库，甚至是位于世界上任何地方的文件服务器，其中包含应用程序使用的一些文件。

现在，开发人员可以选择通过 GUI 添加这些引用/库，这将再次耗费一些时间。此外，当您在 GUI 中搜索库时，会根据搜索引擎的行为为您提供多个选项。有时这会导致开发人员安装不正确的引用。

但是，如果用户知道他们想要的确切库/引用，他们可以使用 CLI 在单个命令中安装该特定库。程序员使用的最常用和最强大的命令行解释器之一是 Nu-get 包管理器(NPM ),程序员在世界各地广泛使用它来非常有效地完成这项任务。

### CLI 命令

下面列出了一些最常见的 CLI 命令，即使是普通用户也会发现它们在工作中很有用:

| **命令名** | **使用** | **例子** |
| **cd** | 将目录更改为用命令指定的目录 | cd 桌面/ |
| **cd…** | 从当前目录上移一个目录 | cd… |
| **ls** | 列出工作目录中的所有文件和目录 | 限位开关（Limit Switch） |
| **mkdir** | 在工作目录中创建一个具有指定名称的新目录 | mkdir 个人 |
| **cp** | 将工作目录中的文件复制到指定目录 | cp learning.txt 个人/ |
| **清除** | 清除当前 CLI 窗口 | 清楚的 |
| **退出** | 退出 CLI 窗口 | 出口 |
| **帮助** | 获取关于所有命令及其用法的帮助 | 帮助 |
| **帮助** | 您还可以指定需要帮助才能获得详细描述的命令 | 帮助光盘 |

**Note**: All these commands are specific to Apple MAC OS. Their syntax may be different in other Operating System like Windows, but their function remains the same.

### CLI 的优点和缺点

与计算机世界中的任何其他工具/技术一样，CLI 也有其自身的优点和一些缺点。

#### 优势

以下是优点:

1.  如果用户知道或能够记住这些命令，那么与 GUI 相比，任务可以非常快速地执行。
2.  可以保存最后执行的命令的历史以供监控。
3.  可以组合命令来创建一个脚本，该脚本可以运行以执行一系列操作，并且可以被调度
4.  以便它可以定期运行，而无需任何人工干预。
5.  在 CLI 上运行的命令使用非常少的 RAM 内存和 CPU 周期来完成。
6.  CLI 命令可以远程部署在系统上，并在用户不知情的情况下执行。

#### 不足之处

以下是缺点:

1.  记住命令的确切语法可能很复杂。然而，现代 CLI 工具提供了自动补全功能，以便在用户忘记的情况下完成基本命令。
2.  仅仅通过查看书面命令，并不总是很清楚该命令将做什么。因此，不能进行任何更改，用户需要确保在执行之前知道该命令的结果。
3.  对于一个新用户来说，习惯 CLI 可能是一个挑战和困惑。
4.  用户可能需要学习大量命令来执行任务。然而，人们大多只记得他们在工作中经常需要的命令。

### 结论

通过这篇文章，我们能够了解很多关于 CLI 的内容，它是什么，它是如何使用的，它在哪里使用得更频繁，最重要的是，为什么在现代世界中 CLI 仍然被用于在计算机中绘制图形。总而言之，尽管计算机和操作系统中的用户界面从 CLI 到 GUI 已经有了很大的发展，但需要看到 CLI 仍然在操作系统级配置和监控中有很多用途和应用。

如果您正在使用 GUI 执行一些重复的任务，并且可以借助命令实现自动化，那么它还可以减少您的一些宝贵时间。总的来说，CLI 确实需要您记住一些命令，并且在开始时可能会有点混乱。但是，如果你足够耐心和敏锐，这个古老的工具仍然可以为你创造奇迹，帮助你节省一些宝贵的时间，并为你执行一些关键任务提供高可靠性。
