#### 1.OBIEE G 有哪几层？

回答；基本上有四种类型的层，分别是表示层、映射层、物理层、业务模型层

#### 2.什么是桥牌桌？

可能会出现两个不相关的表需要连接的情况。在这种情况下，使用一个连接其他两个表的表。第三个表称为桥表，两个表有相同的列。

#### 3.解释一些关于查询库工具的简介？

对于 OBIEE 或 Siebel 管理工具，使用查询存储库工具。它还使用户能够测试元数据存储库工具。此外，它还检查某些元数据对象之间的关系。这些对象通常包括随后应该将物理层表映射到哪个表示层。

#### 4.如何将表示层迁移到不同的服务器上？

表示层取决于服务器的基础。因此，表示层不能单独作为数据库的一个自治方面进行迁移。相反，建立从服务器到特定中央系统的 ODBC 或类似的基本连接，并且通过改变该数据库的逻辑层来执行来自另一服务器的显示语义

#### 5.什么是基于外部表和 Ldap 的安全性？

在 Siebel Analytics server 资料档案库中，不得保存口令和用户 ID。其他选项包括外部表和 LDAP。在外部表中存储了用于用户认证的用户 ID 和密码。在这个外部表中存储了关于不同表的信息和每个用户的访问信息。相反的是轻量级目录的访问协议。这就像对所有不同的目录和不同用户可以看到的目录的访问施加限制。

#### 6.有多少种认证类型？

有许多类型的认证，包括数据库认证、操作系统认证、LDAP 认证、操作系统认证。

#### 7.什么是展示目录 Oracle BI？

Oracle BI 演示目录存储您和其他用户使用 Oracle BI EE 和仪表板创建的对象，如分析和 KPI。用户拥有自己的个人文件夹(我的文件夹)，可以在其中存储对象。只有创建并保存该文件夹内容的用户才能访问个人文件夹中的对象。为了以最合理的方式组织您的内容，用户可以将子文件夹添加到他们的个人文件夹中。

#### 8.解释别名表的用途？

别名表是别名的物理种类的表。它是对物理表的引用，包含该列的所有定义和一些物理表属性。表源逻辑显示了逻辑对象到物理层的映射，以及到物理表、保存的过程和 select 语句的映射。别名表可以引用表中的任何逻辑源。设计物理层的一个重要部分是别名表。

#### 9.解释回购变量和会话的区别？

会话变量类似于动态存储库变量，因为它们通过初始化块获取值。但是，与动态存储库变量不同，会话变量没有被编程为初始化。当用户启动会话时，Oracle BI Server 会创建并初始化会话变量的新实例。存储库变量在任何时候都只有一个值。在管理工具的 Expression Builder 中，可以使用存储库变量来代替文字或常量。Oracle BI Server 将存储库变量值替换为元数据变量本身。

#### 10.什么是代理键？

人工生成的密钥，通常是一个数字，是一个替代密钥。替换键简化了聚合表中的这种连接，从事实表中删除了不必要的列，从而创建了一个更小的事实表。
