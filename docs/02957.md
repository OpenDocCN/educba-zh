## 大数据和数据挖掘的区别

大数据是指可以是结构化、半结构化和非结构化的海量数据。它由 5 个 Vs 组成，即

1.  量:指的是大数据的数据量或数据大小，在大数据中可能达到万亿分之一。
2.  多样性:指不同类型的数据，如社交媒体、网络服务器日志等。
3.  速度:指的是数据增长的速度，数据呈指数级增长，速度非常快。
4.  真实性:它是指数据的不确定性，就像社交媒体意味着数据是否可信。
5.  价值:它指的是我们存储和处理的数据的价值，以及我们如何从这些海量数据中获益。

可以分析大数据以获得洞察力，从而做出更好的决策和战略性业务举措。

### 什么是大数据？

通常，等于或大于 1 Tb 的数据称为大数据。分析师预测，到 2020 年，世界上每个人的数据将达到 5,200 Gbs。

**举例:**人们平均每天花费约 5000 万条推文，沃尔玛每小时处理 100 万笔客户交易。

**为什么大数据很重要？**

大数据的重要性不在于我们有多少数据，而在于你能从这些数据中得到什么。我们可以分析数据以减少成本和时间，做出明智的决策等。

**挑战**:

1.  高效地存储如此大量的数据。
2.  我们如何在给定的时间范围内处理和提取大量数据中的有价值的信息？

**解决方案:** Hadoop 和 Spark 框架

### 什么是数据挖掘(KDD)？

数据挖掘又称数据的知识发现，是指从大量数据即大数据中提取知识。主要用于统计学、机器学习和人工智能。这是“数据库中的知识发现”的步骤。

企业和政府共享他们收集的信息，目的是交叉引用这些信息，以找到关于他们数据库中被跟踪的人的更多信息。

数据挖掘的组成部分主要包括 5 个层次，即:-

1.  提取、转换数据并将其加载到数据仓库中
2.  存储和管理
3.  提供数据访问(通信)
4.  分析(流程)
5.  用户界面(向用户呈现数据)

#### 对数据挖掘的需求

分析存储的交易数据中的关系和模式，以获得有助于更好的业务决策的信息。

数据挖掘有助于信用评级、有针对性的营销、欺诈检测，如通过检查用户过去的交易，检查客户关系，如哪些客户是忠诚的，哪些客户将离开其他公司，哪些类型的交易可能是欺诈。

我们可以使用数据挖掘处理 4 种关系:

1.  类:它用于定位目标
2.  集群:它将把数据项分组到逻辑关系中
3.  关联:数据之间的关系
4.  顺序模式:预测行为模式和趋势。

**数据挖掘的挑战**

1.  挖掘数据库中不同类型的知识
2.  处理噪音和不完整数据
3.  数据挖掘算法的效率和可伸缩性
4.  处理关系和复杂类型的数据
5.  保护数据的安全性、完整性和隐私

### 大数据和数据挖掘的主要区别

下面是大数据和数据挖掘的区别如下

大数据和数据挖掘是两个不同的概念，大数据是指大量数据，而数据挖掘是指深入数据，从少量或大量数据中提取关键知识/模式/信息。

数据挖掘的主要概念是深入分析可进一步用于人工智能、预测分析等的数据模式和关系。但大数据中的主要概念是数据的来源、种类、数量以及如何存储和处理这些数据。
分析大数据以提供业务解决方案或制定业务定义，在决定增长方面发挥着至关重要的作用。

我们可以说，数据挖掘不需要依赖于大数据，因为它可以在少量或大量数据上完成，但大数据肯定依赖于数据挖掘，因为如果我们不能发现大量数据的价值/重要性，那么这些数据就没有任何用处。

### 大数据与数据挖掘对比表

| **功能** | **数据挖掘** | **大数据** |
| **焦点** | 它主要关注数据的许多细节 | 它主要关注数据之间的大量关系 |
| **视图** | 这是数据的特写视图 | 这是数据的大画面 |
| **数据** | 它表达了数据是什么 | 它表达了数据的原因 |
| **体积** | 它可用于小数据或大数据 | 它指的是大量的数据集 |
| **定义** | 这是一种分析数据的技术 | 这是一个概念，而不是一个精确的术语 |
| **数据类型** | 结构化数据、关系数据库和维度数据库。 | 结构化、半结构化和非结构化数据(在 NoSQL) |
| **分析** | 主要是统计分析，侧重于预测和发现小范围内的业务因素。 | 主要是数据分析，侧重于大范围内的商业因素的预测和发现。 |
| **结果** | 主要用于战略决策 | 仪表板和预测措施 |

### 结论

正如我们所见，大数据仅指大量数据，所有大数据解决方案都依赖于数据的可用性。它可以被认为是商业智能和数据挖掘的结合。数据挖掘在大数据上使用不同种类的工具和软件来返回特定的结果。主要是“大海捞针”

简而言之，大数据是资产，而数据挖掘是用于提供有益结果的资产管理者。
