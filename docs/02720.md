## ETL 面试问答介绍

ETL 代表提取-转换-加载。它是数据仓库的系统基础，在数据仓库中，从不同的来源提取数据，然后对数据进行转换，在转换后的数据中对数据进行增强或处理，从而以适当的表示格式保证数据的质量和一致性，最后将数据以表示格式加载到数据仓库中，以便帮助最终用户轻松获取信息并做出决策。

如果你想找一份和 ETL 相关的工作，你需要准备 2022 年的 ETL 面试问题。虽然每个面试都是不同的，工作范围也是不同的，但我们可以帮助你解决顶级 ETL 面试问题和答案，这将帮助你迈出这一步，并在面试中取得成功。

以下是 2022 年 ETL 面试中最常被问到的问题。

### 第 1 部分–ETL 面试问题(基础)

第一部分包括基本的面试问题和答案

#### 1.扩展数据仓库中的 ETL 系统？解释

**答案:**
提取-转换-加载(ETL)系统是数据仓库的基础。正确设计的 ETL 系统从源系统中提取数据，强制执行数据质量和一致性标准，确认数据以便可以一起使用单独的源，并最终以一种准备就绪的格式交付数据，以便应用程序开发人员可以构建应用程序，最终用户可以做出决策。

#### 2.ETL 系统在数据仓库中的意义？

**回答** :
删除错误，纠正缺失数据。它提供了对数据可信度的记录测量。捕获跨国数据的流动，以便妥善保管。调整来自多个来源的数据以便一起使用。构建可供最终用户工具使用的数据。

#### 3.ETL 系统中的日常流程是什么？

**回答** :
构建 ETL 过程的开发/测试/生产套件。理解各种后台数据结构的权衡，包括平面文件、规范化模式、XML 模式和星型连接(维度)模式。分析和提取源数据。构建一个全面的数据清理子系统。将数据结构化为多维模式，以便最有效地交付给最终用户、商业智能工具、数据挖掘工具、OLAP 立方体和分析应用程序。使用相同的技术将数据有效地交付给高度集中和高度分散的数据仓库。调优[整个 ETL 过程](https://www.educba.com/etl-process/)以获得最佳性能。

#### 4.ETL 系统中的数据格式？

**答**:
ETL 中有多种数据格式，包括平面文件、XML 数据集、独立的 DBMS 工作表、规范化的实体/关系(E/R)模式和维度数据模型。

#### 5.ETL 系统中的数据剖析？

**回答** :
数据概要分析是对数据源的质量、范围和上下文的系统检查，以允许构建 ETL 系统。在一个极端的情况下，一个非常干净的数据源在到达数据仓库之前已经得到了很好的维护，只需要最少的转换和人工干预就可以直接加载到最终的维度表和事实表中。

#### 6.什么是 ETL 验证器？

**回答** :
ETL Validator 是一个数据测试工具，它极大地简化了数据集成、数据仓库和数据迁移项目的测试。它利用我们获得专利的 ELV 架构从数据库、平面文件、XML、Hadoop 和 BI 系统等数据源中提取、加载和验证数据。

### 第 2 部分–ETL 面试问题(高级)

现在让我们来看看高级面试问题。

#### 7.ETL 系统中有哪些操作？

**回答** :
以下三个操作:从您的跨国系统(可以是 Oracle、Microsoft 或任何其他关系数据库)中提取数据，通过执行数据清理操作来转换数据。加载是将数据写入目标数据库的过程。

#### 8.列举一些市场上最常用的 ETL 工具？

**答:**
市场上有各种各样的 ETL 工具，但是有一些最受欢迎的 ETL 工具

*   Oracle Warehouse Builder (OWB)
*   SAP 数据服务
*   IBM Infosphere 信息服务器
*   SAS 数据管理
*   电力中心信息
*   数据 ETL 的灵丹妙药
*   数据迁移器(IBI)
*   SQL Server 集成服务(SSIS)

#### 9.ETL 团队的角色和职责是什么？

**回答:**
最高层的 ETL 团队的角色是构建数据仓库的后台。

*   将数据最有效地交付给最终用户工具
*   在清理和符合步骤中增加数据的价值
*   保护和记录数据的传承
*   从原始来源提取数据
*   质量保证和清洁数据
*   整合数据中的标签和测量以实现
*   原始来源的一致性
*   以查询工具可以使用的物理格式传递数据，
*   报告编写器和仪表板。

#### 10.影响分析在 ETL 系统中的作用是什么？

**答:**
影响分析检查与对象(在这种情况下是表或列)相关的元数据，并确定其结构或内容的变化会影响什么。更改数据暂存对象可能会破坏对正确加载数据仓库至关重要的进程。允许对数据暂存对象进行临时更改不利于项目的成功。一旦在临时区域中创建了表，就必须在对其进行任何更改之前执行影响分析。许多 ETL 工具供应商提供了影响分析功能，但是在 ETL 产品概念验证过程中，这一功能经常被忽略，因为它是一个后台功能，并且在数据仓库启动、运行并开始发展之前
并不真正重要。

#### 11.如何处理定长平面文件？

**答案:**
一个定长的文件布局应该包括文件名，字段开始的地方；它的长度；及其数据类型(通常是文本或数字)。有时，会提供结束位置。如果不是，您必须根据它的开始位置和长度计算每个字段的结束位置，如果您的 ETL 工具需要的话。在大多数 ETL 工具中，您很可能需要手动输入一次平面文件的文件布局。输入布局后，该工具会记住该布局，并在每次与实际的平面文件进行交互时期待相同的布局。如果文件布局改变或者数据偏离了指定的位置，ETL 过程必定会失败。处理固定长度的平面文件时，请尝试验证文件中数据的位置是否准确。验证位置的快速检查是测试任何日期(或时间)字段，以确保它是一个有效的日期。如果位置发生变化，日期字段很可能包含字母字符或不合逻辑的数字。可以用同样的方式测试具有非常特殊的域的其他字段。XML 提供了更具体的验证能力。如果数据验证或一致性是一个问题，尝试说服数据提供者以 XML 格式交付数据。

#### 12.为什么要在数据仓库中进行实时 ETL？

**答:**
数据仓库需要为业务决策者提供一组稳定的数据，提供一个可靠的信息平台。对于针对闪烁数据库的最新报告，业务用户被定向到运行业务的生产应用程序。因此，用户不得不去数据仓库查看昨天发生的业务的历史情况，并且不得不查看许多 OLTP 系统以了解今天发生的情况。商业用户从未完全接受这种划分。数据仓库当然绝对需要来自运营的持续的客户信息流，但是越来越多的运营系统也依赖于客户信息的数据仓库丰富。因此，可以预见的是，组织已经开始探索能够支持更一般化的集成场景的体系结构替代方案，以便在应用程序之间移动操作数据，同时以越来越大的紧迫性将操作数据移入和移出仓库。
