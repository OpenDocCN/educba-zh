### Java 是什么？

Java 是詹姆斯·高斯林设计的面向对象的程序设计。它由 Oracle 公司(当时是 Sun 公司)开发和维护。它是一种基于类的通用编程语言，具有并发编程特性。它还具有多线程功能。它是一种静态、安全、强类型的编程语言。它的文件扩展名是。它最早出现在 1995 年。它旨在开发可以一次编写并在任何地方运行的应用程序。它最受客户机-服务器类型的应用程序的欢迎。它是在 GNU 通用公共许可证和 Java 社区进程下授权的。Java 的最新版本是 8，于 2014 年 1 月 14 日发布，Java 9 于 2017 年 7 月 27 日发布，Java 10 于 2018 年 3 月发布。

### Java 8 和 Java 9 的主要区别

Java 8 vs Java 9 都是市场上的热门选择；让我们来讨论一些显著的区别:

1.  Java 8 和 Java 9 是 Java 编程语言下的两个不同版本，其中 Java 9 支持脚本语言和面向对象编程语言。
2.  Java 8 有 lambda 函数和函数式编程特性，而 Java 9 有不同的 API 更新和新特性。
3.  Java 8 更新了对 Java 类型和无符号整数运算的注释支持，而在 Java 9 中，轻量级 JSON APIs 用于生成和使用文档形式的 JSON 数据流。
4.  Java 8 更新了 JNI 库和 Nashorn 引擎，支持 JVM 中的 JavaScript 编译，而在 Java 9 中，更新了 HTTP 2.0 client，取代了 HttpUrlConnection is legacy。
5.  Java 8 具有启动 JavaFX 应用程序的特性，而在 Java 9 中，对分段代码缓存机制进行了性能更新。
6.  Java 8 提供了语言级的注释支持，而在 Java 9 中，存在一个智能编译器特性，可以用来构建更大的项目。
7.  Java 8 提供了一个更新，支持在 Windows XP 操作系统上运行的环境，而在 Java 9 中
8.  Java 8 已经按照 JEP 122 规范移除了永久生成，而 9 具有用于脚本目的的 REPL 特性，它将 Java 和 REPL 结合在一起成为 jshell。
9.  Java 8 有流 API 和与集合相关的批量操作，而 Java 9 有更少的碎片和更短的扫描时间以及更多的扩展。
10.  Java 8 改进了 Java Time API 和 Java IO，而 Java 9 更新了 Money 和 Currency API。
11.  Java 8 还有其他特性，比如对 Comparator 接口的扩展，而 Java 9 改进了 Javadoc 文档和 Java 平台模块系统。
12.  Java 8 从它的 Java 支持中移除了 JDBC-ODBC 桥，而 Java 9 有交互式的 REPL Jshell 来探索 API 特性和其他新的语言特性。
13.  Java 8 去掉了 PermGen 内存特性，而 Java 9 有集合工厂方法来直接填充集合或列表的元素。
14.  Java 8 在一些包装类(如 Integer、Double 和 Long)中增加了一些方法，如 min()、max()和 sum()，而 Java 9 有私有接口方法，在 Java 9 之前的接口中不允许使用私有方法。
