## Hadoop 是开源的吗？

Hadoop 是开源的，为大型数据集提供空间，它存储在具有相似性的软件组上。Hadoop 是 Apache 的一个项目，它由不同的用户使用，也由一个大型社区支持代码贡献。许可证是许可证 2.0。免费的 Hadoop 效率不高，因为其成本来自运营和维护成本，而不是安装成本。

### Hadoop 的特性

我们已经学习了 Hadoop 开源简介，现在我们正在学习 Hadoop 的特性:

<small>Hadoop、数据科学、统计学&其他</small>

#### 1.开放源码

Apache Hadoop 最吸引人的地方在于它是开源的。意味着 Hadoop 开源是免费的。任何人都可以下载并个人或专业使用它。如果有任何花费的话，那很可能是用于存储大量数据的商用硬件。但这仍然使得 Hadoop 价格低廉。

#### 2.商用硬件

Apache Hadoop 运行在商用硬件上。商用硬件意味着您的基础架构不会依赖于任何单一供应商。任何以较低成本提供硬件资源(如存储单元、CPU)的公司。肯定可以跳槽到这样的公司。

#### 3.低成本

作为 [Hadoop 框架是](https://www.educba.com/hadoop-framework/)基于商用硬件和开源软件框架。当在组织中采用它或为您的项目进行新的投资时，它降低了成本。

#### 4.可量测性

系统或应用程序的特性是处理更大量的工作，或易于扩展，以响应对网络、处理、数据库访问或文件系统资源的需求增加。Hadoop 是一个高度可扩展的存储平台。可伸缩性是事物随着时间的推移适应变化的能力。修改通常涉及成长，所以一个大的内涵是适应将是某种扩展或升级。Hadoop 是水平可扩展的。这意味着您可以向现有基础架构添加任意数量的节点或机器。假设您在集群中处理 15 TB 的数据和 8 台机器。您预计下个月会有 6 TB 的数据。但是您的集群只能多处理 3 TB。Hadoop 为您提供了水平扩展功能，这意味着您可以根据您的集群需求添加任意数量的系统。

#### 5.高度稳健

Hadoop 的容错特性让它真的很受欢迎。Hadoop 为您提供了复制因子等功能。这意味着您的数据被复制到复制因子定义的其他节点。您的数据对其他节点来说是安全的。如果发生群集故障，数据将自动传递到另一个位置。这将确保数据处理继续进行，没有任何故障。

#### 6.数据多样性

Apache Hadoop 框架允许您处理任何大小的数据和任何种类的数据。 [Apache Hadoop 框架](https://www.educba.com/hadoop-framework/)帮助您[处理大数据](https://www.educba.com/introduction-to-big-data/)。您将能够存储和处理结构化数据、半结构化数据和非结构化数据。您不受任何数据格式的限制。你不受任何数据量的限制。

#### 7.大数据的多种框架

有不同的工具用于不同的目的。Hadoop 框架有各种各样的工具。Hadoop 框架分为两层。存储层和处理层。存储层称为 Hadoop 分布式文件系统，处理层称为 Map Reduce。在 HDFS 之上，你可以集成到由 [Hadoop 集群](https://www.educba.com/what-is-hadoop-cluster/)支持的任何类型的工具中。Hadoop 可以与多种分析工具集成以充分利用它，如用于机器学习的 Mahout、用于分析和可视化的 R 和 Python、用于实时处理的 Python、Spark、用于 NoSQL 数据库的 MongoDB 和 HBase、用于 BI 的 Pentaho 等。可以集成到 Apache Hive 和 Apache Pig 这样的数据处理工具中。它可以与 Apache Sqoop [和 Apache Flume](https://www.educba.com/apache-flume/) 等数据提取工具集成。

#### 8.快速处理

虽然传统的 ETL 和批处理过程可能需要数小时、数天甚至数周时间来加载大量数据，但实时分析这些数据的需求正变得日益重要。Hadoop 非常擅长大容量批处理，因为它能够进行并行处理。Hadoop 执行批处理的速度比单线程服务器或大型机快 10 倍。数据处理工具通常位于数据所在的同一台服务器上，因此数据处理速度要快得多。如果您正在处理大量的非结构化数据，Hadoop 可以在几分钟内高效处理 TB 级数据，在几小时内处理 Pb 级数据。

#### 9.使用方便

Hadoop 框架基于 Java API。在接受 Hadoop 的同时，作为开发者并没有太大的技术差距。Map Reduce 框架是基于 Java API 的。你需要代码，在 JAVA 本身上写算法。如果您正在使用像 Apache Hive 这样的工具。它基于 SQL。任何具有数据库背景的开发人员都可以轻松采用 Hadoop，并以 Hive 为工具进行工作。

### 结论–Hadoop 是开源的吗？

如今，数字世界中存在 2.7 泽塔字节的数据。未来十年，大数据将主导数据存储和处理环境。数据将成为业务增长的核心模式。需要一种工具来满足所有这些需求。Hadoop 非常适合存储和处理大数据。大数据 Hadoop 的所有上述特性使其对于广泛接受的 Hadoop 来说非常强大。大数据将成为所有工具的中心。Hadoop 是处理大数据的解决方案之一。
