## 卡珊德拉简介

Cassandra 是来自 Apache 的开源数据库应用程序，它不需要 SQL 来获取、查看、更新、删除数据库或数据。因此它被称为 NoSQL 类型的数据库应用程序。可以从各种节点远程和直接访问和管理该数据库。它因其独有的特性而广受欢迎，如更高的性能、扩展和可伸缩性的灵活性、更低的用户延迟、对等体系结构、结构化的无模式、非常少的工作量、维护所需的成本、更快的处理速度、易于学习和使用、简单的数据复制过程等。

### 优势

以下是 Cassandra 的不同优势:

*   **易于数据复制:**如果任何特定节点出现故障，It 部门都可以轻松复制部署在该节点上的数据
*   **数据的分布:**这可以用来在各个节点存储和定位数据，有很大的优势
*   **对等架构:**它不遵循典型的主从概念，而是遵循对等架构，其中每个节点平等参与，承担同等责任。因此，不存在单点故障。
*   **高可用性:**由于不存在单点故障，因此它具有高可用性
*   **高性能:**对于通常处理大规模数据的大型组织来说，它必须非常可靠
*   **无模式概念:**无模式概念是指可以根据我们的需求在内部创建模式

### 我们为什么需要卡珊德拉？

我们通常需要 Cassandra，原因如下:

*   **为复杂问题提供即时简单的解决方案**–它为日志记录和指标收集等复杂问题提供简单高效的解决方案。它为集中式日志记录提供了一个简单、时尚和优雅的特性。
*   **低维护开销—**Cassandra 数据库几乎可以独立运行，因此减少了维护依赖性。
*   **维护成本低–**由于几乎不需要任何维护，因此 it 成本会自动降低。
*   **简单易学–**通常，学习 Cassandra 并为生产水平做好准备所需的时间较少。此外，它有大量的开发人员提供社区支持和精通的文档，这进一步简化了学习曲线。
*   **快速读写一个特性——**使用 Cassandra 在磁盘上读写是非常快速和可行的
