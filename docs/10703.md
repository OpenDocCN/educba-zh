### 第 1 部分 Magento 面试问题(基础)

这第一部分包括基本的 Magento 面试问题和答案

#### Q1。用它的架构定义 Magento？

**回答:**
Magento 是一个基于开源技术创建的电子商务平台，它为在线企业提供了对其电子商务商店的内容、外观和功能的灵活性和控制。它的架构是一个 PHP MVC(模型-视图-控制器)应用程序，所有的控制器在一个文件夹中，所有的模型在另一个文件夹中，等等。基于它们的功能，文件被分组在一起，在 Magento 中被称为模块。Magento 有多种版本，包括:

*   Magento 企业
*   走
*   Magento 社区

#### Q2。Magento 用的是哪种技术？

**答案:**
这是 Magento 面试中最常见的面试问题。Magento 使用了多种技术，包括 web 服务器和数据库组件。它的 web 服务器是用 PHP 脚本语言创建的，而数据库部分由 MySQL 负责。MySQL 使用的数据模型是基于 EAV 的，即实体-属性-值模型，其主要特点是将数据对象存储在树结构中。使用树状结构的一个好处是，数据结构现在可以在不改变数据库定义的情况下变化。但与此同时，基于 EAV 的数据模型方法是昂贵的，只应在开发人员不确定表中的字段数量以及将来可能发生变化时考虑。EAV 技术的一个主要优点是，它允许开发人员向表中添加无限的列。实际上，一个表将保存所有的属性数据，而其他表将保存每个提到的属性的实体和值。

#### Q3。Mage::getSingleton()和 Mage::getModel()有什么区别？

**答:**
Mage:getSingleton()确实只为给定的类创建一个对象。如果已经为同一个类创建了对象，它将不会再次创建它。顾名思义，它利用了单例设计模式。单例设计模式的使用适用于这样的场景:开发人员希望只创建一个对象一次，然后继续使用同一个对象，修改并获取它，但不会创建新的对象。会话对象可能是这种模式的一个理想例子，用户可以在应用程序的不同页面上添加或删除会话值，这样它可以保留值，但不会创建新的会话对象。因为创建新的会话对象意味着丢失最后的更改。这是面试中基本的 Magento 面试问题。

Mage::getModel()确实创建了一个新对象，每次都调用它的类。与单例方法不同，它可以在开发人员需要来自数据库的新数据的场景中使用。

#### Q4。Magento 中的 EAV 和平面模型有什么区别？

**回答:**
这两种模型在存储数据的模式和策略上都有很大的不同。
EAV 更像是数据库模型的规范化形式。基于 EAV 的方法的主要特征是每个列值都存储在各自的数据类型表中。产品和型号相关数据可以存储在各自的数据类型表中。产品 ID、产品名称、产品价格及其创建日期将存储在各自的数据类型表中。这意味着，开发人员需要连接多个表来获得产品细节，这实际上也给基于 EAV 的方法带来了更多的复杂性。
与基于 EAV 的方法不同，平面模型使用单个表。它没有被规范化，不像它的 EAV 对应物，因此使用更多的数据库空间。平面模型的一个主要特点是性能，因为只有一个查询可以加载整个产品，因此开发人员不需要进行复杂的连接查询来获取产品细节，这是 Magento 面试中最常见的问题。

#### Q5。解释一下 Magento 中手柄的概念？

**答案:**
Magento 中一个页面的结构由句柄负责。句柄将决定需要显示哪个块及其位置。每个页面都会调用句柄，其中每个页面请求都可以有多个唯一的句柄。为了处理产品页面，使用了多个产品句柄，这属于一个虚拟产品类型。如果需要调用简单产品的页面，将调用其对应的句柄，而对于虚拟产品，将需要其虚拟句柄。索引的句柄可以通过组合集合(即前端名称)和动作控制器(即索引)来生成。只有在访问句柄的特定 URL 时，才会调用句柄。

#### Q6。Magento 中的 codePool 是什么？

**答案:**
为了在 app/etc/modules/sample _ module . XML 中注册一个新的模块，开发者必须指定一个名为 codePool 的标签文件。有 3 种类型的代码池可用，从核心开始，由核心团队使用，社区代码池由第 3 <sup>第</sup>方扩展使用，本地代码池用于小时模块开发。如果客户需要，本地代码池也可以覆盖核心和社区代码池。

代码池位于/app/code 目录下。

#### Q7。Magento 中有多少种会话类型？

**答:**
使用了两种类型的会话，客户会话和结账会话。所有与客户相关的数据都与客户会话锁定，而与订单和报价相关的数据则存储在结帐会话中。会话类型是单独使用的，因为订单发出后，结帐会话数据信息需要刷新。因此，为订单和客户相关数据建立两个不同的会话是有意义的。示例–客户会话的名字将是$_SESSION['客户']['名字']。类似的例子也适用于结帐会话。这是面试中被问到的有用的问题。

### 第 2 部分 Magento 面试问题(高级)

现在让我们来看看高级 Magento 面试问题。

#### Q8。Magento 有哪些不同的性能提升方式？

**答:**
有几种方法可以提高 Magento 的性能，下面列出了其中一些:

*   禁用任何未使用的模块
*   启用 Magento 缓存
*   优化图像
*   禁用 Magento 日志
*   将外部 CSS/JS 合并成一个文件
*   MySQL 查询缓存
*   启用 gzip 压缩

#### Q9。有哪些优化 Magento 环境和配置的方法？

**答:**
有几种方法可以用来优化 Magento 环境，下面列出了其中一些方法:

*   Magento 应用程序可以使用云计算进行优化。
*   JavaScript 和 CSS 可以合并在一个文件中，这大大减少了加载时间，因为它现在只加载一个文件。
*   为了优化配置，不要将未使用的文件存放在 [web 服务器](https://www.educba.com/web-services-interview-questions/)上。
*   识别并禁用未使用的 Magento 模块。
*   为了提高性能，千万不要在同一台机器上运行 MySQL 和 web server。
*   升级到[最新的 Magento 版本](https://www.educba.com/magento-versions/)，性能更好。
*   Magento 使用复杂的数据库，因此在专用服务器上运行数据库将会极大地提高性能。

#### Q10。什么是 Magento 支付网关？

**答:**
支付网关安全地处理客户和商户之间、商户和支付处理器之间的信用卡数据。它就像一个检查站，防止客户试图收集客户的个人和财务信息，并充当商家和主办银行之间的中间人。

#### Q11。如何在 Magento 向客人展示精选的产品？

**答:**
要仅显示所选产品，请转到工具栏，您会看到:

app/code/core/Mage/Catalog/Block/Product/List/toolbar . PHP

然后转到下面的方法:

`Public function setCollection($collection);`

在那里你会发现下面这段代码:

`$limit= (int)$this->get Limit();
If ($limit) {
$this ->_collection->setPageSize($limit);
}`

在这段代码中，您必须更改变量$limit，并且应该在本地池中覆盖该块。不要直接在核心中进行更改。要检查顾客是否是客人，请使用以下信息:

`Mage:: getSingleton(‘customer/session’) -> isLoggedIn()`

#### Q12。全渠道零售和多渠道零售有什么区别？

**答案:**
多渠道零售为企业提供不同的平台，如网络、手机、电视等。以促进产品销售的增长。

全渠道比多渠道领先一步，无论客户使用何种渠道或媒介，全渠道都能提供相同的品牌体验。

#### Q13。你将如何向 Magento 添加博客？

**回答:**
增加你的搜索引擎排名和销售博客很重要。在 Magento 中添加博客有两种方式:

*   使用默认的 Magento 博客扩展。
*   创建自定义博客模块。

一旦完成，安装博客扩展并配置它。

#### Q14。付费和抓取信息有什么区别？

**答:**
当发票需要设置为已付时使用 Pay。这在支付在线或离线时使用。当实际支付处理在线时使用 Capture，而 capture()方法用于支付。一旦这个 done pay()方法被调用。
