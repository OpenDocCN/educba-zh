## 吉拉组件介绍

以下文章提供了吉拉组件的概述。吉拉是一个项目管理工具，用于跟踪整个项目，它为用户提供不同的功能；其中该组件是吉拉提供的功能之一。该组件只是项目的一个子部分。换句话说，我们可以说，通过使用组件，我们可以将特定项目中的特定问题分组，并制作小组件。有了更好的理解，我们可以考虑描述工作结构的不同例子，比如特定项目中的数据对象、插件、API 和服务。

### 什么是吉拉组件？

吉拉是一个非凡的编程工具，它允许你调整应用程序，将你所在集团的业务流程整合到设备中。由于吉拉适应性强的工作流程，吉拉的工作流程可能会异常活跃。尽管如此，除了简单的工作过程之外，互动(和你的团队的客户体验)还有其他的东西。

<small>网页开发、编程语言、软件测试&其他</small>

从客户如何制作新票，到描述哪些信息焦点是可访问的，以及在工作完成后可以制作哪些报告，吉拉的设计企业(可能)有各种各样的组成部分来合作，形成一个总的循环。另外，这些设置允许我们(无论如何)为使用框架的团队决定客户体验。万一你还没有遇到它，当尝试使用吉拉时，当风险安排没有检查出来，或者当领域模糊或有过多的选择时，这是令人困惑的。

项目主席和吉拉经理必须看到每个设置的最佳实践，以及每个设计如何合作来表征周期和体验。

### 如何使用吉拉组件？

每个团队根据他们的任务使用组件；例如，我们可以考虑软件开发团队有不同的想法，所以根据他们的需求，他们使用其他组件，吉拉提供了如下不同类型的组件。

#### 1.特定项目

这是一个临界点；每个项目中的组件应该是唯一的，并且可以由项目管理员独立管理。在这里，项目管理员可以创建新的组件；他们还可以编辑现有成员。

#### 2.元件引线

在使用组件之前，我们需要定义组件的领导，这样我们可以快速确定谁是特定组件的管理员。因此，根据这些信息，我们可以轻松地设置仪表板并管理他们的工作。

#### 3.自动分配

或者，你可以在发行另一只债券时，为每一部分安排一个令人兴奋的默认受托人。项目主管可以让一个类似指派的默认被任命者作为任务的剩余部分(像风险投资负责人)，让问题不被指派，或者决定让案例直接进入被指派部分的信息。

### 当使用组件时？

我们的头脑中可能充满了如何利用这些元素的想法。然而，就像吉拉的一切一样，调整应用程序的能力给了客户难以置信的力量。伴随着非凡的力量而来的是制造巨大灾难和令人失望的客户体验的能力。

因此，在你开始向你的项目添加新的组成部分之前，暂停一分钟来考虑上面明确的组成部分，以及你需要如何利用这些元素来帮助你的互动；同样，你需要确保你的事业中的组件价值对客户来说是好的。

在安排吉拉时，智能方法始终是最好的。

在考虑为您的任务组件工作的选择时，您应该考虑随之而来的问题:

*   你的小组做的各种工作(或给出类型)
*   他们的周期(或工作流程)

你的同事处理问题的各种能力。然后，利用这些数据对即将到来的问题(或需求)进行有见地的划分，以帮助您的团队有效地描述所有权并更快地开始工作，规划更重要的启示，或通过区分需要特殊考虑或额外互动步骤的问题来帮助支持您的团队。

组件的示例用例如下:

应用程序的功能组件:

*   图形用户界面
*   数据库ˌ资料库
*   安全组件
*   API 服务

我们如何在特定项目中进行分组，如下所示:

*   应用程序的消息传递
*   应用程序的基础结构

这样，我们可以根据自己的需求使用不同的组件。

*   **项目领导:**在这里，我们可以分配与特定项目相关的项目领导。
*   **组件领导:**创建问题后，我们可以分配组件领导。
*   **默认:**默认选项在吉拉也可用。
*   **未分配:**我们可以允许未分配的问题。
